Μία υποομάδα ενεργειών εποπτείας μπορεί να πραγματοποιηθεί απευθείας από το νήμα σχολίων, χωρίς να χρειάζεται να μεταβείτε στη σελίδα Εποπτείας Σχολίων.

Όταν είστε συνδεδεμένοι, κάντε κλικ στο κουμπί επεξεργασίας στην πάνω δεξιά γωνία ενός σχολίου. Ως
συντονιστής θα πρέπει να έχετε τις ακόλουθες επιλογές:

- **Καρφίτρωση** αυτού του σχολίου
- **Διαγραφή** αυτού του σχολίου
- **Διαγραφή** αυτού του σχολίου + **Απαγόρευση χρήστη** (Μόνιμη ή Σκιώδης, περισσότερες λεπτομέρειες αργότερα)
- **Επεξεργασία** αυτού του σχολίου
- **Κλείδωμα** ή **Ξεκλείδωμα** αυτού του σχολίου (περισσότερες λεπτομέρειες παρακάτω)
- Σημειώστε ότι αυτό το σχόλιο είναι **Εγκεκριμένο** (εμφανίστε το) ή **Μη Εγκεκριμένο** (αποκρύψτε το)
- Σημειώστε αυτό το σχόλιο ως **Ανεπιθύμητο** ή **Όχι Ανεπιθύμητο**

### Κλείδωμα ενός σχολίου

Το κλείδωμα ενός μεμονωμένου σχολίου εμποδίζει οποιεσδήποτε νέες απαντήσεις σε αυτό και επίσης εμποδίζει το ίδιο το σχόλιο να επεξεργαστεί ή να διαγραφεί μέχρι να ξεκλειδωθεί. Αυτό ισχύει για όλους, συμπεριλαμβανομένων των διαχειριστών και των συντονιστών. Εάν χρειαστεί να επεξεργαστείτε ή να αφαιρέσετε ένα κλειδωμένο σχόλιο, ξεκλειδώστε το πρώτα, κάντε την αλλαγή και επανακλειδώστε το αν το επιθυμείτε.

Ένα εικονίδιο λουκέτου εμφανίζεται στην πάνω δεξιά γωνία ενός κλειδωμένου σχολίου ώστε οι αναγνώστες να βλέπουν με μια ματιά ότι το νήμα είναι κλειστό. Οι επιλογές Μετάβασης και Διαγραφής κρύβονται για κλειδωμένα σχόλια τόσο στο widget σχολίων όσο και στο public API (`PATCH` και `DELETE` επιστρέφουν `code: 'locked'` εάν κληθούν για κλειδωμένο σχόλιο).

Υπάρχουν δύο προοδευμένες εξαιρέσεις που παρακάμπτουν το κλείδωμα, επειδή διαφορετικά θα άφηναν ορφανά δεδομένα πίσω: όταν ένας χρήστης διαγράφει ολόκληρο τον λογαριασμό του (τα σχόλιά του καθαρίζονται ανεξαρτήτως της κατάστασης κλειδώματος), και όταν ένας συντονιστής απαγορεύει έναν χρήστη με την επιλογή "διαγράψτε όλα τα σχόλια από αυτόν τον χρήστη" (η σάρωση αφαιρεί τα κλειδώματα).

### Κλείσιμο νημάτων σχολίων

Οι συντονιστές και οι διαχειριστές μπορούν να κλειδώσουν, ή να κλείσουν, νήματα σχολίων επιλέγοντας `Close Thread` στο μενού με τις τρεις τελείες στην κορυφή
της περιοχής σχολίων, εφόσον είναι συνδεδεμένοι. Μπορούν να επιλέξουν `Re-Open Thread` αργότερα, οποιαδήποτε στιγμή, για να ανοίξουν ξανά τη δυνατότητα σχολιασμού.

Το κλείσιμο ενός νήματος σχολίων εμποδίζει νέα σχόλια, αλλά εξακολουθεί να επιτρέπει την ψηφοφορία και στους χρήστες να διαγράψουν τα σχόλιά τους εάν το επιθυμούν.

Το κλείσιμο και το ξανά-άνοιγμα νημάτων σχολίων επηρεάζει άμεσα όλους τους χρήστες που βλέπουν το νήμα.

Μπορείτε επίσης να ορίσετε ένα νήμα μόνο για ανάγνωση, το οποίο αφαιρεί και τις επιλογές ψηφοφορίας και διαγραφής, δημιουργώντας έναν κανόνα εξατομίκευσης ειδικά για εκείνη τη σελίδα.

### Ενημέρωση σε πραγματικό χρόνο

Όλες αυτές οι ενέργειες θα ενημερώσουν τα νήματα σχολίων άλλων χρηστών αμέσως χωρίς να χρειάζεται να
φορτώσουν ξανά τη σελίδα. Ωστόσο, οι ενέργειες του συντονιστή όπως η απόκρυψη ενός σχολίου ή η σήμανσή του ως ανεπιθύμητο, δεν
αφαιρούν το σχόλιο από **την οθόνη του συντονιστή**, ώστε, εάν χρειαστεί, να μπορούν γρήγορα να αναιρέσουν την ενέργεια. Για να υποδειχθεί ότι ένα σχόλιο
είναι κρυμμένο, θα επισημανθεί σε σύγκριση με τα άλλα σχόλια (το χρώμα επισημάνσεως εξαρτάται από τον λόγο αφαίρεσης).

For example, given users `A (commenter)`, `B (Moderator 1)`, and `C (Moderator 2)`.

...and the following scenario:

1. `User B (Moderator 1)` αποκρύπτει ένα σχόλιο.
2. Για `User A (commenter)` αυτό το σχόλιο αποκρύπτεται αμέσως.
3. Για `User C (Moderator 2)` αυτό το σχόλιο αποκρύπτεται αμέσως.
4. Για τον χρήστη που έκανε την αλλαγή, `User B (Moderator 1)`, το σχόλιο παραμένει στην οθόνη του, αλλά επισημαίνεται ως αφαιρημένο. Έχουν την επιλογή να αναιρέσουν την ενέργειά τους, οπότε οι άλλοι χρήστες θα δουν ξανά την ενημέρωση, σε πραγματικό χρόνο.
