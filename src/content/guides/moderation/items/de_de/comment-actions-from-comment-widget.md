Ein Teil der Moderationsaktionen kann direkt im Kommentar-Thread selbst durchgeführt werden, ohne zur Seite "Kommentar-Moderation" gehen zu müssen.

Wenn Sie eingeloggt sind, klicken Sie auf die Bearbeiten-Schaltfläche oben rechts an einem Kommentar. Als Moderator sollten Ihnen folgende Optionen zur Verfügung stehen:

- **Anheften** dieses Kommentars
- **Löschen** dieses Kommentars
- **Löschen** dieses Kommentars + **Nutzer sperren** (permanent oder Shadow, mehr Details später)
- **Bearbeiten** dieses Kommentars
- Diesen Kommentar als **Genehmigt** (anzeigen) oder **Nicht genehmigt** (verbergen) markieren
- Diesen Kommentar als **Spam** oder **Kein Spam** markieren

### Kommentar-Threads schließen

Moderatoren und Administratoren können Kommentar-Threads sperren bzw. schließen, indem sie, sofern sie eingeloggt sind, im Drei-Punkte-Menü oben im Kommentarbereich `Close Thread` auswählen. Später können sie jederzeit `Re-Open Thread` auswählen, um Kommentare wieder zuzulassen.

Das Schließen eines Kommentar-Threads verhindert neue Kommentare, erlaubt jedoch weiterhin Abstimmungen und den Nutzern, ihre Kommentare bei Bedarf zu löschen.

Das Schließen und Wiederöffnen von Kommentar-Threads wirkt sich sofort auf alle Benutzer aus, die den Thread gerade ansehen.

Sie können einen Thread auch als schreibgeschützt markieren, wobei dadurch auch die Optionen zum Bewerten und Löschen entfernt werden, indem Sie eine Anpassungsregel speziell für diese Seite erstellen.

### Live-Aktualisierung

Alle diese Aktionen aktualisieren die Kommentar-Threads anderer Benutzer sofort, ohne dass diese die Seite neu laden müssen. Moderatoraktionen wie das Verbergen eines Kommentars oder das Markieren als Spam entfernen den Kommentar jedoch nicht vom Bildschirm **des Moderators**, damit er die Aktion bei Bedarf schnell rückgängig machen kann. Um anzuzeigen, dass ein Kommentar verborgen ist, wird er im Vergleich zu den anderen Kommentaren hervorgehoben (die Hervorhebungsfarbe hängt vom Grund der Entfernung ab).

Zum Beispiel, gegeben seien die Benutzer `A (commenter)`, `B (Moderator 1)`, und `C (Moderator 2)`.

...und folgendes Szenario:

1. `User B (Moderator 1)` verbirgt einen Kommentar.
2. Für `User A (commenter)` wird dieser Kommentar sofort verborgen.
3. Für `User C (Moderator 2)` wird dieser Kommentar sofort verborgen.
4. Für den Benutzer, der die Änderung vorgenommen hat, `User B (Moderator 1)`, bleibt der Kommentar auf dessen Bildschirm, wird jedoch als entfernt hervorgehoben. Er hat die Möglichkeit, seine Aktion rückgängig zu machen; in diesem Fall sehen die anderen Benutzer die Aktualisierung erneut live.
