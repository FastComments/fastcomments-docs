일부 중재 작업은 댓글 중재 페이지로 이동하지 않고도 댓글 스레드 자체에서 직접 수행할 수 있습니다.

로그인한 상태에서 댓글의 오른쪽 상단에 있는 편집 버튼을 클릭하세요. 중재자는 다음과 같은 옵션을 사용할 수 있습니다:

- **고정** 해당 댓글을 고정
- **삭제** 해당 댓글 삭제
- **삭제** 해당 댓글 삭제 + **사용자 차단** (영구 또는 쉐도우, 자세한 내용은 뒤에서 설명)
- **편집** 해당 댓글 편집
- 해당 댓글을 **승인됨** (표시) 또는 **미승인** (숨김)으로 표시
- 해당 댓글을 **스팸** 또는 **스팸 아님**으로 표시

### 댓글 스레드 닫기

중재자와 관리자는 로그인되어 있을 때 댓글 영역 상단의 점 세 개 메뉴에서 `Close Thread`를 선택하여 댓글 스레드를 잠그거나 닫을 수 있습니다. 언제든 이후에 `Re-Open Thread`를 선택하여 댓글을 다시 열 수 있습니다.

댓글 스레드를 닫으면 새 댓글을 달 수는 없지만 투표는 계속 가능하며 사용자가 원하면 자신의 댓글을 삭제할 수 있습니다.

댓글 스레드를 닫거나 다시 열면 그 스레드를 보고 있는 모든 사용자에게 즉시 반영됩니다.

해당 페이지에 대한 커스터마이제이션 규칙을 만들어 스레드를 읽기 전용으로 표시하면 투표 및 삭제 옵션도 제거할 수 있습니다.

### 실시간 업데이트

이러한 모든 작업은 다른 사용자가 페이지를 다시 로드하지 않아도 댓글 스레드를 즉시 업데이트합니다. 그러나 댓글 숨기기나 스팸 표시와 같은 중재자 작업은 **중재자 자신의** 화면에서 댓글을 제거하지 않으므로 필요할 경우 신속하게 작업을 되돌릴 수 있습니다. 댓글이 숨겨졌음을 표시하기 위해 해당 댓글은 다른 댓글과 비교해 강조 표시되며(제거 사유에 따라 강조 색상이 달라집니다).

예를 들어, 사용자 `A (commenter)`, `B (Moderator 1)`, 그리고 `C (Moderator 2)`가 있다고 가정합니다.

...그리고 다음 시나리오:

1. `User B (Moderator 1)`가 댓글을 숨깁니다.
2. `User A (commenter)`에게는 해당 댓글이 즉시 숨겨집니다.
3. `User C (Moderator 2)`에게도 그 댓글은 즉시 숨겨집니다.
4. 변경을 수행한 사용자, `User B (Moderator 1)`의 화면에서는 그 댓글이 남아 있지만 제거된 것으로 강조 표시됩니다. 이들은 작업을 되돌릴 옵션이 있으며, 되돌리면 다른 사용자들도 실시간으로 다시 업데이트를 보게 됩니다.
