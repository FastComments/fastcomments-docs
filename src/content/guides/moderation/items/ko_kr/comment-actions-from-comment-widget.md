댓글 스레드 자체에서 댓글 관리 페이지로 이동하지 않고도 일부 중재 작업을 직접 수행할 수 있습니다.

로그인한 상태에서 댓글 오른쪽 상단의 편집 버튼을 클릭하면 중재자로서 다음 옵션들을 사용할 수 있습니다:

- 해당 댓글을 **고정**합니다
- 해당 댓글을 **삭제**합니다
- 해당 댓글을 **삭제** + **사용자 차단** (영구 또는 섀도우, 자세한 내용은 아래)
- 해당 댓글을 **편집**합니다
- 해당 댓글을 **잠금** 또는 **잠금 해제**합니다 (자세한 내용은 아래)
- 해당 댓글을 **승인됨**(보이기) 또는 **승인되지 않음**(숨기기)으로 표시합니다
- 해당 댓글을 **스팸** 또는 **스팸 아님**으로 표시합니다

### 댓글 잠그기

개별 댓글을 잠그면 해당 댓글에 대한 새로운 답글 작성이 차단되며, 잠금이 해제될 때까지 그 댓글 자체의 편집 또는 삭제 역시 불가능해집니다. 이는 관리자와 중재자를 포함한 모든 사용자에게 적용됩니다. 잠긴 댓글을 편집하거나 삭제해야 하는 경우에는 먼저 잠금을 해제한 다음 변경을 하고, 필요하면 다시 잠글 수 있습니다.

잠긴 댓글의 오른쪽 상단에는 스레드가 닫혀 있음을 독자가 한눈에 알 수 있도록 자물쇠 아이콘이 표시됩니다. 댓글 위젯과 퍼블릭 API에서 잠긴 댓글에 대해 Edit 및 Delete 메뉴 항목은 숨겨지며 (`PATCH` 및 `DELETE`는 잠긴 댓글에 대해 호출될 경우 `code: 'locked'`를 반환합니다).

잠금을 우회하는 의도적인 예외가 두 가지 있습니다. 그렇지 않으면 고아 데이터가 남을 수 있는 경우입니다: 사용자가 계정을 완전히 삭제할 때(계정 삭제 시 댓글은 잠금 상태와 관계없이 정리됩니다)와 중재자가 "delete all comments from this user" 옵션을 사용하여 사용자를 차단할 때(스윕이 잠금을 해제합니다).

### 댓글 스레드 닫기

중재자와 관리자는 로그인한 상태에서 댓글 영역 상단의 점 3개 메뉴에서 `Close Thread`를 선택하여 댓글 스레드를 잠그거나 닫을 수 있습니다. 나중에 언제든지 `Re-Open Thread`를 선택하여 댓글 작성을 다시 허용할 수 있습니다.

댓글 스레드를 닫으면 새 댓글 작성이 차단되지만, 투표는 계속 가능하며 사용자가 원할 경우 자신의 댓글을 삭제할 수 있습니다.

댓글 스레드를 닫거나 다시 여는 작업은 스레드를 보고 있는 모든 사용자에게 즉시 적용됩니다.

또한 해당 페이지에 대해 별도의 커스터마이제이션 규칙을 만들어 스레드를 읽기 전용으로 표시하면 투표 및 삭제 옵션도 제거할 수 있습니다.

### 실시간 업데이트

이 모든 작업은 다른 사용자가 페이지를 다시 로드할 필요 없이 댓글 스레드를 즉시 업데이트합니다. 그러나 댓글을 숨기거나 스팸으로 표시하는 등의 중재자 작업은 중재자의 화면에서 해당 댓글을 제거하지 않으므로 필요 시 빠르게 작업을 취소할 수 있습니다. 숨겨진 댓글임을 표시하기 위해 다른 댓글과 비교하여 강조 표시되며(제거 사유에 따라 강조 색상이 달라짐) 표시됩니다.

예를 들어 사용자 `A (commenter)`, `B (Moderator 1)`, 및 `C (Moderator 2)`가 있다고 가정하고,

다음과 같은 시나리오가 있을 때:

1. `User B (Moderator 1)`가 댓글을 숨깁니다.
2. `User A (commenter)`에게 그 댓글은 즉시 숨겨집니다.
3. `User C (Moderator 2)`에게 그 댓글은 즉시 숨겨집니다.
4. 변화를 만든 사용자, `User B (Moderator 1)`의 화면에는 댓글이 남아 있지만 제거된 것으로 강조 표시됩니다. 그들은 자신의 행동을 취소할 수 있는 옵션이 있으며, 취소할 경우 다른 사용자들은 다시 실시간으로 업데이트를 보게 됩니다.
