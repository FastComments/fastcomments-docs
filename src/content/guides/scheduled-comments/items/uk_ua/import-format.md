### Приклад формату

CSV запланованих коментарів виглядає так:

|ID |Parent ID|URL ID       |URL|Name         |Avatar|Comment                              | Hours | Minutes | Seconds |
|---|---------|-------------|---|-------------|------|-------------------------------------|-------|---------|---------|
|1  |         |scheduledtest|   |Test Person 1|      |Hello!                               | 0     | 0       | 3       |
|2  |1        |scheduledtest|   |Test Person 2|      |How are you?                         | 0     | 0       | 10      |
|3  |2        |scheduledtest|   |Test Person 3|      |Leave them alone                     | 0     | 1       | 30      |
|4  |         |scheduledtest|   |Test Person 4|      |Does anyone have a cute cat to share?| 1     | 10      | 0       |


### Деталі формату

CSV файл запланованих коментарів підтримує наступні стовпці:

- ID
- Parent ID
- URL ID
- URL
- Name
- Avatar
- Comment
- Hours
- Minutes
- Seconds

Наступні стовпці є **обов'язковими**:

- ID
- URL ID
- Name
- Comment
- Hours
- Minutes
- Seconds

Вам знадобиться стовпець Parent ID, якщо ви хочете підтримувати автоматизовані вкладені відповіді.

### Як працює формат

Формат імпорту працює наступним чином:

1. Ви визначаєте рядок у CSV для кожного коментаря, який хочете опублікувати.
2. Коментар публікується після вказаної затримки (години + хвилини + секунди).
   1. Для імпортів з ручним розкладом затримки відносні до моменту, коли ви натискаєте "play" в інтерфейсі, після завершення імпорту - **не коли імпорт починається**.
   2. Для імпортів з автоматичним розкладом затримка відраховується від моменту завантаження сторінки.
3. Ви повинні визначити ID. Ви можете просто використовувати зростаючі ідентифікатори, такі як 1, 2, 3, 4, 5.
4. Якщо ви хочете використовувати вкладені відповіді, просто встановіть значення стовпця `Parent ID` в `ID` іншого коментаря.
5. Поле `Comment` підтримує той самий синтаксис, який FastComments підтримує у своєму віджеті коментарів для стилізації тексту та додавання зображень.
6. Поле `Avatar`, якщо використовується, має бути публічно доступним зображенням. Воно буде скопійоване та буде обслуговуватися з нашого CDN.
7. Ви можете видалити всі коментарі після відтворення або якщо відтворення зупинено.
8. Видалення відбувається в реальному часі, тому ви можете повторно використовувати той самий запланований імпорт знову і знову.
