### Пример формата

CSV запланированных комментариев выглядит так:

|ID |Parent ID|URL ID       |URL|Name         |Avatar|Comment                              | Hours | Minutes | Seconds |
|---|---------|-------------|---|-------------|------|-------------------------------------|-------|---------|---------|
|1  |         |scheduledtest|   |Test Person 1|      |Hello!                               | 0     | 0       | 3       |
|2  |1        |scheduledtest|   |Test Person 2|      |How are you?                         | 0     | 0       | 10      |
|3  |2        |scheduledtest|   |Test Person 3|      |Leave them alone                     | 0     | 1       | 30      |
|4  |         |scheduledtest|   |Test Person 4|      |Does anyone have a cute cat to share?| 1     | 10      | 0       |


### Детали формата

CSV файл запланированных комментариев поддерживает следующие столбцы:

- ID
- Parent ID
- URL ID
- URL
- Name
- Avatar
- Comment
- Hours
- Minutes
- Seconds

Следующие столбцы являются **обязательными**:

- ID
- URL ID
- Name
- Comment
- Hours
- Minutes
- Seconds

Вам понадобится столбец Parent ID, если вы хотите поддерживать автоматизированные вложенные ответы.

### Как работает формат

Формат импорта работает следующим образом:

1. Вы определяете строку в CSV для каждого комментария, который хотите опубликовать.
2. Комментарий публикуется после указанной задержки (часы + минуты + секунды).
   1. Для импортов с ручным расписанием задержки относительны к моменту, когда вы нажимаете "play" в интерфейсе, после завершения импорта - **не когда импорт начинается**.
   2. Для импортов с автоматическим расписанием задержка отсчитывается от момента загрузки страницы.
3. Вы должны определить ID. Вы можете просто использовать возрастающие идентификаторы, такие как 1, 2, 3, 4, 5.
4. Если вы хотите использовать вложенные ответы, просто установите значение столбца `Parent ID` в `ID` другого комментария.
5. Поле `Comment` поддерживает тот же синтаксис, который FastComments поддерживает в своем виджете комментариев для стилизации текста и добавления изображений.
6. Поле `Avatar`, если используется, должно быть публично доступным изображением. Оно будет скопировано и будет обслуживаться с нашего CDN.
7. Вы можете удалить все комментарии после воспроизведения или если воспроизведение остановлено.
8. Удаление происходит в реальном времени, поэтому вы можете повторно использовать тот же запланированный импорт снова и снова.
