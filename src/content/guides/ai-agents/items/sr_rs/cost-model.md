Трошкови агента су засновани на токенима. Сваки LLM позив враћа број токена, платформа то претвара у центи у USD користећи стопу по токену модела, и ти центи се наплаћују против буџета агента и корисника (tenant).

### Шта се наплаћује

- **Сви LLM позиви**, укључујући позив који не производи ниједну акцију алата ("агент је одлучио да не уради ништа"). Инференција се плаћа чак и када не изађе ниједна акција.
- **Позиви у режиму dry-run**. Dry-run значи "не делуј, али ипак позови LLM" — LLM позив кошта исто. Види [Режим Dry-Run](#dry-run-mode).
- **Поновљени покрети (replay-ови)**. Реплеји су dry-run извршавања против историјских коментара. Они коштају токене. Види [Тест покретања (реплеји)](#test-runs-replays).

### Шта се не наплаћује

- **Окидачи који никада не изазивају LLM позив.** Случајеви одбачени пре LLM (прекорачење буџета, ограничење по брзини, неподударање опсега, неважеће обрачунавање, спречавање петље) не коштају токене. Види [Разлози одбацивања](#drop-reasons).
- **Диспачање алата.** Позивање `pin_comment` или било ког другог алата само по себи не кошта токене — само LLM круг-повратак кошта.
- **`search_memory`.** То је само за читање и не производи свој LLM круг-повратак.

### Трошак по покретању

Једно покретање агента може више пута позвати LLM — резултат сваког позива алата се враћа назад у модел тако да он може позвати други алат или завршити. Дакле, `tokensUsed` за покретање је збир свих LLM круг-повратка у том покретању.

Највећи доприноси трошку по покретању:

- **Дуги [почетни упити](#personality-prompt) и [смернице за заједницу](#community-guidelines)** — они се додају у свако покретање.
- **[Опције контекста](#context-options)** — контекст треда, историја корисника, метаподаци странице. Свака опција додаје токене.
- **Сам текст коментара** — дужи коментари коштају више.
- **Више позива алата у једном покретању** — резултат сваког алата се шаље назад моделу.
- **Читања меморије** — `search_memory` враћа до 25 записа (ограничено на укупно 8000 карактера садржаја). Већина тих бајтова улази у следећи упит.

**Максимални број токена по тригеру** (подразумевано 20.000) ограничaва величину одговора по LLM позиву. Не ограничaва величину уноса.

### Конверзија токена у центи

Платформа примењује једну стопу по пакет-изнајмљивачу (`flexLLMCostCents` по `flexLLMUnit` токена). Цена по токену је на нивоу пакета, не по моделу — оба доступна модела ([GLM 5.1 and GPT-OSS Turbo](#choosing-a-model)) се наплаћују по истој стопи у оквиру датог пакета. [Run Detail View](#run-detail-view) приказује трошак по покретању у вашој валути када се покретање заврши.

### Где се трошак евидентира

Свака извршена рунда евидентира свој сирови број токена и трошак по покретању. Дневни и месечни збирни подаци се агрегирају на [страници аналитике](#analytics-page).

### Како читати трошкове

- **Трошак по покретању**: [Run Detail View](#run-detail-view) -> `Cost` поље.
- **Дневни / месечни агрегат**: [страница аналитике](#analytics-page) -> графикони коришћења буџета и дневног трошка.
- **Трошак по акцији**: такође на Run Detail View, корисно за подешавање када је петља алата агента неуобичајено дуга.
