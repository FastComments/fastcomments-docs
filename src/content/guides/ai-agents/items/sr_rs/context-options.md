Секција **Контекст** на формулару за уређивање контролише колико информација агент добија приликом сваког покретања. Више контекста доноси боље одлуке али повећава трошак у токенима по покретању, па желите само оно што агент заиста треба.

### Шта је увек укључено

Чак и кад су сва поља за потврду искључена, агентова контекст порука садржи:

- Тип догађаја који је покренуо тригер (нпр. `COMMENT_ADD`, `COMMENT_FLAG_THRESHOLD`).
- URL странице и ID URL-а (к када је познато).
- Коментар који је покренуо покретање, ако постоји - ID, ID аутора корисника, приказно име аутора, текст коментара, број гласова, број пријава, ознаке за спам/одобрен/прегледано, ID родитеља. Емаил аутора се **никада** не шаље провајдеру LLM (минимизација PII).
- Претходни текст коментара за `COMMENT_EDIT` тригере (да агент може упоредити пре/после).
- Смер гласа за `COMMENT_VOTE_THRESHOLD` тригере.
- Кориснички ID који је покренуо тригер и **badge ID** (за тригере значки модератора).
- Каталог значки вашег тенанта (**badge catalog**) (име, приказни назив, опис) када је агенту дозвољено да додељује значке, тако да агент може да одабере одговарајућу без да ви наводите значке у упутству.

 Sav непоуздани текст - тела коментара, имена аутора, наслови страница, сам документ са смерницама - је **ограђен** у контекст поруци маркерима попут `<<<COMMENT_TEXT>>> ... <<<END>>>`. Системски промпт платформе упућује модел да никад не следи инструкције унутар тих ограда. Ово је платформа за заштиту од убризгавања промпта; не морате то понављати у свом промпту.

### Три чекбокса

#### Укључи родитељски коментар и претходне одговоре у истој теми

Додаје:
- **Родитељски коментар** - ID, аутор, текст.
- **Одговори истог родитеља** - претходни одговори на истог родитеља у истој нит.

Корисно за: било који агент који одговара на коментар у контексту (поздравни агенти, резимирачи нити, модератори који читају одговоре у конверзацијама).

Трошак: мали до средњи. Ограничено бројем одговора на истог родитеља у датој нити.

#### Укључи фактор поверења коментатора, старост налога, историју банова и недавне коментаре

Додаје блок **AUTHOR_HISTORY**:

- Старост налога у данима од пријаве.
- Фактор поверења (0-100) - FastComments оцена која резимира колико је корисник поуздан на овом сајту. Погледајте страницу [Spam Detection](/guide-moderation.html#spam-detection) у водичу за модерацију.
- Број претходних банова.
- Укупно коментара на овом сајту.
- Број дупликата садржаја - ако је корисник недавно постављао идентичан текст (сигнал против спама).
- Сигнал преко налога са исте IP адресе - број коментара са исте IP адресе под другим налозима (сигнал о алтернативним налозима). Хеш IP адресе се никад не шаље LLM-у.
- Недавни коментари - до 5 најновијих коментара корисника, сваки скраћен на 300 карактера, ограђени као непоуздан текст.

Корисно за: било ког агента за модерацију. Без овога модел често бира да забрани нове налоге и дугорочне кориснике добре вере који имају сличан став.

Трошак: средњи. Недавни коментари додају највише токена.

#### Укључи наслов странице, поднаслов, опис и meta тагове

Додаје блок **PAGE_CONTEXT** - наслов, поднаслов, опис и било који meta тагови које је FastComments ухватио за страницу.

Корисно за: поздравне агенте и резимираче нити, где познавање о чему је страница значајно побољшава квалитет излаза.

Трошак: мали.

### Правила заједнице

Четврто поље, **Community guidelines**, је слободно-текстуални блок политике који се укључује у контекстну поруку са улогом корисника при сваком покретању, ограђен као непоуздан текст на исти начин као тела коментара и други садржаји које доставља корисник. Агент га чита као политички текст, али платформа га не третира као системску инструкцију. Погледајте [Правила заједнице](#community-guidelines) за шта ставити у њега.

### Додавање контекста селективно

Ови чекбокси се примењују по агенту, а не глобално. Чест образац:

- Поздравни агент: PAGE_CONTEXT укључен, thread context искључен, user history искључен.
- Модератор: thread context искључен, user history укључен, page context искључен.
- Резимирач нити: thread context укључен, page context укључен, user history искључен.

Настојте да обезбедите минимални контекст који агенту треба да буде тачан у позивима које заправо прави - додатни контекст кошта токене при сваком покретању, чак и када га агент не користи.
