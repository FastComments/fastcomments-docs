Τα webhooks του agent είναι HTTP callbacks που πυροδοτούνται από την πλατφόρμα όταν μια εκτέλεση agent ολοκληρώνεται ή όταν μια έγκριση αλλάζει κατάσταση. Χρησιμοποιήστε τα για να προωθείτε τη δραστηριότητα των agent στα δικά σας συστήματα — πίνακες ελέγχου moderation, αρχεία ελέγχου, κανάλια Slack, εργαλεία κλιμάκωσης.

Ρυθμίζεται στην καρτέλα **Webhooks** στη [σελίδα AI Agents](https://fastcomments.com/auth/my-account/ai-agents).

### Τι παραδίδεται

Τέσσερις τύποι συμβάντων:

- **`trigger.succeeded`** - μια εκτέλεση agent ολοκληρώθηκε με επιτυχία.
- **`trigger.failed`** - μια εκτέλεση agent απέτυχε με σφάλμα.
- **`approval.requested`** - μια ενέργεια τοποθετήθηκε σε ουρά για ανθρώπινη έγκριση.
- **`approval.decided`** - μια έγκριση εγκρίθηκε, απορρίφθηκε ή η εκτέλεσή της απέτυχε.

Δείτε τα [Συμβάντα Webhook](#webhook-events) για το πότε πυροδοτείται κάθε γεγονός, και τα [Φορτία Webhook](#webhook-payloads) για το σχήμα του καθενός.

### Τι δεν παραδίδεται

- **Webhooks ανά ενέργεια εργαλείου.** Μια εκτέλεση που καλεί `pin_comment` δεν πυροδοτεί ξεχωριστό webhook για το pin — η ενέργεια περιλαμβάνεται στο payload `trigger.succeeded` της εκτέλεσης. Αν θέλετε παράδοση ανά ενέργεια, αναλύστε τον πίνακα `actions` στο payload του trigger.
- **Απορριφθέντα triggers.** Ένα trigger που δεν αποστέλλεται (λόγω υπέρβασης προϋπολογισμού, λάθος scope) δεν πυροδοτεί webhook. Οι απορρίψεις είναι ορατές μόνο στη [Σελίδα Analytics](#analytics-page).
- **Triggers που παράγονται από αναπαραγωγές.** Οι δοκιμαστικές εκτελέσεις δεν πυροδοτούν webhooks. Δείτε [Δοκιμαστικές Εκτελέσεις (Replays)](#test-runs-replays).

### Διαμόρφωση

Για κάθε webhook που ρυθμίζετε:

- **URL** - το endpoint HTTPS στο οποίο θα γίνει POST.
- **Domain** - το domain σχολίου στο οποίο εφαρμόζεται αυτό το webhook (ο tenant σας μπορεί να φιλοξενεί πολλαπλά domains). `*` ταιριάζει με όλα τα domains; `*.example.com` είναι ένα glob; ένα ακριβές domain ταιριάζει μόνο με αυτό.
- **Events** - σε ποιους από τους τέσσερις τύπους συμβάντων θα εγγραφεί.
- **Agents** - κενό για "όλοι οι agents", ή μια λίστα συγκεκριμένων agent IDs για φιλτράρισμα.
- **Method** - POST ή PUT (προεπιλογή POST).
- **No-retry status codes** - κωδικοί HTTP απόκρισης που πρέπει να θεωρούνται τελικές αποτυχίες και να μην επαναπροσπαθούνται (π.χ., 410 Gone, 422 Unprocessable). Δείτε [Webhook Retries](#webhook-retries).

Πολλαπλά webhooks μπορούν να εγγραφούν στο ίδιο συμβάν — κάθε ένα τοποθετείται σε δική του ουρά και παραδίδεται στο δικό του URL.

### Αντιστοίχιση ανά domain

Ένα δεδομένο συμβάν παραδίδεται σε **κάθε webhook του οποίου το πεδίο `domain` ταιριάζει με το domain του σχολίου**. Η αντιστοίχιση χρησιμοποιεί ένα απλό glob:

- Ακριβές: `example.com` ταιριάζει μόνο με `example.com`.
- Wildcard star: `*` ταιριάζει με κάθε domain.
- Subdomain glob: `*.example.com` ταιριάζει με `blog.example.com`, `forum.example.com`, αλλά όχι με το ίδιο το `example.com`.

Το domain σε ένα payload είναι το πρώτο μη-null αποτέλεσμα από: το `domain` του σχολίου, το URL που αναλύεται σε σχέση με τη διαμόρφωση domain του tenant σας, ή την αναζήτηση `Page` μέσω `urlId`.

### Φιλτράρισμα ανά agent

Το πεδίο **Agents** επιτρέπει σε ένα webhook να εγγραφεί μόνο σε συγκεκριμένους agents. Κενό σημαίνει "όλοι οι agents". Όταν δεν είναι κενό, το webhook πυροδοτείται μόνο για agents που περιλαμβάνονται στη λίστα.

Αυτό είναι χρήσιμο όταν έχετε ένα webhook για συμβάντα moderation και ένα άλλο για συμβάντα engagement, και τα δύο να κατευθύνονται σε διαφορετικά downstream συστήματα.

### Δοκιμαστική αποστολή

Το UI διαμόρφωσης webhook έχει ένα κουμπί **Test send** που στέλνει ένα ψεύτικο payload στο URL ώστε να επαληθεύσετε τη συνδεσιμότητα, την υπογραφή και τον κωδικό απόκρισης του endpoint σας χωρίς να περιμένετε ένα πραγματικό συμβάν.

### Αρχεία καταγραφής παράδοσης

Κάθε παράδοση (και κάθε επαναπροσπάθεια) καταγράφεται στη σελίδα [Webhook Delivery Logs](#webhook-logs) ώστε να δείτε τι συνέβη στο δίκτυο.

### Υπογραφή

Κάθε webhook υπογράφεται με HMAC-SHA256 χρησιμοποιώντας το API secret του tenant σας. Δείτε [Webhook Signing](#webhook-signing).

### Προϋποθέσεις

Τα agent webhooks απαιτούν έγκυρη χρέωση στον tenant. Χρησιμοποιούν την ίδια υποδομή υπογραφής/secret με τα υπάρχοντα webhooks σχολίων σας — αν έχετε ήδη ενσωματώσει τα webhooks σχολίων (δείτε τον [Οδηγό Webhooks](/guide-webhooks.html)), η ενσωμάτωση των agent webhooks έχει την ίδια μορφή με νέους τύπους συμβάντων.
