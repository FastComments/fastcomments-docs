Αυτή είναι η πεντάλεπτη διαδρομή από «έχουμε AI Agents» έως «ένας agent ανταποκρίνεται σε ζωντανή κίνηση, ελεγχόμενος από εγκρίσεις». Αν θέλετε την εκτενή έκδοση, κάθε βήμα συνδέει στη σελίδα που το καλύπτει αναλυτικά.

### 1. Open the AI Agents page

Μεταβείτε στη σελίδα [AI Agents](https://fastcomments.com/auth/my-account/ai-agents) στον λογαριασμό σας. Την πρώτη φορά που θα προσγειωθείτε εδώ θα δείτε είτε:

- Μια κενή κατάσταση με κουμπιά **Browse templates** και **Start from scratch** (έχετε διαθέσιμους agents για δημιουργία), είτε
- Μια σελίδα upsell εάν το πλάνο σας δεν περιλαμβάνει agents - δείτε [Plans and Eligibility](#plans-and-eligibility).

### 2. Pick a starter template

Κάντε κλικ στο **Browse templates**. Επιλέξτε ένα από τα:

- [Moderator](#template-moderator) - ελέγχει επισημασμένα ή νέα σχόλια, προειδοποιεί τους πρωτοεμφανιζόμενους, κλιμακώνει προς αποκλεισμό μόνο μετά από προειδοποίηση.
- [Welcome Greeter](#template-welcome-greeter) - απαντά σε σχολιαστές που σχολιάζουν για πρώτη φορά.
- [Top Comment Pinner](#template-top-comment-pinner) - καρφιτσώνει ουσιαστικά σχόλια μόλις υπερβούν ένα όριο ψήφων.
- [Thread Summarizer](#template-thread-summarizer) - δημοσιεύει μια ουδέτερη περίληψη σε μεγάλες συζητήσεις.

Κάθε πρότυπο ανοίγει μια προκαταχωρημένη φόρμα επεξεργασίας με την επιλογή **Status: Dry Run** ήδη επιλεγμένη.

### 3. Review and save

Στη φόρμα επεξεργασίας, κάντε τουλάχιστον τα εξής:

- **Internal name.** Ένας σύντομος αναγνωριστικός τίτλος που χρησιμοποιείται στους πίνακες διαχείρισης.
- **Display name.** Αυτό που εμφανίζεται δημόσια όταν ο agent δημοσιεύει ένα σχόλιο.
- **Initial prompt.** Επεξεργαστείτε το prompt του προτύπου ώστε να ταιριάζει με τη δική σας φωνή και τους ειδικούς σας κανόνες.
- **Approvals.** Επιλέξτε τις ενέργειες που θα απαιτούν ανθρώπινη έγκριση πριν εκτελεστούν. Συστήνουμε τουλάχιστον `ban_user` για οποιονδήποτε agent τύπου moderation. Δείτε [Approval Workflow](#approval-workflow).

Κάντε κλικ στο **Save agent**.

### 4. Watch it in dry-run

Ο agent είναι πλέον ενεργός σε **Δοκιμαστική λειτουργία**. Θα λαμβάνει τα triggers του, θα καλεί το μοντέλο και θα καταγράφει ενέργειες στη σελίδα [Run History](#run-history) - με την ετικέτα **Δοκιμαστική λειτουργία** σε κάθε γραμμή - αλλά δεν εκτελεί πραγματικές ενέργειες. Επισκεφτείτε μερικές λεπτομέρειες εκτέλεσης (δείτε [Run Detail View](#run-detail-view)) και κοιτάξτε:

- Τις ενέργειες που επέλεξε ο agent.
- Την αιτιολόγηση και την εμπιστοσύνη σε κάθε ενέργεια.
- Το πλήρες transcript του LLM.

Εάν ο agent παίρνει αποφάσεις με τις οποίες διαφωνείτε, επεξεργαστείτε το αρχικό prompt ή ενεργοποιήστε περισσότερες approvals.

### 5. Run a test against past comments

Από τη σελίδα λίστας agents, κάντε κλικ στο **Test run** στην γραμμή του agent. Η φόρμα έχει ένα μόνο αριθμητικό πεδίο **Days** (1 έως 90). Το μέγεθος του δείγματος και το ανώτατο όριο στα σχόλια που αξιολογούνται εμφανίζονται πληροφοριακά - υπολογίζονται server-side, δεν τα ορίζει ο χρήστης. Η αναπαραγωγή τρέχει εναντίον ιστορικών σχολίων χωρίς να παίρνει πραγματικές ενέργειες και αναφέρει τι ο agent **θα** είχε κάνει σε σχέση με το τι συνέβη πραγματικά (εάν το σχόλιο εγκρίθηκε αργότερα, χαρακτηρίστηκε ως spam, διαγράφηκε κ.λπ.). Δείτε [Test Runs (Replays)](#test-runs-replays).

### 6. Flip to Enabled

Όταν είστε ικανοποιημένοι με τα αποτελέσματα της δοκιμαστικής λειτουργίας και της αναπαραγωγής, επεξεργαστείτε τον agent και αλλάξτε την **Status** σε **Enabled**. Από εδώ και πέρα, εκτελούνται πραγματικές ενέργειες. Η σελίδα Run History πλέον εμφανίζει ζωντανές εκτελέσεις χωρίς την ετικέτα δοκιμαστικής λειτουργίας, και κάθε ενέργεια που επισημάνατε για έγκριση εμφανίζεται στα [approvals inbox](#approval-workflow).
