Μια **έγκριση** είναι μια κλήση εργαλείου σε ουρά που απαιτεί από έναν άνθρωπο να την εγκρίνει ή να την απορρίψει πριν η πλατφόρμα την εκτελέσει.

### Διαμόρφωση εγκρίσεων

Στην φόρμα επεξεργασίας του agent, η ενότητα **Approvals** απαριθμεί κάθε εργαλείο που επιτρέπεται στον agent να καλεί (η λίστα επιτρεπόμενων) και σας επιτρέπει να επισημάνετε αυτά που πρέπει να εξεταστούν από άνθρωπο. Τα εργαλεία που δεν έχουν επιλεγεί εκτελούνται αμέσως. Τα επιλεγμένα εργαλεία μπαίνουν σε ουρά.

Τα μη επιτρεπόμενα εργαλεία *αρνούνται απευθείας*, δεν μπαίνουν σε ουρά — οι εγκρίσεις εφαρμόζονται μόνο σε εργαλεία που διαφορετικά επιτρέπονται.

### Τι συμβαίνει όταν πυροδοτείται μια ελεγχόμενη ενέργεια

1. Ο agent επιλέγει μια κλήση εργαλείου (π.χ. `ban_user`) με επιχειρήματα, αιτιολόγηση και βαθμό εμπιστοσύνης.
2. Αντί να εκτελεστεί, η πλατφόρμα τοποθετεί σε ουρά μια έγκριση στην κατάσταση `PENDING` με το όνομα του εργαλείου, τα επιχειρήματα, την αιτιολόγηση, την εμπιστοσύνη και ένα στιγμιότυπο πλαισίου που περιγράφει το trigger που το προέκυψε.
3. Αποστέλλονται ειδοποιήσεις στους κριτές (βλ. [Approval Notifications](#approval-notifications)).
4. Η εκτέλεση του agent ολοκληρώνεται και καταγράφεται — η ενέργεια εμφανίζεται με **Pending approval** στην [Run Detail View](#run-detail-view).

### Ανασκόπηση εγκρίσεων

Το inbox εγκρίσεων στο [my-account/ai-agent-approvals](https://fastcomments.com/auth/my-account/ai-agent-approvals) εμφανίζει εκκρεμείς, εγκεκριμένες, απορριφθείσες και εγκρίσεις με αποτυχία εκτέλεσης. Για κάθε μία:

- **Tool name and arguments** - ακριβώς τι θέλει να κάνει ο agent.
- **Agent reasoning** - η αιτιολόγηση που παρείχε ο agent.
- **Confidence** - η αυτο-βαθμολογημένη εμπιστοσύνη του agent, από 0.0 έως 1.0.
- **Context snapshot** - το σχόλιο, η σελίδα και ο χρήστης που στοχεύει η ενέργεια.

Δύο κουμπιά: **Έγκριση** και **Απόρριψη**. Η Έγκριση εκτελεί στην πραγματικότητα το εργαλείο· η Απόρριψη απορρίπτει την ενέργεια.

### Καταστάσεις κατάστασης έγκρισης

Μια έγκριση διασχίζει αυτές τις καταστάσεις:

| State | Meaning |
|---|---|
| `PENDING` | Αναμονή για ανθρώπινη απόφαση. |
| `APPROVED` | Ενας άνθρωπος ενέκρινε και η ενέργεια εκτελέστηκε. |
| `REJECTED` | Ενας άνθρωπος απέρριψε την ενέργεια. Η ενέργεια δεν εκτελέστηκε. |
| `EXECUTION_FAILED` | Ενας άνθρωπος ενέκρινε αλλά η εκτέλεση απέτυχε (π.χ. το στοχευόμενο σχόλιο είχε ήδη διαγραφεί). |
| `EXECUTING` | Μεταβατική: ένας άνθρωπος κλικαρε το Έγκριση και η ενέργεια τρέχει. Χρησιμοποιείται για να σειριοποιήσει ταυτόχρονες κλικ-εγκρίσεις ώστε ένα εργαλείο με πραγματικές παρενέργειες να μην τρέξει δύο φορές. |

Η κατάσταση `EXECUTING` έχει σημασία όταν δύο κριτές πατούν Έγκριση ταυτόχρονα — ένας κερδίζει, ο άλλος βλέπει ότι η έγκριση έχει ήδη προχωρήσει.

### Τι καθαρίζεται

Οι εκκρεμείς εγκρίσεις παραμένουν εκκρεμείς μέχρι να αποφασιστεί κάτι. Λήγουν αυτόματα **90 ημέρες** μετά τη δημιουργία. Οι εγκρίσεις σε οποιαδήποτε άλλη κατάσταση επίσης διαγράφονται μετά από 90 ημέρες για λόγους διαχείρισης αποθήκευσης.

Τα πεδία της έγκρισης "decided by" / "decided at" / "executed at" / "execution result" συμπληρώνονται καθώς η έγκριση προχωράει ανάμεσα στις καταστάσεις — όλα ορατά στην προβολή λεπτομερειών του inbox.

### Ενσωμάτωση Webhook

Δύο γεγονότα webhook πυροδοτούνται καθώς οι εγκρίσεις προχωρούν:

- **`approval.requested`** - στην εισαγωγή `PENDING`.
- **`approval.decided`** - στην μετάβαση σε `APPROVED`, `REJECTED`, ή `EXECUTION_FAILED`.

Και τα δύο υπογράφονται όπως κάθε άλλο webhook. Βλ. [Webhook Events](#webhook-events) και [Webhook Payloads](#webhook-payloads).

### Ενίσχυση ασφάλειας: known-tool gate

Οι εγκρίσεις αρνούνται να τοποθετήσουν σε ουρά οποιοδήποτε όνομα εργαλείου δεν αναγνωρίζεται ως εργαλείο του agent. Αυτό είναι ένας έλεγχος «άμυνα σε βάθος»: ακόμη κι αν μελλοντικά κάποιο μονοπάτι κώδικα περάσει ένα όνομα εργαλείου παραγόμενο από LLM στη ροή εγκρίσεων, αυτή η συμβολοσειρά δεν μπορεί ποτέ να καταλήξει ως εγγεγραμμένη έγκριση. Το σύνολο των γνωστών ονομάτων εργαλείων είναι το ίδιο σύνολο που αναφέρεται στην [Tools Reference](#tools-overview).

### Συνήθη μοτίβα ελέγχου

- **Brand-new moderation agent** - φρακάρετε `ban_user`, `mark_comment_spam`, `mark_comment_approved`, `send_email`. Παρακολουθήστε το inbox για μερικές εβδομάδες, και μετά αφαιρέστε το gating από εργαλεία με χαμηλά ποσοστά σφάλματος.
- **Long-term moderation agent** - κρατήστε `ban_user` και οποιεσδήποτε μη αναστρέψιμες ενέργειες (`deleteAllUsersComments`, `banIP`) πάντα φραγμένες.
- **EU region** - το `ban_user` είναι ενεργοποιημένο λόγω του Άρθρου 17 ανεξαρτήτως των επιλογών σας. Βλ. [EU DSA Article 17 Compliance](#eu-dsa-compliance).

### Τι οι εγκρίσεις **δεν** κάνουν

- Δεν καθυστερούν τις υπόλοιπες κλήσεις εργαλείων του agent. Η εκτέλεση του agent μπορεί να κληθεί να καλέσει πολλά εργαλεία, και μόνο τα ελεγχόμενα μπαίνουν σε ουρά — τα υπόλοιπα εκτελούνται κανονικά.
- Δεν αναιρούν την εκτέλεση του agent αν ο άνθρωπος απορρίψει. Το μη-ελεγχόμενο μέρος της εκτέλεσης έχει ήδη ολοκληρωθεί.
