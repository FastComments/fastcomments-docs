Инструменты агента — это действия, которые он может выполнять. Форма редактирования агента содержит раздел **Allowed tool calls**, где вы отмечаете инструменты, которые этому агенту разрешено использовать, и раздел **Approvals**, где вы отмечаете действия, требующие одобрения человека до их выполнения.

Существует три уровня для любого инструмента:

- **Disallowed** — агент не видит и не может использовать его.
- **Allowed, no approval** — агент использует его напрямую. Записывается в историю выполнения.
- **Allowed, with approval** — вызов агента ставится в очередь на проверку человеком и выполняется только после одобрения.

Запрещённые инструменты бесшумны: агент не может запрашивать их, и платформа полностью отклоняет такие запросы. Инструменты, требующие одобрения, всегда проходят через [approvals inbox](#approval-workflow).

### Аудиторский след для каждого действия

Каждое действие агента фиксируется с кратким обоснованием (1–2 предложения, объясняющие причину) и оценкой уверенности (0.0–1.0). Оба поля отображаются в [Run Detail View](#run-detail-view) и на каждом [approval](#approval-workflow). Поиск в памяти — это единственное исключение только для чтения: он не записывается как действие и всегда доступен независимо от списка разрешённых инструментов.

### Справочник по инструментам

#### Публикация комментариев

Позволяет агенту опубликовать комментарий от своего имени. Комментарий отображается публично под именем агента. Используется приветствующими и суммаризирующими агентами. Обратимо — любой модератор может удалить плохой комментарий. Ограничьте одобрением, если для вашего сообщества требуется, чтобы каждое публичное сообщение проверялось человеком.

#### Редактирование комментария

Позволяет агенту переписать текст комментария в рамках области действия. Оригинальный текст сохраняется в аудите комментария. Используйте только в узких случаях — удаление PII, которое пользователь случайно выдал, или исправление собственного ответа агента. Не предназначено для переписывания мнений или смягчения тона. См. [Edit comment](#tool-edit-comment) для полной страницы.

#### Голосование за комментарии

Позволяет агенту ставить плюс или минус к комментарию. Голос учитывается в общем счёте комментария так же, как и любой другой голос. Большинство сообществ предпочитают, чтобы боты не голосовали; не включено ни в один стартовый шаблон. Если вы разрешаете это, голосование обратимо.

#### Закрепить / открепить комментарий

Позволяет агенту закрепить комментарий вверху страницы или открепить уже закреплённый. Платформа не обеспечивает правило «один закреплённый комментарий на тред», поэтому агенту, закрепляющему комментарии, следует сначала откреплять предыдущий закреплённый комментарий. Чтобы узнать, что уже закреплено на той же странице, агент может вызвать инструмент только для чтения `get_pinned_comments` (см. ниже). Используется в шаблоне Top Comment Pinner.

#### Заблокировать / разблокировать комментарий

Позволяет агенту запретить дальнейшие ответы под комментарием или восстановить возможность отвечать. Заблокированный комментарий остаётся видимым. Полезно для охлаждения накалённых тем, совмещается с отложенным снятием блокировки. Чтобы узнать, что в настоящее время заблокировано на той же странице, агент может вызвать инструмент только для чтения `get_locked_comments` (см. ниже).

#### Отметить / снять отметку спама

Позволяет агенту пометить комментарий как спам (скрывая его от читателей и кормя классификатор спама) или снять эту метку. Основной инструмент для любого модерационного агента. Обратимо.

#### Одобрить / снять одобрение комментария

Позволяет агенту показать удерживаемый комментарий читателям или скрыть уже видимый. Наиболее полезно для арендаторов, которые удерживают новые комментарии на проверку модераторами.

#### Отметить комментарий как просмотренный

Инструмент состояния очереди: помечает комментарий как «модератор (или агент) посмотрел это». Не меняет видимость. Низкий риск; редко ставится под ограничение.

#### Присвоить значок

Позволяет агенту выдать пользователю значок, настроенный для вашего арендатора. Обратимо модератором. Когда этот инструмент включён, агент видит значки вашего арендатора и самостоятельно выбирает подходящий, поэтому вам не нужно вставлять идентификаторы значков в правила сообщества или первоначальный промпт. Чтобы направить, какой значок выдавать за какое поведение, обращайтесь к значкам по их **Display Label** в промпте.

#### Отправить электронное письмо

Позволяет агенту отправить простое текстовое письмо автору комментария в области действия триггера. Агент никогда не видит адрес электронной почты получателя — он выбирает комментарий, а платформа доставляет письмо на тот адрес, который оставил комментатор при публикации. Адрес отправителя — брендированный отправитель вашего арендатора (с DKIM), когда домен комментария соответствует настроенному домену, иначе — отправитель по умолчанию платформы. Используйте экономно — электронная почта самый затратный по последствиям инструмент, и плохие письма трудно отменить.

#### Сохранение / поиск памяти агента

Два взаимосвязанных инструмента, которые читают и записывают общий пул заметок о пользователе, для которого сработал триггер. Память общая для всех агентов в вашем арендаторе, поэтому заметки триажного агента информируют решения модераторского агента. Поиск — только для чтения и всегда доступен; сохранение редко ставится под ограничение. См. [Agent Memory System](#agent-memory-system) для полного дизайна.

#### Get pinned comments / Get locked comments

Два инструмента только для чтения, которые перечисляют закреплённые (или заблокированные) комментарии на той же странице (`urlId`), на которой сработал триггер. Они не принимают аргументов — страница читается из контекста триггера, поэтому агент не может переключиться на другие страницы. Используйте их, когда агенту нужно действовать по отношению к комментарию, который уже закреплён или заблокирован — обычно это первый вызов перед `unpin_comment` или `unlock_comment`, или перед закреплением нового комментария, чтобы сначала можно было открепить существующий.

Каждый инструмент ставится под ограничение отдельно в **Allowed tool calls** (администратор отмечает `List pinned comments on the current page` или `List locked comments on the current page`). Они не могут быть скрыты за одобрением — инструменты только для чтения не имеют побочных эффектов, требующих одобрения. Их вызов не записывается как действие в истории выполнения; в истории появляется только последующий вызов `unpin_comment` / `unlock_comment` / `pin_comment` (если он есть). Список ограничен 20 последними совпадениями на вызов.

Важно понимать: когда один из этих инструментов возвращает commentId, этот commentId добавляется в область действия агента для данного выполнения, поэтому последующий вызов `unpin_comment` / `unlock_comment` проходит проверку безопасности по цели инструмента платформы. Без предварительного вызова инструмента обнаружения агент не может действовать над комментариями, которые не входят в непосредственную область триггера. Таким образом, агент, работающий в стиле «открепить», обычно имеет включенные оба инструмента (например, `get_pinned_comments` и `unpin_comment`).

#### Предупредить пользователя

Отправляет приватное личное сообщение с предупреждением пользователю по поводу конкретного комментария и атомарно записывает предупреждение в память агента. Политика эскалации платформы строится вокруг этого инструмента — сначала предупреждать, блокировать только в случае повторного нарушения. См. [Warn user](#tool-warn-user) для полной страницы.

#### Заблокировать пользователя

Самый тяжёлый по последствиям инструмент, который может вызвать агент. Блокирует пользователя на фиксированный срок, опционально в режиме shadow ban, опционально также блокируя IP, опционально также удаляя все комментарии пользователя. Две разрушительные опции (IP, удаление всех комментариев) ставятся под дополнительные согласия на форме редактирования. В регионе ЕС все блокировки требуют одобрения человека (см. [EU DSA Article 17 Compliance](#eu-dsa-compliance)). См. [Ban user](#tool-ban-user) для полной страницы.

### Дополнительные опции инструмента бана

Инструмент бана предоставляет две разрушительные опции — delete-all-comments и ban-by-IP — которые полностью скрыты от модели, пока вы не включите их через раздел **Ban options** на форме редактирования. Даже если модель сгенерирует параметр по ошибке, платформа отклонит значения, которые вы не включили. См. [Ban user](#tool-ban-user).
