Инструменты агента — это действия, которые он может выполнять. В форме редактирования агента есть раздел **Разрешённые вызовы инструментов (Allowed tool calls)**, где вы отмечаете инструменты, которые этому агенту разрешено использовать, и раздел **Утверждения (Approvals)**, где вы отмечаете действия, требующие одобрения человека перед вступлением в силу.

There are three levels for any tool:

- **Disallowed** - the agent cannot see or use it.
- **Allowed, no approval** - the agent uses it directly. Recorded in run history.
- **Allowed, with approval** - the agent's call is queued for human review and only runs when a human approves.

Запрещённые инструменты молчат: агент не может запрашивать их, и платформа отклоняет такие запросы сразу. Инструменты, требующие одобрения, всегда проходят через [папку входящих утверждений](#approval-workflow).

### Журнал аудита для каждого действия

Каждое действие агента записывается с кратким обоснованием (1–2 предложения, объясняющие причину) и оценкой уверенности (0.0–1.0). Оба элемента отображаются в [Просмотре деталей выполнения](#run-detail-view) и в каждом [одобрении](#approval-workflow). Поиск по памяти — единственное исключение только для чтения: он не фиксируется как действие и всегда доступен независимо от списка разрешённых инструментов.

### Справочник инструментов

#### Публикация комментариев

Позволяет агенту опубликовать комментарий от своего имени. Комментарий отображается публично под отображаемым именем агента. Используется приветственными (greeter) и суммаризирующими (summarizer) агентами. Обратимо — любой модератор может удалить неподобающий комментарий. Обычно разрешается без одобрения; включайте проверку, если вашему сообществу нужно, чтобы каждое публичное сообщение проходило проверку человеком.

#### Голосование по комментариям

Позволяет агенту голосовать за или против комментария. Голос учитывается в общем счёте комментария как любой другой. Большинство сообществ предпочитает, чтобы боты не голосовали; в стартовых шаблонах это не включено. Если вы всё же разрешаете это, голосование можно отменить.

#### Закрепить / открепить комментарий

Позволяет агенту закрепить комментарий вверху страницы или открепить уже закреплённый. Платформа не ограничивает правило «один закреплённый комментарий на тред», поэтому агенту, который закрепляет комментарии, следует сначала откреплять предыдущий закреплённый комментарий. Используется в шаблоне Top Comment Pinner. Обратимо; обычно разрешается без одобрения.

#### Заблокировать / разблокировать комментарий

Позволяет агенту запретить дальнейшие ответы под комментарием или восстановить возможность отвечать. Заблокированный комментарий остаётся видимым. Полезно для «остывания» накалённых тредов, в сочетании с отложенной разблокировкой. Обратимо, но видно вашему сообществу; для критичных сообществ рассмотрите включение проверки человеком.

#### Пометить / снять пометку спам

Позволяет агенту пометить комментарий как спам (скрывая его от читателей и передавая в спам-классификатор) или снять эту пометку. Базовый инструмент для любого модерационного агента. Обратимо. Настоятельно рекомендуется требовать одобрения в первые недели, пока вы не заработаете доверие к агенту.

#### Одобрить / снять одобрение с комментария

Позволяет агенту показать удерживаемый комментарий читателям или скрыть уже видимый. Наиболее полезно для тенантов, которые удерживают новые комментарии для проверки модераторами. Снятие одобрения с видимого комментария — действие с высокой степенью риска; рассмотрите требование одобрения человека.

#### Отметить комментарий как рассмотренный

Инструмент состояния очереди: помечает комментарий как «модератор (или агент) просмотрел это». Не меняет видимость. Низкий риск; редко требует одобрения.

#### Выдать значок

Позволяет агенту присвоить пользователю значок из конфигурации значков вашего тенанта. Могут отменить модераторы. Редко требует одобрения. Агент должен знать ID значка, поэтому включите соответствующие идентификаторы в ваши [правила сообщества](#community-guidelines) или [начальную подсказку](#personality-prompt).

#### Отправка электронной почты

Позволяет агенту отправить plain-text письмо с адреса `noreply@fastcomments.com` на выбранный им адрес. Используйте экономно — электронная почта имеет наибольшие издержки, и исправить неудачные письма сложно. Настоятельно рекомендуется требовать одобрения и направлять письма для одобрения тому, кто владеет почтовым ящиком, на который агент в итоге будет отправлять сообщения.

#### Сохранение / поиск памяти агента

Два взаимосвязанных инструмента, которые читают и записывают общий пул заметок о пользователе, для которого сработал триггер. Память общая для всех агентов вашего тенанта, поэтому заметки триаж-агента информируют решения модераторного агента. Поиск — только для чтения и всегда доступен; сохранение редко требует одобрения. См. [Система памяти агента](#agent-memory-system) для полного описания.

#### Предупредить пользователя

Отправляет приватное DM-предупреждение пользователю по конкретному комментарию и атомарно записывает предупреждение в память агента. Политика эскалации платформы построена вокруг этого инструмента — сначала предупреждение, бан только при повторном нарушении. Менее часто требует одобрения, чем `ban_user`, но рассмотрите включение проверки в первые недели работы агента. См. [Предупредить пользователя](#tool-warn-user) для полной страницы.

#### Забанить пользователя

Самый серьёзный инструмент, который может вызвать агент. Блокирует пользователя на фиксированный срок, опционально как теневой бан, опционально также банит IP, опционально также удаляет все комментарии пользователя. Две деструктивные опции (IP, delete-all) доступны только при дополнительных согласиях в форме редактирования. Даже если модель выдумает параметр, платформа отклонит значения, которые вы не включили. См. [Забанить пользователя](#tool-ban-user).

### Подопции инструмента бана

Инструмент Ban предоставляет две деструктивные опции — delete-all-comments и ban-by-IP — которые полностью скрыты от модели, пока вы не включите их через секцию **Ban options** в форме редактирования. Даже если модель выдумает параметр, платформа отклонит значения, которые вы не активировали. См. [Забанить пользователя](#tool-ban-user).
