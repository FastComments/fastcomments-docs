Раздел **Context** в форме редактирования контролирует, сколько информации агент получает при каждом запуске. Больше контекста даёт лучшие решения, но увеличивает стоимость в токенах за запуск, поэтому вы должны включать только то, что агенту действительно нужно.

### Что всегда включено

Даже если все флажки сняты, сообщение контекста агента включает:

- Тип события-триггера (например, `COMMENT_ADD`, `COMMENT_FLAG_THRESHOLD`).
- URL страницы и ID URL (когда известны).
- Комментарий, который вызвал запуск, если он есть — ID, ID автора, отображаемое имя автора, текст комментария, счёт голосов, количество флагов, флаги spam/approved/reviewed, parent ID. Электронная почта автора **никогда** не отправляется поставщику LLM (минимизация передачи PII).
- Предыдущий текст комментария для триггеров `COMMENT_EDIT` (чтобы агент мог сравнить «до/после»).
- Направление голоса для триггеров `COMMENT_VOTE_THRESHOLD`.
- ID пользователя, вызвавшего событие, и ID бейджа (для триггеров по бейджам модератора).
- Каталог бейджей вашего тенанта (name, display label, description), когда агенту разрешено присуждать бейджи, чтобы агент мог выбрать подходящий, не требуя от вас перечислять бейджи в подсказке.

Весь ненадёжный текст — тела комментариев, имена авторов, заголовки страниц, сам документ с правилами — **берётся в ограждённые блоки** в сообщении контекста маркерами вроде `<<<COMMENT_TEXT>>> ... <<<END>>>`. Системная подсказка платформы инструктирует модель никогда не выполнять инструкции внутри этих ограждений. Это защита от инъекций в подсказку; вам не нужно повторять её в вашей подсказке.

### Три флажка

#### Включить родительский комментарий и предыдущие ответы в той же ветке

Добавляет:
- **Родительский комментарий** — ID, автор, текст.
- **Сиблинг-реплаи** — предыдущие ответы тому же родителю в той же ветке.

Полезно для: любых агентов, которые отвечают на комментарий в контексте (приветствующие, суммаризаторы веток, модераторы, читающие ответы в беседе).

Стоимость: от небольшой до средней. Ограничена тем, сколько сиблингов существует в данной ветке.

#### Включить фактор доверия комментатора, возраст аккаунта, историю банов и недавние комментарии

Добавляет блок **AUTHOR_HISTORY**:

- **Возраст аккаунта в днях** с момента регистрации.
- **Trust factor (0-100)** — оценка FastComments, суммирующая, насколько доверяют пользователю на этом сайте. См. страницу [Обнаружение спама](/guide-moderation.html#spam-detection) в руководстве по модерации.
- **Количество предыдущих банов.**
- **Общее количество комментариев на этом сайте.**
- **Количество дублированного контента** — если пользователь недавно публиковал идентичный текст (сигнал антиспама).
- **Сигнал о кросс-аккаунтах с того же IP** — количество комментариев с того же IP под другими аккаунтами (сигнал о мультиаккаунтах). Сам хеш IP никогда не отправляется в LLM.
- **Недавние комментарии** — до 5 последних комментариев пользователя, каждый усечён до 300 символов, взятый в ограждённые блоки как ненадёжный текст.

Полезно для: любого модерационного агента. Без этих данных модель будет банить и новые аккаунты, и давних добросовестных пользователей с одинаковой манерой поведения.

Стоимость: средняя. Наибольшее количество токенов добавляют недавние комментарии.

#### Включить заголовок страницы, подзаголовок, описание и мета-теги

Добавляет блок **PAGE_CONTEXT** — title, subtitle, description и любые мета-теги, которые FastComments захватил для страницы.

Полезно для: приветствующих и суммаризаторов веток, где знание того, о чём страница, существенно улучшает качество вывода.

Стоимость: небольшая.

### Руководство сообщества

Четвёртое поле, **Community guidelines**, — это блок политики свободного текста, включаемый в сообщение контекста с ролью пользователя при каждом запуске, взятый в ограждённые блоки как ненадёжный текст так же, как тела комментариев и другой контент, предоставленный пользователями. Агент читает его как текст политики, но платформа не рассматривает его как системную инструкцию. См. раздел [Руководство сообщества](#community-guidelines) о том, что туда поместить.

### Выборочное добавление контекста

Эти флажки применяются на уровне агента, а не глобально. Распространённый шаблон:

- Приветствующий: page context **включён**, thread context **выключен**, user history **выключен**.
- Модератор: thread context **выключен**, user history **включён**, page context **выключен**.
- Суммаризатор ветки: thread context **включён**, page context **включён**, user history **выключен**.

Стремитесь предоставлять минимальный контекст, необходимый агенту, чтобы корректно выполнить именно те вызовы, которые он делает — лишний контекст стоит токенов при каждом запуске, даже когда агент им не пользуется.
