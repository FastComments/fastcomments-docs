**ID Προτύπου:** `welcome_greeter`

Ο Χαιρετιστής Καλωσορίσματος απαντά θερμά σε σχολιαστές που σχολιάζουν για πρώτη φορά. Είναι το πρότυπο με τον χαμηλότερο κίνδυνο (χωρίς καταστροφικά εργαλεία) και ένας καλός πρώτος agent για να το βγάλετε live.

### Ενεργοποιητές

- **Νέος χρήστης δημοσιεύει το πρώτο του σχόλιο σε αυτόν τον ιστότοπο** (`NEW_USER_FIRST_COMMENT`).

Αυτό το γεγονός ενεργοποιείται ακριβώς μία φορά ανά χρήστη, οπότε ο agent δεν μπορεί να κάνει loop. Δείτε [Ενεργοποιητής: Πρώτο σχόλιο νέου χρήστη](#trigger-new-user-first-comment).

### Επιτρεπόμενα εργαλεία

- [`write_comment`](#tools-overview)

Αυτό είναι το μόνο εργαλείο - ο agent κυριολεκτικά δεν μπορεί να μετριάσει, να ψηφίσει, να αποκλείσει ή να στείλει DM.

### Συνιστώμενες προσθήκες πριν την ενεργοποίηση

- **Ορίστε το Όνομα Εμφάνισης** σε κάτι φιλικό - "Community Bot", το μασκότ του ιστότοπού σας, ή το όνομα της μάρκας σας. Το όνομα εμφάνισης είναι αυτό που βλέπουν οι αναγνώστες συνδεδεμένο με την απάντηση καλωσορίσματος.
- **Τσεκάρετε "Include page title, subtitle, description, and meta tags"** στο [Context Options](#context-options). Οι απαντήσεις του χαιρετιστή βελτιώνονται αισθητά όταν μπορεί να αναφερθεί σε αυτό που πραγματικά αφορά η σελίδα.
- **Λάβετε υπόψη περιορισμούς γλώσσας** αν λειτουργείτε σε πολλαπλές γλώσσες. Μια απάντηση καλωσορίσματος στη λάθος γλώσσα είναι πιο ενοχλητική από την απουσία απάντησης. Δείτε [Πεδίο: Φίλτρα URL και Γλώσσας](#scope-url-locale).

### Γιατί δεν χρειάζονται εγκρίσεις

Ο agent γράφει μόνο νέα σχόλια και μόνο σε ενεργοποίηση μίας φοράς. Στην χειρότερη περίπτωση: ένας αμήχανος χαιρετισμός. Δεν υπάρχει καταστροφική ενέργεια που να απαιτεί έγκριση. Οι περισσότεροι χειριστές το τρέχουν χωρίς καμία έγκριση, μόλις η δοκιμαστική εκτέλεση (dry-run) φαίνεται καθαρή.
