Το τμήμα **Context** στη φόρμα επεξεργασίας ελέγχει πόσες πληροφορίες λαμβάνει ο agent σε κάθε εκτέλεση. Περισσότερο πλαίσιο παράγει καλύτερες αποφάσεις αλλά αυξάνει το κόστος σε tokens ανά εκτέλεση, οπότε θέλετε μόνο ό,τι ο agent πραγματικά χρειάζεται.

### Τι περιλαμβάνεται πάντα

Ακόμα και με όλα τα πλαίσια ελέγχου αποεπιλεγμένα, το μήνυμα πλαισίου του agent περιλαμβάνει:

- Τον **τύπο συμβάντος ενεργοποίησης** (π.χ. `COMMENT_ADD`, `COMMENT_FLAG_THRESHOLD`).
- Το **URL της σελίδας και το URL ID** (όταν είναι γνωστά).
- Το **σχόλιο** που προκάλεσε την εκτέλεση, αν υπάρχει - ID, user ID του συγγραφέα, εμφανιζόμενο όνομα συγγραφέα, κείμενο σχολίου, μετρήσεις ψήφων, πλήθος σηματισμών, σημαίες spam/εγκεκριμένο/ελέγχθηκε, parent ID. Το email του συγγραφέα **ποτέ** δεν αποστέλλεται στον πάροχο LLM (μείωση PII).
- Το **προηγούμενο κείμενο σχολίου** για ενεργοποιήσεις `COMMENT_EDIT` (ώστε ο agent να μπορεί να συγκρίνει πριν/μετά).
- Η **κατεύθυνση ψήφου** για τις ενεργοποιήσεις `COMMENT_VOTE_THRESHOLD`.
- Το **user ID** που προκάλεσε την ενέργεια και το **badge ID** (για ενεργοποιήσεις badge moderator).
- Ο **κατάλογος badges του tenant** σας (όνομα, εμφανιζόμενη ετικέτα, περιγραφή) όταν ο agent επιτρέπεται να απονείμει badges, ώστε ο agent να μπορεί να επιλέξει ένα κατάλληλο χωρίς να χρειάζεται να τα περιγράψετε στο prompt.

Όλο το μη αξιόπιστο κείμενο - σώματα σχολίων, ονόματα συγγραφέων, τίτλοι σελίδων, το ίδιο το έγγραφο οδηγιών - είναι **περιφραγμένο** στο μήνυμα πλαισίου με δείκτες όπως `<<<COMMENT_TEXT>>> ... <<<END>>>`. Το system prompt της πλατφόρμας δίνει εντολή στο μοντέλο να μην ακολουθεί ποτέ οδηγίες μέσα σε αυτά τα περιφραγμένα τμήματα. Αυτή είναι η άμυνα της πλατφόρμας ενάντια σε prompt-injection· δεν χρειάζεται να το επαναλάβετε στο prompt σας.

### Τα τρία πλαίσια ελέγχου

#### Include parent comment and prior replies in the same thread

Προσθέτει:
- Το **γονικό σχόλιο** - ID, συγγραφέας, κείμενο.
- **Αδελφικές απαντήσεις** - οι προηγούμενες απαντήσεις στον ίδιο γονέα στο ίδιο νήμα.

Χρήσιμο για: οποιονδήποτε agent απαντά σε σχόλιο εντός συμφραζομένων (welcome greeters, συνοψιστές νήματος, moderators που διαβάζουν απαντήσεις σε συζητήσεις).

Κόστος: μικρό έως μεσαίο. Περιορίζεται από τον αριθμό αδελφών που υπάρχουν σε ένα νήμα.

#### Include commenter's trust factor, account age, ban history, and recent comments

Προσθέτει το μπλοκ **AUTHOR_HISTORY**:

- **Ηλικία λογαριασμού σε ημέρες** από την εγγραφή.
- **Trust factor (0-100)** - το σκορ FastComments που συνοψίζει πόσο αξιόπιστος είναι ο χρήστης σε αυτόν τον ιστότοπο. Δείτε τη σελίδα [Spam Detection](/guide-moderation.html#spam-detection) στον οδηγό moderation.
- **Προηγούμενος αριθμός αποκλεισμών.**
- **Συνολικά σχόλια σε αυτόν τον ιστότοπο.**
- **Αριθμός διπλού περιεχομένου** - αν ο χρήστης έχει δημοσιεύσει ίδιο κείμενο πρόσφατα (σήμα κατά spam).
- **Σήμα cross-account με ίδια IP** - πλήθος σχολίων από την ίδια IP υπό άλλους λογαριασμούς (σήμα alt-account). Το hash της IP δεν αποστέλλεται ποτέ στο LLM.
- **Πρόσφατα σχόλια** - έως 5 από τα πιο πρόσφατα σχόλια του χρήστη, το καθένα περικομμένο στα 300 χαρακτήρες, περιφραγμένο ως μη αξιόπιστο κείμενο.

Χρήσιμο για: οποιονδήποτε moderation agent. Χωρίς αυτό, το μοντέλο αποκλείει νέους λογαριασμούς και παλαιούς χρήστες με καλή πίστη που έχουν παρόμοια συμπεριφορά.

Κόστος: μεσαίο. Τα πρόσφατα σχόλια προσθέτουν τα περισσότερα tokens.

#### Include page title, subtitle, description, and meta tags

Προσθέτει το μπλοκ **PAGE_CONTEXT** - τίτλος, υπότιτλος, περιγραφή και τυχόν meta tags που έχει καταγράψει το FastComments για τη σελίδα.

Χρήσιμο για: welcome greeters και συνοψιστές νήματος, όπου η γνώση για το θέμα της σελίδας βελτιώνει σημαντικά την ποιότητα του αποτελέσματος.

Κόστος: μικρό.

### Οδηγίες κοινότητας

Το τέταρτο πεδίο, **Community guidelines**, είναι ένα πεδίο ελεύθερου κειμένου πολιτικής που περιλαμβάνεται στο μήνυμα πλαισίου ρόλου χρήστη σε κάθε εκτέλεση, περιφραγμένο ως μη αξιόπιστο κείμενο με τον ίδιο τρόπο που περιφράσσονται τα σώματα σχολίων και άλλο κείμενο που προέρχεται από χρήστες. Ο agent το διαβάζει ως πολιτικό κείμενο αλλά η πλατφόρμα δεν το αντιμετωπίζει ως system instruction. Δείτε τις [Community Guidelines](#community-guidelines) για τι να βάλετε σε αυτό.

### Επιλεκτική προσθήκη πλαισίου

Αυτά τα πλαίσια ελέγχου εφαρμόζονται ανά agent, όχι παγκόσμια. Ένα κοινό μοτίβο:

- Welcome greeter: page context **on**, thread context **off**, user history **off**.
- Moderator: thread context **off**, user history **on**, page context **off**.
- Thread summarizer: thread context **on**, page context **on**, user history **off**.

Χρησιμοποιήστε το ελάχιστο πλαίσιο που χρειάζεται ο agent για να είναι σωστός στις κλήσεις που κάνει πραγματικά - το επιπλέον πλαίσιο κοστίζει tokens σε κάθε εκτέλεση, ακόμη και όταν ο agent δεν το χρησιμοποιεί.
