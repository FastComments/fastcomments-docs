Ένα agent's **tools** είναι οι ενέργειες που μπορεί να κάνει. Η φόρμα επεξεργασίας agent έχει μια ενότητα **Allowed tool calls** όπου τσεκάρετε τα εργαλεία που επιτρέπεται να χρησιμοποιεί αυτός ο agent, και μια ενότητα **Approvals** όπου τσεκάρετε τις ενέργειες που θα απαιτούν ανθρώπινη έγκριση πριν να τεθούν σε ισχύ.

Υπάρχουν τρία επίπεδα για κάθε εργαλείο:

- **Disallowed** - ο agent δεν μπορεί να το δει ή να το χρησιμοποιήσει.
- **Allowed, no approval** - ο agent το χρησιμοποιεί άμεσα. Καταγράφεται στο ιστορικό εκτέλεσης.
- **Allowed, with approval** - το αίτημα του agent τίθεται σε ουρά για ανθρώπινη ανασκόπηση και εκτελείται μόνο όταν ένας άνθρωπος το εγκρίνει.

Τα μη επιτρεπόμενα εργαλεία είναι αθόρυβα: ο agent δεν μπορεί να τα ζητήσει και η πλατφόρμα τα απορρίπτει εντελώς. Τα εργαλεία που απαιτούν έγκριση πάντοτε περνάνε από τα [approvals inbox](#approval-workflow).

### Audit trail on every action

Κάθε ενέργεια που εκτελεί ο agent καταγράφεται με μια σύντομη αιτιολόγηση (1-2 προτάσεις που εξηγούν το γιατί) και ένα σκορ εμπιστοσύνης (0.0-1.0). Και τα δύο εμφανίζονται στην [Run Detail View](#run-detail-view) και σε κάθε [approval](#approval-workflow). Η αναζήτηση στη μνήμη είναι η μόνη εξαίρεση μόνο-για-ανάγνωση: δεν καταγράφεται ως ενέργεια και είναι πάντα διαθέσιμη ανεξαρτήτως της λίστας επιτρεπόμενων.

### Tool reference

#### Posting comments

Επιτρέπει στον agent να δημοσιεύσει ένα σχόλιο ως ο ίδιος. Το σχόλιο εμφανίζεται δημόσια κάτω από το εμφανιζόμενο όνομα του agent. Χρησιμοποιείται από agents που χαιρετούν και που συνοψίζουν. Αναιρετικό - οποιοσδήποτε moderator μπορεί να αφαιρέσει ένα κακό σχόλιο. Συνήθως επιτρέπεται χωρίς έγκριση· περιορίστε το αν η κοινότητά σας χρειάζεται κάθε δημόσιο μήνυμα να ελέγχεται από άνθρωπο.

#### Voting on comments

Επιτρέπει στον agent να ψηφίσει θετικά ή αρνητικά ένα σχόλιο. Η ψήφος συνυπολογίζεται στο συνολικό πλήθος ψήφων του σχολίου όπως κάθε άλλη ψήφος. Οι περισσότερες κοινότητες προτιμούν να μην έχουν bots που ψηφίζουν· δεν είναι ενεργοποιημένο σε κανένα starter template. Αν το επιτρέψετε, η ψήφος είναι αναιρέσιμη (reversible).

#### Pin / unpin a comment

Επιτρέπει στον agent να καρφώσει ένα σχόλιο στην κορυφή της σελίδας ή να αφαιρέσει καρφίτσωμα από ένα σχόλιο που έχει ήδη καρφωθεί. Η πλατφόρμα δεν επιβάλλει κανόνα ενός-καρφιτσώματος-ανά-νήμα, οπότε ένας agent που καρφώνει πρέπει να έχει εντολή να αφαιρεί πρώτα το προηγούμενο καρφωμένο σχόλιο. Χρησιμοποιείται από το Top Comment Pinner template. Αναιρέσιμο· συνήθως επιτρέπεται χωρίς έγκριση.

#### Lock / unlock a comment

Επιτρέπει στον agent να αποτρέψει περαιτέρω απαντήσεις κάτω από ένα σχόλιο, ή να επαναφέρει τις απαντήσεις. Το κλειδωμένο σχόλιο παραμένει ορατό. Χρήσιμο για αποκλιμάκωση σε θερμά νήματα, συχνά σε συνδυασμό με deferred unlock. Αναιρέσιμο αλλά ορατό στην κοινότητά σας· σκεφτείτε να το βάλετε πίσω από έγκριση σε κοινότητες με υψηλό ρίσκο.

#### Mark / unmark spam

Επιτρέπει στον agent να σημειώσει ένα σχόλιο ως spam (κρύβοντάς το από τους αναγνώστες και τροφοδοτώντας τον ταξινομητή spam) ή να αφαιρέσει αυτήν τη σήμανση. Το βασικό εργαλείο για οποιονδήποτε moderation agent. Αναιρέσιμο. Σοβαρά σκεφτείτε να το βάλετε πίσω από έγκριση τις πρώτες εβδομάδες μέχρι να κερδίσετε εμπιστοσύνη στον agent.

#### Approve / un-approve a comment

Επιτρέπει στον agent να δείξει ένα κρατημένο σχόλιο στους αναγνώστες, ή να κρύψει ένα ήδη ορατό. Πολύ χρήσιμο σε tenants που κρατούν νέα σχόλια για ανασκόπηση από moderators. Υψηλό ρίσκο όταν απο-εγκρίνετε ένα ορατό σχόλιο - σκεφτείτε να το βάλετε πίσω από έγκριση.

#### Mark a comment reviewed

Ένα εργαλείο κατάστασης ουράς: επισημαίνει ένα σχόλιο ως «ένας moderator (ή agent) το έχει δει». Δεν αλλάζει την ορατότητα. Χαμηλό ρίσκο· σπάνια τοποθετείται πίσω από έγκριση.

#### Award a badge

Επιτρέπει στον agent να δώσει σε έναν χρήστη ένα badge από τη ρύθμιση badges του tenant σας. Αναιρετό από έναν moderator. Σπάνια κλειδωμένο. Ο agent πρέπει να γνωρίζει το ID του badge, οπότε συμπεριλάβετε τα σχετικά IDs στις [community guidelines](#community-guidelines) ή στο [initial prompt](#personality-prompt).

#### Send email

Επιτρέπει στον agent να στείλει ένα plain-text email από `noreply@fastcomments.com` σε μια διεύθυνση που επιλέγει. Χρησιμοποιήστε με φειδώ - το email είναι το εργαλείο με το υψηλότερο κόστος επανόρθωσης και τα κακά email είναι δύσκολα να αναστραφούν. Σοβαρά σκεφτείτε να το βάλετε πίσω από έγκριση, και δρομολογήστε τις εγκρίσεις email σε όποιον διαχειρίζεται το inbox που ο agent θα καταλήξει να στέλνει.

#### Save / search agent memory

Δύο ζευγαρωμένα εργαλεία που διαβάζουν και γράφουν σε μια κοινή δεξαμενή σημειώσεων για τον χρήστη για τον οποίο ενεργοποιήθηκε ο trigger. Η μνήμη μοιράζεται σε όλους τους agents στο tenant σας, οπότε οι σημειώσεις ενός triage agent επηρεάζουν τις αποφάσεις ενός moderator agent. Η αναζήτηση είναι μόνο-για-ανάγνωση και πάντα διαθέσιμη· η αποθήκευση σπάνια κλειδώνεται. Δείτε το [Agent Memory System](#agent-memory-system) για το πλήρες σχέδιο.

#### Warn a user

Στέλνει ένα ιδιωτικό DM προειδοποίησης σε έναν χρήστη σχετικά με ένα συγκεκριμένο σχόλιο, και καταγράφει ατομικά την προειδοποίηση στη μνήμη του agent. Η πολιτική κλιμάκωσης της πλατφόρμας χτίζεται γύρω από αυτό το εργαλείο - πρώτα προειδοποιείτε, απαγορεύετε μόνο αν ο χρήστης ξαναπαραβεί. Λιγότερο συχνά κλειδωμένο από το `ban_user`, αλλά σκεφτείτε να το βάλετε πίσω από έγκριση τις πρώτες εβδομάδες λειτουργίας ενός agent. Δείτε το [Warn user](#tool-warn-user) για την πλήρη σελίδα.

#### Ban a user

Το πιο σημαντικό εργαλείο που μπορεί να καλέσει ένας agent. Απαγορεύει έναν χρήστη για μια σταθερή διάρκεια, προαιρετικά ως shadow ban, προαιρετικά επίσης απαγορεύοντας την IP, προαιρετικά επίσης διαγράφοντας όλα τα σχόλια του χρήστη. Οι δύο καταστροφικές επιλογές (IP, delete-all) απαιτούν επιπλέον opt-ins στη φόρμα επεξεργασίας. Στην περιοχή της ΕΕ, όλες οι απαγορεύσεις απαιτούν ανθρώπινη έγκριση (βλέπε [EU DSA Article 17 Compliance](#eu-dsa-compliance)). Σοβαρά σκεφτείτε να το βάλετε πίσω από έγκριση παντού. Δείτε το [Ban user](#tool-ban-user) για την πλήρη σελίδα.

### Ban-tool sub-options

Το εργαλείο Ban αποκαλύπτει δύο καταστροφικές επιλογές - delete-all-comments και ban-by-IP - που είναι κρυφές από το μοντέλο εντελώς μέχρι να τις ενεργοποιήσετε μέσω της ενότητας **Ban options** στη φόρμα επεξεργασίας. Ακόμα και αν το μοντέλο επινοήσει την παράμετρο, η πλατφόρμα απορρίπτει τιμές που δεν έχετε ενεργοποιήσει. Δείτε το [Ban user](#tool-ban-user).
