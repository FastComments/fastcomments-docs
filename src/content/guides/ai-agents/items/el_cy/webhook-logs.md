Κάθε webhook πράκτορα έχει το δικό του αρχείο καταγραφής παραδόσεων. Προσπελάζεται από τη [webhook list page](https://fastcomments.com/auth/my-account/ai-agents/webhooks) μέσω του κουμπιού **Logs** σε κάθε γραμμή webhook.

### Τι υπάρχει στη σελίδα

Ένας σελιδοποιημένος πίνακας με μία γραμμή ανά προσπάθεια παράδοσης:

| Column | Meaning |
|---|---|
| When | Πότε συνέβη η προσπάθεια. |
| Event | Ο τύπος του γεγονότος (`trigger.succeeded`, `approval.requested`, κ.λπ.). |
| Status | Η κατάσταση παράδοσης. |
| StatusCode | HTTP κωδικός κατάστασης που επέστρεψε το endpoint σας, όταν είναι διαθέσιμος. |
| Description | Μια σύντομη περιγραφή του αποτελέσματος. Για αποτυχημένες παραδόσεις όπου δεν λήφθηκε HTTP απάντηση, το υποκείμενο μήνυμα σφάλματος καταγράφεται ως `{error: <message>}`. |

Η σελίδα υποστηρίζει μόνο σελιδοποίηση - δεν υπάρχουν φίλτρα κατάστασης, τύπου γεγονότος ή χρονικού εύρους.

### Τι μπορείτε να κάνετε από τα αρχεία καταγραφής

- **Αποφασίστε εάν ένας κωδικός κατάστασης πρέπει να προστεθεί στους Κωδικούς κατάστασης χωρίς επανάληψη.** Αν δείτε το endpoint σας να επιστρέφει συνεχώς τον ίδιο `4xx`, αυτό είναι ένδειξη ότι θέλετε να τον προσθέσετε στους **Κωδικούς κατάστασης χωρίς επανάληψη** ώστε η πλατφόρμα να σταματήσει να επιχειρεί ξανά.

### Πληροφορίες αποτυχίας

Όταν μια παράδοση αποτυγχάνει χωρίς HTTP απάντηση (αποτυχία DNS, σύνδεση απορρίφθηκε, χρονικό όριο, σφάλμα TLS, κ.λπ.), το ακατέργαστο μήνυμα σφάλματος καταγράφεται ως `{error: <message>}`. Η πλατφόρμα δεν τα κατηγοριοποιεί σε ονοματισμένους κάδους όπως `TIMEOUT` ή `DNS_ERROR` - διαβάστε απευθείας το μήνυμα σφάλματος για να δείτε τι συνέβη.

Για HTTP απαντήσεις, η στήλη StatusCode δείχνει τον κωδικό που επέστρεψε το endpoint σας. Συνηθισμένες περιπτώσεις:

- **All attempts: `401` or `403`** - το endpoint σας απορρίπτει την υπογραφή. Ελέγξτε ότι υπολογίζετε το HMAC πάνω σε `${timestamp}.${body}` και χρησιμοποιείτε το σωστό secret. Βλ. [Webhook Signing](#webhook-signing).
- **All attempts: `422`** - το endpoint σας θεωρεί το payload άκυρο. Είτε διορθώστε το endpoint σας, είτε προσθέστε το `422` στους **Κωδικούς κατάστασης χωρίς επανάληψη** αν η απόρριψη είναι αναμενόμενη για ορισμένα γεγονότα.

### Πλαίσιο ανά παράδοση

Κάθε εγγραφή καταγραφής περιέχει:

- `webhookId` - ποια ρύθμιση webhook παρήγαγε αυτή την παράδοση.
- `agentId` - σε ποιον πράκτορα αφορά η παράδοση.
- `triggerId` ή `approvalId` - το υποκείμενο αρχείο.
- `domain` - το ταιριασμένο domain.

Μπορείτε να τα χρησιμοποιήσετε για να συσχετίσετε μια αποτυχημένη παράδοση με την εκτέλεση στην οποία αφορά στο [Ιστορικό Εκτελέσεων](#run-history).

### Διατήρηση

Οι εγγραφές `AgentSyncLog` έχουν οριζόντια TTL 1 έτους βάσει του `createdAt` ανεξάρτητα από το αποτέλεσμα - οι επιτυχείς και οι αποτυχημένες παραδόσεις διατηρούνται για το ίδιο χρονικό διάστημα. Μετά τη λήξη διατήρησης, η εγγραφή καταγραφής εξαφανίζεται.

Αν χρειάζεστε μακροχρόνιο έλεγχο (audit), το βιώσιμο μοτίβο είναι: το **ίδιο το endpoint** να διατηρεί τις παραδόσεις που λαμβάνει. Αυτό αποσυνδέει το αρχείο ελέγχου σας από την πολιτική διατήρησης της πλατφόρμας.

### Δοκιμαστική αποστολή

Το κουμπί **Δοκιμαστική αποστολή** στη φόρμα ρύθμισης webhook γράφει μια ψεύτικη παράδοση στον ίδιο πίνακα καταγραφής ώστε να μπορείτε να επαληθεύσετε τη συνδεσιμότητα end-to-end πριν βασιστείτε σε πραγματικά γεγονότα. Οι δοκιμαστικές παραδόσεις επισημαίνονται σαφώς στο αρχείο καταγραφής ώστε να μην μολύνουν τα στατιστικά αποτυχιών παραγωγής.
