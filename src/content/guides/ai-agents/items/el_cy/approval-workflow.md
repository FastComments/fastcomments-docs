Ένα **approval** είναι μια κλήση εργαλείου σε ουρά που απαιτεί ανθρώπινη έγκριση ή απόρριψη πριν η πλατφόρμα την εκτελέσει.

### Διαμόρφωση εγκρίσεων

Στη φόρμα επεξεργασίας του agent, η ενότητα **Approvals** απαριθμεί κάθε εργαλείο που επιτρέπεται να καλεί ο agent (η λίστα επιτρεπόμενων) και σας επιτρέπει να σημειώσετε αυτά που πρέπει να ελεγχθούν από άνθρωπο. Τα εργαλεία που δεν είναι σημειωμένα εκτελούνται αμέσως. Τα σημειωμένα εργαλεία τοποθετούνται σε ουρά.

Τα μη επιτρεπόμενα εργαλεία *απορρίπτονται απευθείας*, δεν μπαίνουν σε ουρά — οι εγκρίσεις ισχύουν μόνο για εργαλεία που διαφορετικά θα επιτρέπονταν.

### Τι συμβαίνει όταν ενεργοποιηθεί μια ελεγχόμενη ενέργεια

1. Ο agent επιλέγει μια κλήση εργαλείου (π.χ. `ban_user`) με ορίσματα, τεκμηρίωση (justification) και βαθμό εμπιστοσύνης (confidence).
2. Αντί να εκτελέσει, η πλατφόρμα τοποθετεί μια έγκριση σε κατάσταση `PENDING` με το όνομα του εργαλείου, τα ορίσματα, την τεκμηρίωση, το confidence και ένα στιγμιότυπο συμφραζομένων που περιγράφει τον trigger που το προκάλεσε.
3. Αποστέλλονται ειδοποιήσεις στους αξιολογητές (βλ. [Approval Notifications](#approval-notifications)).
4. Η εκτέλεση του agent ολοκληρώνεται και καταγράφεται — η ενέργεια εμφανίζεται ως **Pending approval** στην [Προβολή Λεπτομερειών Εκτέλεσης](#run-detail-view).

### Ανασκόπηση εγκρίσεων

Το inbox εγκρίσεων στο [my-account/ai-agent-approvals](https://fastcomments.com/auth/my-account/ai-agent-approvals) απαριθμεί τις εγκρίσεις σε κατάσταση pending, approved, rejected και execution-failed. Για κάθε μία εμφανίζεται:

- **Tool name and arguments** - ακριβώς αυτό που θέλει να κάνει ο agent.
- **Agent reasoning** - η τεκμηρίωση που παρείχε ο agent.
- **Confidence** - ο αυτοαξιολογημένος βαθμός εμπιστοσύνης του agent, από 0.0 έως 1.0.
- **Context snapshot** - το σχόλιο, η σελίδα και ο χρήστης στους οποίους στοχεύει η ενέργεια.

Δύο κουμπιά: **Έγκριση** και **Απόρριψη**. Η Έγκριση εκτελεί πραγματικά το εργαλείο· η Απόρριψη απορρίπτει το αίτημα.

### Καταστάσεις κατάστασης έγκρισης

Μια έγκριση περνά από αυτές τις καταστάσεις:

| State | Meaning |
|---|---|
| `PENDING` | Αναμονή για ανθρώπινη απόφαση. |
| `APPROVED` | Ένας άνθρωπος ενέκρινε και η ενέργεια εκτελέστηκε. |
| `REJECTED` | Ένας άνθρωπος απέρριψε. Η ενέργεια δεν εκτελέστηκε. |
| `EXECUTION_FAILED` | Ένας άνθρωπος ενέκρινε αλλά η ενέργεια απέτυχε (π.χ. το στοχευόμενο σχόλιο είχε ήδη διαγραφεί). |
| `EXECUTING` | Προσωρινή κατάσταση: ένας άνθρωπος πάτησε Έγκριση και η ενέργεια βρίσκεται σε εκτέλεση. Χρησιμοποιείται για να σειριοποιεί τα ταυτόχρονα κλικ Έγκρισης ώστε ένα εργαλείο με πραγματικές παρενέργειες να μην τρέχει δύο φορές. |

Η κατάσταση `EXECUTING` έχει σημασία όταν δύο αξιολογητές πατούν ταυτόχρονα Έγκριση — ένας «κερδίζει», ο άλλος βλέπει ότι η έγκριση έχει ήδη προχωρήσει.

### Τι διαγράφεται

Οι εκκρεμείς εγκρίσεις παραμένουν σε αναμονή μέχρι να αποφασιστεί. Λήγουν αυτόματα **90 ημέρες** μετά τη δημιουργία. Οι εγκρίσεις σε οποιαδήποτε άλλη κατάσταση επίσης διαγράφονται μετά από 90 ημέρες για λόγους καθαριότητας αποθηκευτικού χώρου.

Τα πεδία "decided by" / "decided at" / "executed at" / "execution result" της έγκρισης συμπληρώνονται καθώς η έγκριση μετακινείται μεταξύ καταστάσεων — όλα ορατά στην προβολή λεπτομερειών του inbox.

### Ενσωμάτωση Webhook

Δύο γεγονότα webhook πυροδοτούνται καθώς οι εγκρίσεις προχωρούν:

- **`approval.requested`** - κατά την εισαγωγή σε `PENDING`.
- **`approval.decided`** - κατά τη μετάβαση σε `APPROVED`, `REJECTED`, ή `EXECUTION_FAILED`.

Και τα δύο υπογράφονται όπως κάθε άλλο webhook. Βλ. [Webhook Events](#webhook-events) και [Webhook Payloads](#webhook-payloads).

### Ενίσχυση ασφάλειας: known-tool gate

Οι εγκρίσεις αρνούνται να τοποθετήσουν σε ουρά οποιοδήποτε όνομα εργαλείου δεν είναι αναγνωρισμένο εργαλείο του agent. Πρόκειται για έναν έλεγχο άμυνας σε βάθος: ακόμη και αν ένα μελλοντικό μονοπάτι κώδικα περάσει ένα όνομα εργαλείου που προέρχεται από LLM στη ροή έγκρισης, αυτή η συμβολοσειρά δεν μπορεί ποτέ να καταλήξει ως εγγεγραμμένη έγκριση. Το σύνολο των γνωστών ονομάτων εργαλείων είναι το ίδιο σύνολο που απαριθμείται στο [Tools Reference](#tools-overview).

### Συνήθη μοτίβα gating

- **Νεοσύστατος agent moderation** - κλειδώστε τα `ban_user`, `mark_comment_spam`, `mark_comment_approved`, `send_email`. Παρακολουθήστε το inbox για μερικές εβδομάδες και μετά αφαιρέστε τον έλεγχο από εργαλεία με χαμηλό ποσοστό σφαλμάτων.
- **Μακροχρόνιος agent moderation** - κρατήστε τα `ban_user` και οποιεσδήποτε μη αναστρέψιμες ενέργειες (`deleteAllUsersComments`, `banIP`) πάντα υπό έλεγχο.
- **Περιοχή ΕΕ** - το `ban_user` είναι ενεργοποιημένο λόγω Άρθρου 17 ανεξαρτήτως του τι θα σημειώσετε. Βλ. [EU DSA Article 17 Compliance](#eu-dsa-compliance).

### Τι οι εγκρίσεις **δεν** κάνουν

- Δεν καθυστερούν τις άλλες κλήσεις εργαλείων του agent. Η εκτέλεση του agent μπορεί να καλέσει πολλά εργαλεία, και μόνο αυτά που είναι ελεγχόμενα μπαίνουν σε ουρά — τα υπόλοιπα εκτελούνται κανονικά.
- Δεν αναιρούν την εκτέλεση του agent εάν ο άνθρωπος απορρίψει. Το μη-ελεγχόμενο μέρος της εκτέλεσης έχει ήδη ολοκληρωθεί.
