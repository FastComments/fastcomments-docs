Τα agent webhooks είναι HTTP callbacks που ενεργοποιούνται από την πλατφόρμα όταν μια εκτέλεση agent ολοκληρώνεται ή όταν αλλάζει η κατάσταση μιας έγκρισης. Χρησιμοποιήστε τα για να προωθήσετε τη δραστηριότητα των agents στα δικά σας συστήματα — πίνακες ελέγχου moderation, αρχεία ελέγχου (audit logs), κανάλια Slack, εργαλεία κλιμάκωσης.

Διαμορφώνονται στην καρτέλα **Webhooks** στη [Σελίδα AI Agents](https://fastcomments.com/auth/my-account/ai-agents).

### Τι παραδίδεται

Τέσσερις τύποι γεγονότων:

- **`trigger.succeeded`** - μια εκτέλεση agent ολοκληρώθηκε επιτυχώς.
- **`trigger.failed`** - μια εκτέλεση agent παρουσίασε σφάλμα.
- **`approval.requested`** - μια ενέργεια τοποθετήθηκε σε ουρά για ανθρώπινη έγκριση.
- **`approval.decided`** - μια έγκριση εγκρίθηκε, απορρίφθηκε, ή η εκτέλεση απέτυχε.

Δείτε [Webhook Events](#webhook-events) για το πότε εκπυρσοκροτούν τα γεγονότα, και [Webhook Payloads](#webhook-payloads) για το σχήμα του καθενός.

### Τι δεν παραδίδεται

- **Per-tool-action webhooks.** Μια εκτέλεση που καλεί `pin_comment` δεν πυροδοτεί ξεχωριστό webhook για το pin — η ενέργεια περιλαμβάνεται στο payload `trigger.succeeded` της εκτέλεσης. Αν θέλετε παράδοση ανά ενέργεια, αναλύστε τον πίνακα `actions` στο payload του trigger.
- **Dropped triggers.** Ένα trigger που δεν αποστέλλεται (λόγω υπέρβασης προϋπολογισμού, λάθος scope) δεν πυροδοτεί webhook. Οι απορρίψεις είναι ορατές μόνο στη [σελίδα Analytics](#analytics-page).
- **Replay-produced triggers.** Οι δοκιμαστικές εκτελέσεις δεν πυροδοτούν webhooks. Δείτε [Test Runs (Replays)](#test-runs-replays).

### Διαμόρφωση

Για κάθε webhook που ορίζετε:

- **URL** - το HTTPS endpoint στο οποίο γίνεται POST.
- **Domain** - το domain σχολίων στο οποίο εφαρμόζεται αυτό το webhook (ο tenant σας μπορεί να φιλοξενεί πολλαπλά domains). `*` ταιριάζει με όλα τα domains· `*.example.com` είναι glob· ένα ακριβές domain ταιριάζει μόνο με αυτό.
- **Events** - σε ποια από τους τέσσερις τύπους γεγονότων να εγγραφεί.
- **Agents** - κενό για "όλοι οι agents", ή μια λίστα με συγκεκριμένα IDs agents για φιλτράρισμα.
- **Method** - POST ή PUT (προεπιλογή POST).
- **No-retry status codes** - κωδικοί απάντησης HTTP που πρέπει να θεωρούνται τελικές αποτυχίες και να μην ξαναπροσπαθούν (π.χ., 410 Gone, 422 Unprocessable). Δείτε [Webhook Retries](#webhook-retries).

Πολλαπλά webhooks μπορούν να εγγραφούν στο ίδιο γεγονός — το καθένα μπαίνει στην ουρά ανεξάρτητα και παραδίδεται στο δικό του URL.

### Αντιστοίχιση ανά domain

Ένα δεδομένο γεγονός αποστέλλεται σε **κάθε webhook του οποίου το πεδίο `domain` ταιριάζει με το domain του σχολίου**. Η αντιστοίχιση χρησιμοποιεί έναν απλό glob:

- Ακριβές: `example.com` ταιριάζει μόνο με `example.com`.
- Wildcard αστέρι: `*` ταιριάζει με κάθε domain.
- Subdomain glob: `*.example.com` ταιριάζει με `blog.example.com`, `forum.example.com`, αλλά όχι με το ίδιο το `example.com`.

Το domain στο payload είναι το πρώτο μη-null αποτέλεσμα από: το `domain` του σχολίου, το URL που αναλύθηκε με βάση τη ρύθμιση domain του tenant σας, ή την αναζήτηση `Page` μέσω `urlId`.

### Φιλτράρισμα ανά agent

Το πεδίο **Agents** επιτρέπει σε ένα webhook να εγγραφεί μόνο για συγκεκριμένους agents. Κενό σημαίνει "όλοι οι agents". Όταν δεν είναι κενό, το webhook ενεργοποιείται μόνο για τους agents της λίστας.

Αυτό είναι χρήσιμο όταν έχετε ένα webhook για γεγονότα moderation και ένα άλλο για γεγονότα engagement, και τα δύο δρομολογούν σε διαφορετικά downstream συστήματα.

### Δοκιμαστική αποστολή

Το UI ρύθμισης webhook διαθέτει ένα κουμπί **Test send** που στέλνει ένα ψεύτικο payload στο URL ώστε να επαληθεύσετε τη συνδεσιμότητα, την υπογραφή και τον κωδικό απάντησης του endpoint σας χωρίς να περιμένετε ένα πραγματικό γεγονός.

### Καταγραφές παράδοσης

Κάθε παράδοση (και κάθε προσπάθεια επανάληψης) εμφανίζεται στη σελίδα [Webhook Delivery Logs](#webhook-logs) ώστε να μπορείτε να δείτε τι συνέβη στο δίκτυο.

### Υπογραφή

Κάθε webhook υπογράφεται με HMAC-SHA256 χρησιμοποιώντας το API secret του tenant σας. Δείτε [Webhook Signing](#webhook-signing).

### Προϋποθέσεις

Τα agent webhooks απαιτούν έγκυρη χρέωση στον tenant. Χρησιμοποιούν την ίδια υποδομή υπογραφής/μυστικού όπως τα υπάρχοντα webhooks σχολίων σας — αν έχετε ήδη ενσωματώσει webhooks σχολίων (δείτε τον [Οδηγό Webhooks](/guide-webhooks.html)), η ενσωμάτωση των agent webhooks έχει την ίδια δομή με νέους τύπους γεγονότων.
