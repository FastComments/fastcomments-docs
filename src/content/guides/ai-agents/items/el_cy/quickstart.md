Αυτή είναι η πεντάλεπτη διαδρομή από «we have AI Agents» μέχρι «an agent is responding to live traffic, gated by approvals.» Αν θέλετε τη μακροσκελέστερη εκδοχή, κάθε βήμα συνδέεται με τη σελίδα που το καλύπτει λεπτομερώς.

### 1. Ανοίξτε τη σελίδα AI Agents

Πηγαίνετε στα [AI Agents](https://fastcomments.com/auth/my-account/ai-agents) στο λογαριασμό σας. Την πρώτη φορά που θα προσγειωθείτε εδώ θα δείτε είτε:

- Μια κατάσταση χωρίς περιεχόμενο με κουμπιά **Browse templates** και **Start from scratch** (έχετε διαθέσιμους agents για δημιουργία), ή
- Μια σελίδα upsell αν το πρόγραμμά σας δεν περιλαμβάνει agents - δείτε [Plans and Eligibility](#plans-and-eligibility).

### 2. Επιλέξτε ένα αρχικό πρότυπο

Κάντε κλικ στο **Browse templates**. Επιλέξτε ένα από τα:

- [Moderator](#template-moderator) - εξετάζει επισημασμένα ή νέα σχόλια, προειδοποιεί τους πρώτους παραβάτες, και κλιμακώνει σε αποκλεισμό μόνο μετά από προειδοποίηση.
- [Welcome Greeter](#template-welcome-greeter) - απαντά σε χρήστες που σχολιάζουν για πρώτη φορά.
- [Top Comment Pinner](#template-top-comment-pinner) - καρφιτσώνει ουσιαστικά σχόλια όταν ξεπεράσουν ένα όριο ψήφων.
- [Thread Summarizer](#template-thread-summarizer) - δημοσιεύει μια ουδέτερη περίληψη σε μεγάλες συζητήσεις.

Κάθε πρότυπο ανοίγει μια προκαταγεμισμένη φόρμα επεξεργασίας με την **Status: Dry Run** ήδη επιλεγμένη.

### 3. Επανεξέταση και αποθήκευση

Στη φόρμα επεξεργασίας, κάντε τουλάχιστον τα εξής:

- **Internal name.** Μια σύντομη ταυτότητα που χρησιμοποιείται στους πίνακες διαχείρισης.
- **Display name.** Αυτό που εμφανίζεται δημόσια όταν ο agent δημοσιεύει ένα σχόλιο.
- **Initial prompt.** Επεξεργαστείτε το prompt του προτύπου ώστε να ταιριάζει στη φωνή σας και στους συγκεκριμένους κανόνες σας.
- **Approvals.** Επιλέξτε τις ενέργειες που θα απαιτούν ανθρώπινη αναθεώρηση πριν ενεργοποιηθούν. Προτείνουμε τουλάχιστον το `ban_user` για οποιονδήποτε agent με σκοπό τη moderation. Δείτε [Approval Workflow](#approval-workflow).

Κάντε κλικ στο **Save agent**.

### 4. Παρακολουθήστε το σε dry-run

Ο agent είναι τώρα ενεργός σε **Dry Run**. Θα λαμβάνει τα triggers του, θα καλεί το μοντέλο, και θα καταγράφει ενέργειες στη σελίδα [Run History](#run-history) - με την ένδειξη **Dry Run** σε κάθε γραμμή - αλλά δεν εκτελεί πραγματικές ενέργειες. Επισκεφθείτε μερικές λεπτομέρειες εκτέλεσης (δείτε [Run Detail View](#run-detail-view)) και δείτε:

- Τις ενέργειες που επέλεξε ο agent.
- Την αιτιολόγηση και την αυτοπεποίθηση για κάθε ενέργεια.
- Το πλήρες transcript του LLM.

Αν ο agent παίρνει αποφάσεις με τις οποίες διαφωνείτε, επεξεργαστείτε το initial prompt ή επιλέξτε περισσότερα approvals.

### 5. Τρέξτε ένα τεστ σε παλιά σχόλια

Από τη σελίδα λίστας agents, κάντε κλικ στο **Test run** στη γραμμή του agent. Η φόρμα έχει ένα μόνο αριθμητικό πεδίο **Days** (1 έως 90). Το μέγεθος του δείγματος και το ανώτατο όριο στα σχόλια που αξιολογούνται εμφανίζονται πληροφοριακά - υπολογίζονται server-side, δεν τίθενται από τον χρήστη. Το replay τρέχει σε ιστορικά σχόλια χωρίς να εκτελεί πραγματικές ενέργειες και αναφέρει τι ο agent **θα** είχε κάνει σε σύγκριση με αυτό που συνέβη στην πραγματικότητα (εάν το σχόλιο αργότερα εγκρίθηκε, χαρακτηρίστηκε spam, διαγράφηκε, κ.λπ.). Δείτε [Test Runs (Replays)](#test-runs-replays).

### 6. Αλλάξτε σε Enabled

Όταν είστε ικανοποιημένοι με το dry-run και τα αποτελέσματα του replay, επεξεργαστείτε τον agent και αλλάξτε την **Status** σε **Enabled**. Από εδώ και πέρα, οι πραγματικές ενέργειες εκτελούνται. Η σελίδα Run History τώρα δείχνει live runs χωρίς την ένδειξη dry-run, και οποιαδήποτε ενέργεια έχετε επισημάνει για έγκριση εμφανίζεται στο [approvals inbox](#approval-workflow).

### Τι ακολουθεί

- Διαμορφώστε [Webhooks](#webhooks-overview) αν θέλετε εξωτερικά συστήματα να αντιδρούν σε γεγονότα των agents.
- Προσθέστε [Community Guidelines](#community-guidelines) για να κρατήσετε τις αποφάσεις των agents ευθυγραμμισμένες με την γραπτή πολιτική σας.
