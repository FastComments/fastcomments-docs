代理的**工具**是它可以采取的动作。代理编辑表单有一个 **Allowed tool calls** 部分，在那里勾选此代理被允许使用的工具；还有一个 **Approvals** 部分，在那里勾选在生效前需要人工批准的操作。

对于任何工具都有三个级别：

- **Disallowed** - 代理无法看到或使用它。
- **Allowed, no approval** - 代理直接使用。记录在运行历史中。
- **Allowed, with approval** - 代理的调用会排队等待人工审核，只有在人工批准后才会运行。

被禁止的工具是静默的：代理不能请求它们，平台会直接拒绝。需要审批的工具总是通过 [approvals inbox](#approval-workflow)。

### 每个操作的审计轨迹

代理采取的每个操作都会记录一个简短的理由（1-2 句解释原因）和一个置信度分数（0.0-1.0）。两者都会出现在 [Run Detail View](#run-detail-view) 和每次 [approval](#approval-workflow) 上。搜索记忆是唯一的只读例外：它不会被记录为一次操作，并且无论白名单如何总是可用。

### 工具参考

#### Posting comments

允许代理以自身身份发布评论。评论会以代理的显示名称公开显示。迎宾代理和摘要代理会使用此工具。可逆——任何版主都可以删除不当评论。通常在不需要审批的情况下允许；如果您的社区要求每条面向公众的消息都由人工审核，则应对其进行审批限制。

#### Editing a comment

允许代理重写范围内评论的文本。原始文本会保存在评论的审计日志中。保留用于狭义场景——例如编辑用户泄露的个人身份信息（PII），或修改代理自己之前的回复。不应用于改写观点或软化语气。**强烈建议将其设置为需要审批。** 有关完整页面，请参见 [Edit comment](#tool-edit-comment)。

#### Voting on comments

允许代理对评论投赞成或反对票。投票会像其他投票一样计入评论的总票数。大多数社区不希望机器人参与投票；在任何入门模板中均未启用。如果您允许它，投票是可撤销的。

#### Pin / unpin a comment

允许代理将评论置顶到页面顶部，或取消已置顶的评论。平台不强制实施每个主题仅允许一个置顶的规则，因此应指示置顶代理先取消先前的置顶评论。由 Top Comment Pinner 模板使用。可撤销；通常无需审批即可允许。

#### Lock / unlock a comment

允许代理阻止对评论的进一步回复，或恢复回复。被锁定的评论仍然可见。适用于对激烈讨论进行冷却并配合延迟解锁。可撤销但会对社区可见；在高风险社区中考虑将其设为需要审批。

#### Mark / unmark spam

允许代理将评论标记为垃圾（对读者隐藏并送入垃圾信息分类器）或清除此标记。任何审核型代理的基本工具。可撤销。在您建立对代理的信任的最初几周内，强烈建议将其设为需要审批。

#### Approve / un-approve a comment

允许代理将被保持的评论显示给读者，或隐藏已显示的评论。在那些将新评论保留供版主审核的租户中最有用。对已显示评论进行取消批准的风险较高——考虑将其设为需要审批。

#### Mark a comment reviewed

一个队列状态工具：将评论标记为“版主（或代理）已查看”。不会更改可见性。风险低；很少设为需要审批。

#### Award a badge

允许代理向您为租户配置的用户颁发徽章。版主可撤销。很少设为需要审批。当启用此工具时，代理可以看到您租户的徽章并自行选择合适的徽章，因此您无需在社区准则或初始提示中粘贴徽章标识符。如果您想指引应为哪些行为颁发哪个徽章，请在提示中使用徽章的**显示标签**来引用它们。

#### Send email

允许代理向触发器范围内的评论作者发送纯文本电子邮件。代理永远看不到收件人的电子邮件地址——它选择一条评论，平台将发送到该评论者发布时留下的地址。发件地址是当评论域名与已配置域匹配时使用您租户的品牌发件人（带 DKIM），否则使用平台默认地址。谨慎使用——电子邮件是摩擦成本最高的工具，错误的邮件很难撤销。强烈建议将其设为需要审批，并将审批邮件路由到代理最终会发送邮件的收件箱所有者。

#### Save / search agent memory

两个配套工具，用于读取和写入有关触发器触发的用户的共享笔记池。记忆在您租户的所有代理之间共享，因此分诊代理的笔记会为版主代理的决策提供信息。搜索为只读且始终可用；保存很少受限。有关完整设计，请参见 [Agent Memory System](#agent-memory-system)。

#### Warn a user

向用户就某条特定评论发送私密私信警告，并原子性地将警告记录到代理记忆中。平台的升级策略围绕此工具构建——先警告，只有在用户再次违规时才封禁。比 `ban_user` 更少设为需要审批，但在代理运作的最初几周内仍应考虑设置审批。有关完整页面，请参见 [Warn user](#tool-warn-user)。

#### Ban a user

代理可以调用的最具后果性的工具。对用户进行固定时长的封禁，可选择作为影子封禁、可选择同时封禁 IP、可选择同时删除该用户的所有评论。两个破坏性选项（IP、删除全部）在编辑表单上通过额外的选择进行授权后才可用。在欧盟区域，所有封禁都需要人工批准（参见 [EU DSA Article 17 Compliance](#eu-dsa-compliance)）。强烈建议在所有地方将其设为需要审批。有关完整页面，请参见 [Ban user](#tool-ban-user)。

### Ban-tool 子选项

Ban 工具暴露了两个破坏性选项 - delete-all-comments 和 ban-by-IP - 在您通过编辑表单上的 **Ban options** 部分选择启用之前，这些选项对模型完全隐藏。即使模型虚构该参数，平台也会拒绝您未选择的值。详见 [Ban user](#tool-ban-user)。
