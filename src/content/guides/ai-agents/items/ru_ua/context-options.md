Раздел **Context** в форме редактирования управляет тем, сколько информации агент получает при каждом запуске. Больше контекста улучшает качество решений, но увеличивает стоимость в токенах за запуск, поэтому нужно включать только то, что действительно необходимо агенту.

### Что всегда включено

Даже при снятии всех флажков в сообщение контекста агента включается:

- Тип события-триггера (например `COMMENT_ADD`, `COMMENT_FLAG_THRESHOLD`).
- URL страницы и ID URL (если известно).
- Комментарий, который вызвал запуск, если он есть — ID, ID автора, отображаемое имя автора, текст комментария, количество голосов, количество флагов, флаги spam/approved/reviewed, ID родителя. Электронная почта автора **никогда** не отправляется поставщику LLM (минимизация PII).
- Предыдущий текст комментария для триггеров `COMMENT_EDIT` (чтобы агент мог сравнить до/после).
- Направление голосования для триггеров `COMMENT_VOTE_THRESHOLD`.
- ID вызывающего пользователя и ID бейджа (для триггеров бейджей модераторов).

Весь ненадёжный текст — тела комментариев, имена авторов, заголовки страниц, сам документ с руководящими принципами — **ограждается** в сообщении контекста маркерами вроде `<<<COMMENT_TEXT>>> ... <<<END>>>`. Системный промпт платформы инструктирует модель никогда не выполнять инструкции внутри этих ограждений. Это защита платформы от prompt-инъекций; вам не нужно повторять это в своём промпте.

### Три чекбокса

#### Включить родительский комментарий и предыдущие ответы в том же треде

Добавляет:
- **Родительский комментарий** — ID, автор, текст.
- **Соседние ответы** — предыдущие ответы на тот же родительский комментарий в том же треде.

Полезно для: любых агентов, которые отвечают на комментарий в контексте (приветственные боты, суммаризаторы тредов, модераторы, читающие ответы в беседах).

Стоимость: от небольшой до средней. Ограничено количеством ответов в данном треде.

#### Включить фактор доверия комментатора, возраст аккаунта, историю банов и недавние комментарии

Добавляет блок **AUTHOR_HISTORY**:

- **Возраст аккаунта в днях** с момента регистрации.
- **Фактор доверия (0-100)** — оценка FastComments, которая суммирует, насколько пользователю доверяют на этом сайте. См. страницу [Обнаружение спама](/guide-moderation.html#spam-detection) в руководстве по модерации.
- **Количество прежних банов.**
- **Общее количество комментариев на этом сайте.**
- **Количество дубликатов контента** — если пользователь публиковал идентичный текст недавно (сигнал антиспама).
- **Сигнал перекрёстных аккаунтов по тому же IP** — количество комментариев с того же IP под другими аккаунтами (сигнал о дополнительных аккаунтах). Сам хэш IP никогда не отправляется поставщику LLM.
- **Недавние комментарии** — до 5 последних комментариев пользователя, каждый обрезается до 300 символов и ограждается как ненадёжный текст.

Полезно для: любых модерационных агентов. Без этого модель может блокировать новые аккаунты и давних добросовестных пользователей с похожим поведением.

Стоимость: средняя. Недавние комментарии добавляют наибольшее количество токенов.

#### Включить заголовок страницы, подзаголовок, описание и мета-теги

Добавляет блок **PAGE_CONTEXT** — заголовок, подзаголовок, описание и любые мета-теги, которые FastComments собрал для страницы.

Полезно для: приветственных агентов и суммаризаторов тредов, когда знание тематики страницы значительно улучшает качество вывода.

Стоимость: небольшая.

### Правила сообщества

Четвёртое поле, **Community guidelines**, — это блок политики в свободной форме, который включается в сообщение контекста роли пользователя при каждом запуске и ограждается как ненадёжный текст так же, как тела комментариев и другой контент, предоставленный пользователями. Агент читает его как текст политики, но платформа не рассматривает его как системную инструкцию. См. [Правила сообщества](#community-guidelines), чтобы узнать, что в него помещать.

### Выборочное добавление контекста

Эти чекбоксы применяются для каждого агента отдельно, а не глобально. Распространённые варианты:

- Приветствующий агент: контекст страницы **вкл.**, контекст треда **выкл.**, история пользователя **выкл.**.
- Модератор: контекст треда **выкл.**, история пользователя **вкл.**, контекст страницы **выкл.**.
- Суммаризатор треда: контекст треда **вкл.**, контекст страницы **вкл.**, история пользователя **выкл.**.

Стремитесь давать минимально необходимый контекст, достаточный для корректной работы агента при фактических вызовах — лишний контекст увеличивает расход токенов при каждом запуске, даже если агент его не использует.
