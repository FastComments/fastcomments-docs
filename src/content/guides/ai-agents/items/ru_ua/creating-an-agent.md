С [страницы AI Agents](https://fastcomments.com/auth/my-account/ai-agents) вы можете создать агента двумя способами:

- **Из шаблона.** Нажмите **Browse templates** и выберите один из четырёх встроенных стартовых агентов. Форма заполняется заранее, и статус агента — **Тестовый запуск**. См. [Starter Templates](#starter-templates).
- **С нуля.** Нажмите **Create new agent**. Форма открывается пустой.

В любом случае та же форма редактирования используется для сохранения и последующего редактирования. Эта страница описывает форму сверху вниз.

### Основы

- **Internal name.** Короткий идентификатор, используемый только в админских панелях (история запусков, аналитика, журналы аудита). Подходит нижний регистр с подчеркиваниями: `moderator`, `welcome_greeter`. Если внутреннее имя шаблона уже занято, форма автоматически добавит суффикс (`tos_enforcer_2` и т.д.).
- **Display name.** Отображается публично каждый раз, когда агент публикует комментарий. Это то, что увидят ваши читатели.
- **Status.** Отключено, Тестовый запуск или Включено. Новые агенты по умолчанию всегда в Тестовом запуске. См. [Status States](#status-states).

### Model

Выберите LLM. См. [Choosing a Model](#choosing-a-model).

### Budget

Необязательные дневные и месячные лимиты в валюте вашего аккаунта, плюс контрольный список **Alert thresholds** (по умолчанию 80% и 100%). См. [Budgets Overview](#budgets-overview) и [Budget Alerts](#budget-alerts).

### Personality

**Initial prompt** — это системный промпт, который задаёт тон, роль и правила принятия решений. Обычный текст, без синтаксиса шаблонов. См. [Personality and the Initial Prompt](#personality-prompt).

### Context

Набор полей Context содержит три флажка, текстовую область с руководящими принципами и поля области действия (scope):

- Включать родительский комментарий и предыдущие ответы в той же ветке.
- Включать фактор доверия комментатора, возраст аккаунта, историю банов и недавние комментарии.
- Включать заголовок страницы, подзаголовок, описание и мета-теги.
- Необязательный блок текста **Community guidelines**, который будет предварительно добавляться к каждому промпту.
- **Restrict to specific pages** — allowlist шаблонов URL (по одному на строку). Пустое значение означает весь тенант.
- **Restrict to specific locales** — allowlist локалей через двухпанельный селектор. Пустое значение означает все локали.

Больше контекста даёт лучшие решения, но увеличивает стоимость токенов за запуск. См. [Context Options](#context-options), [Community Guidelines](#community-guidelines) и [Scope: URL and Locale Filters](#scope-url-locale).

### Triggers

Выберите как минимум одно событие из списка. Для триггеров vote-threshold и flag-threshold вы также должны установить порог. Необязательное поле **Delay before running** откладывает выполнение после срабатывания триггера (полезно для порогов флагов, где голоса ещё уточняются). См. [Trigger Events Overview](#triggers-overview) и [Deferred Triggers](#trigger-deferred-delay).

### Allowed tool calls

Отметьте **Allow any tool calls**, чтобы открыть полный набор инструментов. В противном случае отметьте конкретные инструменты, которыми агент может пользоваться — запрещённые инструменты будут удалены из палитры модели и отклонены при отправке. Подраздел **Ban options** ограничивает вариации разрушительных банов (delete-all-comments, ban-by-IP) и требует явного согласия. См. [Allowed Tool Calls Overview](#tools-overview) и [Tool: ban_user](#tool-ban-user).

### Approvals

Отметьте действия, которые должны быть одобрены человеком до того, как агент их выполнит. Одобрения применимы только к инструментам, которые агенту разрешено вызывать; запрещённые инструменты отклоняются сразу. В регионе ЕС **ban_user** включён в силу Статьи 17 Digital Services Act. См. [Approval Workflow](#approval-workflow) и [EU DSA Article 17 Compliance](#eu-dsa-compliance).

### Approval notifications

Если одобрения включены, выберите, кому отправляются уведомления по электронной почте:

- **All admins and moderators** — владельцы аккаунта, супер-админы и админы модераторов комментариев.
- **Specific users** — отобранные вручную через двухпанельный селектор.

Индивидуальная частота доставки уведомлений для каждого рецензента (немедленно, почасовой дайджест, суточный дайджест) задаётся в их профиле. См. [Approval Notifications](#approval-notifications).

### Stats

Только для чтения. Общее количество запусков, метка времени последнего запуска и ID последнего комментария, написанного агентом (если есть).

### Save

Нажмите **Save agent**. Страница перенаправит обратно к списку агентов. Новые агенты сразу могут получать триггеры в режиме тестового запуска.

### Editing later

Каждая строка на странице списка агентов содержит действия для каждого агента: **Edit**, **Clone**, **Runs**, **Replays**, **Test run**, **Analytics**, **Approvals** и **Delete**. Редактирование агента не влияет ретроспективно на уже записанные запуски — история сохраняется. Снапшоты повторов (replay snapshots) также фиксируют конфигурацию агента на момент запуска повтора, поэтому результаты сохранённого повтора остаются воспроизводимыми даже после редактирования промпта.
