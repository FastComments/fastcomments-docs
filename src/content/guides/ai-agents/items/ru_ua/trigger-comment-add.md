Запускает агента каждый раз, когда на странице, охваченной [областью действия](#scope-url-locale) агента, публикуется новый комментарий.

### Контекст, который получает агент

- Новый комментарий целиком - текст, автор, голоса, ID родителя, ID URL страницы.
- По желанию: родительский комментарий и предыдущие ответы в той же ветке, если включён [контекст ветки](#context-options).
- По желанию: коэффициент доверия комментатора, возраст аккаунта, история банов и недавние комментарии, если включён [контекст истории пользователя](#context-options).
- По желанию: метаданные страницы, если включён [контекст страницы](#context-options).

### Примечательно

- Триггер срабатывает **после** того, как комментарий был сохранён. Агент может ссылаться на него напрямую в вызовах инструментов.
- Он **не** срабатывает для комментариев, написанных другим агентом в том же тенанте.
- Срабатывает для как подтверждённых, так и неподтверждённых комментариев. Если в вашем тенанте требуется одобрение модератора перед тем, как комментарий станет видимым (см. [Как работает одобрение](/guide-moderation.html#moderation-approvals) в руководстве по модерации), триггер срабатывает при создании комментария, а не при его последующем одобрении. Модератор-бот может быть настроен на одобрение комментариев за вас после проверки.

### Общие случаи использования

- **Модерация** - проверять комментарий на соответствие правилам сообщества, помечать спам или предупреждать новичков.
- **Приветствие** - хотя [Триггер: первый комментарий нового пользователя](#trigger-new-user-first-comment) обычно лучше подходит для приветствий, так как он срабатывает один раз для каждого пользователя.
- **Суммаризация ветки** - обычно в паре с [задержкой триггера](#trigger-deferred-delay), чтобы ветка успела устаканиться перед запуском агента.
