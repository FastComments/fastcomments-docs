在編輯表單上的 **上下文 (Context)** 區段控制代理程式在每次執行時收到多少資訊。更多的上下文會產生更好的決策但會提高每次執行的 token 成本，所以你只想包含代理實際需要的資訊。

### 始終包含的項目

即使所有核取方塊都未勾選，代理程式的上下文訊息仍會包含：

- 觸發事件類型，例如 `COMMENT_ADD`、`COMMENT_FLAG_THRESHOLD`。
- 當已知時的頁面 URL 與 URL ID。
- 觸發執行的 **評論**（如果有）——ID、作者使用者 ID、作者顯示名稱、評論文字、投票數、檢舉數、spam/approved/reviewed 標記、父評論 ID。作者的電子郵件絕對不會傳給 LLM 提供者（PII 最小化）。
- 對於 `COMMENT_EDIT` 觸發，包含 **先前的評論文字**（以便代理可以比較前後差異）。
- 對於 `COMMENT_VOTE_THRESHOLD` 觸發，包含 **投票方向**。
- 觸發者的使用者 ID 與徽章 ID（用於版主徽章觸發）。
- 當代理被允許頒發徽章時，包含你租戶的 **徽章目錄**（名稱、顯示標籤、描述），以便代理在不需要你在提示中詳述徽章的情況下選擇合適的徽章。

所有不受信任的文字——評論內容、作者名稱、頁面標題、指引文件本身——在上下文訊息中都會以像 `<<<COMMENT_TEXT>>> ... <<<END>>>` 的標記進行「圍欄(fenced)」。平台的系統提示會指示模型絕對不要遵從這些圍欄內的指令。這是平台的提示注入防禦；你不需要在你的提示中重複它。

### 三個核取方塊

#### 將父評論與同一討論串中的先前回覆一起包含

新增：
- **父評論** - ID、作者、文字。
- **同層回覆** - 對同一父評論的先前回覆（位於相同討論串中）。

適用於：任何需要在上下文中回應評論的代理（歡迎致詞者、討論串摘要器、閱讀會話中回覆的版主）。

成本：小到中等。受限於特定討論串中同層回覆的數量。

#### 包含評論者的信任指數、帳號年資、封鎖紀錄與近期評論

新增 **AUTHOR_HISTORY** 區塊：

- 自註冊以來的 **帳號年齡（天）**。
- **信任指數（0-100）** — FastComments 的分數，用以總結該使用者在此站點的信任程度。請參閱 [垃圾訊息偵測](/guide-moderation.html#spam-detection) 頁面（位於版務指南）。
- 先前的封鎖次數。
- 該使用者在此站的總留言數。
- **重複內容計數** — 如果使用者近期發表過相同文字（反垃圾訊號）。
- **相同 IP 的跨帳號訊號** — 來自相同 IP 在其他帳號下的評論數（分帳號訊號）。IP 的雜湊值本身絕不會傳送給 LLM。
- **最近的評論** — 最多 5 筆使用者最近的評論，每則截斷為 300 個字元，並以不受信任的文字方式圍欄。

適用於：任何版務代理。沒有這些資訊時，模型容易對新帳號以及長期良善的使用者採取相同的封鎖姿態。

成本：中等。最近的評論會增加最多的 tokens。

#### 包含頁面標題、副標題、描述與 meta 標籤

新增 **PAGE_CONTEXT** 區塊——標題、副標題、描述，以及 FastComments 為該頁面擷取的任何 meta 標籤。

適用於：歡迎致詞者與討論串摘要器，了解頁面內容會大幅提升輸出品質。

成本：小。

### 社群守則

第四個欄位，**Community guidelines**，是一個自由文字的政策區塊，會在每次執行時包含在使用者角色的上下文訊息中，並以與評論內容和其他使用者提供的文字相同的方式作為不受信任文字圍欄。代理將其視為政策文字，但平台不會將其當作系統指令來處理。請參閱 [社群守則](#community-guidelines) 了解應放入內容的範例。

### 選擇性地新增上下文

這些核取方塊是針對每個代理設定，而非全域設定。一個常見的模式：

- 歡迎致詞者：頁面上下文 **開啟**、討論串上下文 **關閉**、使用者歷史 **關閉**。
- 版主：討論串上下文 **關閉**、使用者歷史 **開啟**、頁面上下文 **關閉**。
- 討論串摘要器：討論串上下文 **開啟**、頁面上下文 **開啟**、使用者歷史 **關閉**。

在代理實際會執行的呼叫上，盡量只提供代理正確運作所需的最少上下文——額外的上下文在每次執行時都會花費 tokens，即使代理並未使用它。
