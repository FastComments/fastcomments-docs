에이전트의 **도구(tools)** 는 에이전트가 수행할 수 있는 동작입니다. 에이전트 편집 폼에는 이 에이전트가 사용할 수 있도록 허용할 도구를 선택하는 **Allowed tool calls** 섹션과, 실행되기 전에 사람이 승인해야 하는 동작을 선택하는 **Approvals** 섹션이 있습니다.

도구에는 세 가지 수준이 있습니다:

- **Disallowed** - 에이전트는 해당 도구를 보거나 사용할 수 없습니다.
- **Allowed, no approval** - 에이전트가 직접 사용합니다. 실행 기록에 기록됩니다.
- **Allowed, with approval** - 에이전트의 호출은 사람의 검토를 위해 대기열에 들어가며 사람이 승인할 때만 실행됩니다.

차단된 도구는 침묵합니다: 에이전트가 요청할 수 없고 플랫폼이 이를 전면 거부합니다. 승인이 필요한 도구는 항상 [승인 수신함](#approval-workflow)을 통해 처리됩니다.

### 모든 작업에 대한 감사 추적

에이전트가 수행하는 모든 동작은 간단한 정당성(왜 했는지 1-2문장으로 설명)과 신뢰도 점수(0.0-1.0)와 함께 기록됩니다. 두 항목은 [Run Detail View](#run-detail-view)와 모든 [승인](#approval-workflow)에 표시됩니다. 메모리 검색은 읽기 전용의 예외로, 동작으로 기록되지 않으며 허용 목록과 관계없이 항상 사용 가능합니다.

### 도구 참고

#### 댓글 게시

에이전트가 자신으로서 댓글을 게시할 수 있게 합니다. 댓글은 에이전트의 표시 이름으로 공개적으로 표시됩니다. 인사 담당(greeter) 및 요약자(summarizer) 에이전트에서 사용됩니다. 되돌릴 수 있음 - 어떤 중재자라도 문제가 있는 댓글을 제거할 수 있습니다. 보통 승인 없이 허용되지만, 모든 공개 메시지가 사람의 검토를 받아야 하는 커뮤니티라면 승인이 필요하도록 설정하세요.

#### 댓글 편집

에이전트가 범위 내 댓글의 텍스트를 다시 작성할 수 있게 합니다. 원본 텍스트는 댓글의 감사 로그에 보존됩니다. 사용 범위를 좁게 제한하세요 - 사용자가 유출한 개인식별정보(PII)를 삭제하거나 에이전트가 이전에 쓴 답변을 수정하는 경우 등에만 사용합니다. 의견을 재작성하거나 어조를 부드럽게 하는 데 사용하지 마세요. **승인 뒤에 두는 것을 강력히 고려하세요.** 전체 페이지는 [Edit comment](#tool-edit-comment)를 참조하세요.

#### 댓글에 대한 투표

에이전트가 댓글에 찬성 또는 반대 투표를 할 수 있게 합니다. 투표는 다른 투표와 마찬가지로 댓글의 투표 합계에 반영됩니다. 대부분의 커뮤니티는 봇의 투표를 원하지 않으며, 어떤 스타터 템플릿에도 기본적으로 활성화되어 있지 않습니다. 허용할 경우 투표는 되돌릴 수 있습니다.

#### 댓글 고정 / 고정 해제

에이전트가 페이지 상단에 댓글을 고정하거나 이미 고정된 댓글의 고정을 해제할 수 있게 합니다. 플랫폼은 스레드당 하나만 고정하는 규칙을 강제하지 않으므로, 고정하는 에이전트는 이전에 고정된 댓글의 고정을 먼저 해제하도록 지시해야 합니다. Top Comment Pinner 템플릿에서 사용됩니다. 되돌릴 수 있음; 보통 승인 없이 허용됩니다.

#### 댓글 잠금 / 잠금 해제

에이전트가 댓글 아래의 추가 답글을 차단하거나 답글을 복원할 수 있게 합니다. 잠긴 댓글은 계속 표시됩니다. 과열된 스레드의 쿨다운으로 유용하며, 지연된 잠금 해제와 함께 사용됩니다. 되돌릴 수 있으나 커뮤니티에 노출되므로, 중요도가 높은 커뮤니티에서는 승인을 요구하는 것을 고려하세요.

#### 스팸 표시 / 표시 해제

에이전트가 댓글을 스팸으로 표시(독자에게 숨기고 스팸 분류기에 피드)하거나 해당 플래그를 제거할 수 있게 합니다. 모든 중재 에이전트의 기본적인 도구입니다. 되돌릴 수 있음. 에이전트에 대한 신뢰를 쌓는 초기 몇 주 동안에는 승인을 요구하는 것을 강력히 고려하세요.

#### 댓글 승인 / 승인 취소

에이전트가 보류된 댓글을 독자에게 보여주거나 이미 표시된 댓글을 숨길 수 있게 합니다. 주로 새 댓글을 중재자 검토를 위해 보류하는 테넌트에서 유용합니다. 표시된 댓글을 비승인 처리하는 것은 위험 부담이 크므로 전역적으로 승인을 요구하는 것을 고려하세요.

#### 댓글 검토로 표시

큐 상태 도구: 댓글을 "중재자(또는 에이전트)가 확인함"으로 표시합니다. 가시성은 변경하지 않습니다. 위험 부담이 낮아 드물게만 승인을 요구합니다.

#### 배지 수여

에이전트가 테넌트에 구성된 배지를 사용자에게 줄 수 있게 합니다. 중재자가 되돌릴 수 있습니다. 드물게만 승인을 요구합니다. 이 도구가 활성화되면 에이전트는 테넌트의 배지를 보고 스스로 적절한 것을 선택할 수 있으므로 커뮤니티 지침이나 초기 프롬프트에 배지 식별자를 붙여넣을 필요가 없습니다. 어떤 행동에 어떤 배지를 수여할지 유도하고 싶다면 프롬프트에서 배지의 **표시 라벨(Display Label)** 로 참조하세요.

#### 이메일 전송

에이전트가 트리거의 범위에 있는 댓글 작성자에게 일반 텍스트 이메일을 보낼 수 있게 합니다. 에이전트는 수신자의 이메일 주소를 절대 보지 못합니다 - 에이전트는 댓글을 선택하고 플랫폼이 그 댓글 작성자가 남긴 주소로 전달합니다. 발신 주소는 댓글의 도메인이 구성된 도메인과 일치하면 테넌트의 브랜드 발신자(DKIM 적용)이며, 그렇지 않으면 플랫폼 기본 발신자입니다. 신중히 사용하세요 - 이메일은 가장 높은 마찰을 가진 도구이고 잘못 보낸 이메일은 되돌리기 어렵습니다. 승인을 강력히 고려하고, 에이전트가 결국 이메일을 보낼 대상의 받은편지함을 소유한 사람에게 승인 이메일을 경로로 지정하세요.

#### 에이전트 메모리 저장 / 검색

트리거가 발생한 사용자에 대한 공유된 메모 풀을 읽고 쓰는 두 개의 쌍으로 구성된 도구입니다. 메모리는 테넌트의 모든 에이전트 간에 공유되므로, 분류(triage) 에이전트의 메모가 중재 에이전트의 결정에 영향을 줍니다. 검색은 읽기 전용이며 항상 사용 가능하고; 저장은 드물게만 승인을 요구합니다. 전체 설계는 [Agent Memory System](#agent-memory-system)을 참조하세요.

#### 사용자 경고

특정 댓글에 대해 사용자에게 비공개 DM 경고를 보내고 경고를 에이전트 메모리에 원자적으로 기록합니다. 플랫폼의 에스컬레이션 정책은 이 도구를 중심으로 구성되어 있습니다 - 먼저 경고하고, 사용자가 재범하면 차단(ban)합니다. `ban_user`보다 덜 자주 승인을 요구하지만, 에이전트 도입 초기 몇 주 동안에는 승인을 고려하세요. 전체 페이지는 [Warn user](#tool-warn-user)를 참조하세요.

#### 사용자 차단

에이전트가 호출할 수 있는 가장 중대한 도구입니다. 고정 기간으로 사용자를 차단하며, 선택적으로 그림자 차단(shadow ban), 선택적으로 IP 차단, 선택적으로 사용자의 모든 댓글 삭제를 수행합니다. 두 가지 파괴적 옵션(IP, 모두 삭제)은 편집 폼의 **Ban options** 섹션에서 옵트인할 때까지 모델에게 완전히 숨겨집니다. 모델이 매개변수를 환각하더라도, 플랫폼은 당신이 옵트인하지 않은 값은 거부합니다. EU 지역에서는 모든 차단이 사람의 승인을 필요로 합니다(자세한 내용은 [EU DSA 제17조 준수](#eu-dsa-compliance) 참조). 어디서나 승인을 요구하는 것을 강력히 고려하세요. 전체 페이지는 [Ban user](#tool-ban-user)를 참조하세요.

### 차단 도구의 하위 옵션

Ban 도구는 두 가지 파괴적 옵션인 `delete-all-comments` 및 `ban-by-IP` 를 제공합니다. 이 옵션들은 편집 폼의 **Ban options** 섹션을 통해 옵트인하기 전까지 모델에서 완전히 숨겨집니다. 모델이 매개변수를 환각하더라도, 플랫폼은 당신이 옵트인하지 않은 값을 거부합니다. 자세한 내용은 [Ban user](#tool-ban-user)를 참조하세요.
