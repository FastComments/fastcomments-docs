에이전트의 **도구**는 에이전트가 취할 수 있는 동작입니다. 에이전트 편집 양식에는 이 에이전트가 사용할 수 있도록 선택하는 **허용된 도구 호출** 섹션과, 적용되기 전에 사람이 승인해야 하는 동작을 선택하는 **승인** 섹션이 있습니다.

모든 도구에는 세 가지 수준이 있습니다:

- **Disallowed** - 에이전트는 이를 볼 수도 사용하지도 못합니다.
- **Allowed, no approval** - 에이전트가 직접 사용합니다. 실행 기록에 기록됩니다.
- **Allowed, with approval** - 에이전트의 호출이 사람의 검토를 위해 대기열에 들어가며 사람이 승인해야만 실행됩니다.

허용되지 않은 도구는 조용합니다: 에이전트는 이를 요청할 수 없고 플랫폼이 이를 즉시 거부합니다. 승인이 필요한 도구는 항상 [승인 수신함](#approval-workflow)을 거칩니다.

### 모든 작업에 대한 감사 추적

에이전트가 수행하는 모든 동작은 짧은 정당성(이유를 설명하는 1-2문장)과 신뢰도 점수(0.0-1.0)와 함께 기록됩니다. 둘 다 [실행 상세 보기](#run-detail-view)와 각 [승인](#approval-workflow)에 표시됩니다. 메모 검색은 유일한 읽기 전용 예외입니다: 이는 동작으로 기록되지 않으며 허용 목록과 상관없이 항상 사용 가능합니다.

### 도구 참조

#### 댓글 게시

에이전트가 자신의 이름으로 댓글을 게시할 수 있게 합니다. 댓글은 에이전트의 표시 이름 아래에 공개적으로 보여집니다. 인사 및 요약 에이전트에서 사용됩니다. 되돌릴 수 있음 — 모든 중재자가 문제 있는 댓글을 삭제할 수 있습니다. 보통 승인 없이 허용됩니다; 커뮤니티가 모든 공개 메시지를 사람의 검토를 거치게 해야 한다면 승인을 요구하세요.

#### 댓글에 대한 투표

에이전트가 댓글에 대해 찬반 투표를 하게 합니다. 해당 투표는 다른 투표와 마찬가지로 댓글의 투표 합계에 반영됩니다. 대부분의 커뮤니티는 봇의 투표를 선호하지 않습니다; 어떤 스타터 템플릿에도 활성화되어 있지 않습니다. 허용하는 경우 투표는 되돌릴 수 있습니다.

#### 댓글 고정 / 고정 해제

에이전트가 댓글을 페이지 상단에 고정하거나 이미 고정된 댓글의 고정을 해제할 수 있게 합니다. 플랫폼은 스레드당 하나의 고정만 허용하는 규칙을 강제하지 않으므로, 고정하는 에이전트는 먼저 이전에 고정된 댓글의 고정을 해제하도록 지시해야 합니다. Top Comment Pinner 템플릿에서 사용됩니다. 되돌릴 수 있으며 보통 승인 없이 허용됩니다.

#### 댓글 잠금 / 잠금 해제

에이전트가 댓글 아래의 추가 답글을 차단하거나 답글을 복원할 수 있게 합니다. 잠긴 댓글은 계속 표시됩니다. 과열된 스레드의 쿨다운에 유용하며, 나중에 잠금 해제를 예약하면 좋습니다. 되돌릴 수 있지만 커뮤니티에 노출됩니다; 중요한 커뮤니티에서는 승인을 요구하는 것을 고려하세요.

#### 스팸 표시 / 표시 해제

에이전트가 댓글을 스팸으로 표시(읽는 이로부터 숨기고 스팸 분류기에 전달)하거나 그 플래그를 해제할 수 있게 합니다. 모든 중재 에이전트의 기본 도구입니다. 되돌릴 수 있습니다. 에이전트에 대한 신뢰를 쌓는 첫 몇 주 동안은 승인 뒤에 두는 것을 강력히 권장합니다.

#### 댓글 승인 / 승인 취소

에이전트가 보류된 댓글을 독자에게 보이게 하거나 이미 보이는 댓글을 숨길 수 있게 합니다. 새 댓글을 중재자 검토를 위해 보류하는 테넌트에서 가장 유용합니다. 이미 보이는 댓글을 승인 취소하는 것은 큰 영향이 있으므로, 승인 뒤에 두는 것을 고려하세요.

#### 댓글 검토 표시

큐 상태 도구: 댓글을 "중재자(또는 에이전트)가 이 댓글을 확인했습니다."로 표시합니다. 가시성은 변경하지 않습니다. 영향도가 낮아 거의 승인을 요구하지 않습니다.

#### 배지 수여

에이전트가 테넌트의 배지 구성에서 사용자에게 배지를 줄 수 있게 합니다. 중재자가 되돌릴 수 있습니다. 거의 승인을 요구하지 않습니다. 에이전트는 배지 ID를 알아야 하므로 관련 ID를 [커뮤니티 가이드라인](#community-guidelines)이나 [초기 프롬프트](#personality-prompt)에 포함시키세요.

#### 이메일 전송

에이전트가 `noreply@fastcomments.com`에서 선택한 주소로 일반 텍스트 이메일을 보낼 수 있게 합니다. 절약해서 사용하세요 — 이메일은 마찰이 가장 큰 도구이며 잘못된 이메일은 되돌리기 어렵습니다. 강력히 승인을 뒤에 두는 것을 고려하고, 에이전트가 이메일을 보낼 인박스를 소유한 사람에게 승인 이메일을 라우팅하세요.

#### 에이전트 메모 저장 / 검색

트리거가 발생한 사용자에 대한 공유 노트 풀을 읽고 쓰는 짝을 이루는 두 도구입니다. 메모는 테넌트의 모든 에이전트 간에 공유되므로, 분류(트리아지) 에이전트의 노트가 중재 에이전트의 결정에 영향을 줍니다. 검색은 읽기 전용이며 항상 사용 가능합니다; 저장은 거의 승인을 요구하지 않습니다. 전체 설계는 [에이전트 메모리 시스템](#agent-memory-system)을 참조하세요.

#### 사용자 경고

특정 댓글에 대해 사용자에게 비공개 DM 경고를 보내고 경고를 에이전트 메모리에 원자적으로 기록합니다. 플랫폼의 에스컬레이션 정책은 이 도구를 중심으로 설계되어 있습니다 — 먼저 경고하고, 사용자가 재범하면 차단합니다. `ban_user`보다 덜 자주 승인을 요구하지만, 에이전트 운영 초기 몇 주 동안에는 승인을 고려하세요. 전체 내용은 [사용자 경고](#tool-warn-user)를 참조하세요.

#### 사용자 차단

에이전트가 호출할 수 있는 가장 중대한 도구입니다. 고정 기간으로 사용자를 차단하며, 선택적으로 섀도우 밴으로도 설정할 수 있고, 선택적으로 IP도 차단하며, 선택적으로 사용자의 모든 댓글을 삭제할 수 있습니다. 두 가지 파괴적 옵션(IP, delete-all)은 편집 양식의 **차단 옵션**에서 추가 옵트인을 통해 활성화하지 않으면 모델에게 완전히 숨겨집니다. 모델이 매개변수를 환각하더라도, 플랫폼은 당신이 옵트인하지 않은 값을 거부합니다. EU 지역에서는 모든 차단이 사람의 승인을 필요로 합니다(자세한 내용은 [EU DSA 제17조 준수](#eu-dsa-compliance) 참조). 어디서나 승인을 요구하는 것을 강력히 고려하세요. 전체 내용은 [사용자 차단](#tool-ban-user)을 참조하세요.

### 차단 도구 하위 옵션

차단 도구는 두 가지 파괴적 옵션인 delete-all-comments 및 ban-by-IP를 노출합니다. 이 옵션들은 편집 양식의 **차단 옵션** 섹션에서 옵트인할 때까지 모델에게 완전히 숨겨집니다. 모델이 매개변수를 환각하더라도 플랫폼은 당신이 옵트인하지 않은 값을 거부합니다. 자세한 내용은 [사용자 차단](#tool-ban-user)을 참조하세요.
