Секція **Context** у формі редагування контролює, скільки інформації агент отримує під час кожного запуску. Більше контексту дає кращі рішення, але збільшує кількість токенів за запуск, тому варто надавати лише те, що агенту дійсно потрібно.

### What's always included

Навіть якщо всі прапорці знято, контекстне повідомлення агента включає:

- Тип **trigger event** (наприклад, `COMMENT_ADD`, `COMMENT_FLAG_THRESHOLD`).
- URL сторінки та ID URL (коли відомо).
- **Коментар**, що спричинив запуск, якщо такий є — ID, ID автора, відображуване ім'я автора, текст коментаря, кількість голосів, кількість флагів, позначки spam/approved/reviewed, ID батьківського коментаря. Електронна пошта автора **ніколи** не відправляється провайдеру LLM (мінімізація PII).
- **Попередній текст коментаря** для тригерів `COMMENT_EDIT` (щоб агент міг порівняти до/після).
- **Напрямок голосу** для тригерів `COMMENT_VOTE_THRESHOLD`.
- **ID користувача**, що спричинив тригер, та **ID бейджу** (для тригерів бейджів модератора).
- Каталог **бейджів** вашого тенанту (назва, відображувальна мітка, опис), коли агент може присвоювати бейджі, щоб агент міг обрати підходящий без того, щоб ви виписували бейджі в промпті.

Увесь ненадійний текст — тіла коментарів, імена авторів, заголовки сторінок, сам документ з правилами — **огороджується** в контекстному повідомленні маркерами на кшталт `<<<COMMENT_TEXT>>> ... <<<END>>>`. Системний промпт платформи наказує моделі ніколи не виконувати інструкцій всередині цих огорож. Це захист від інʼєкцій у промпт платформи; вам не потрібно повторювати це у своєму промпті.

### The three checkboxes

#### Include parent comment and prior replies in the same thread

Додає:
- **Батьківський коментар** — ID, автор, текст.
- **Суперницькі відповіді** — попередні відповіді на той самий батько в тій самій гілці.

Корисно для: будь-якого агента, який відповідає на коментар у контексті (привітальні боти, підсумувачі гілок, модератори, що читають відповіді в розмовах).

Вартість: від мала до середня. Обмежується кількістю суперників у конкретній гілці.

#### Include commenter's trust factor, account age, ban history, and recent comments

Додає блок **AUTHOR_HISTORY**:

- **Вік акаунта у днях** від моменту реєстрації.
- **Trust factor (0-100)** — скор FastComments, що узагальнює, наскільки довіряють користувачу на цьому сайті. Див. сторінку [Spam Detection](/guide-moderation.html#spam-detection) у посібнику з модерації.
- **Кількість попередніх банів.**
- **Загальна кількість коментарів** на цьому сайті.
- **Кількість дубльованого контенту** — якщо користувач нещодавно публікував ідентичні тексти (сигнал проти спаму).
- **Сигнал крос-акаунтів з тієї самої IP** — кількість коментарів з тієї самої IP під іншими акаунтами (сигнал про фіктивні акаунти). Сам хеш IP ніколи не відправляється в LLM.
- **Нещодавні коментарі** — до 5 останніх коментарів користувача, кожен обрізано до 300 символів, огороджені як ненадійний текст.

Корисно для: будь-якого модераційного агента. Без цього модель може блокувати нові акаунти та давніх добросовісних користувачів через однакову поведінку.

Вартість: середня. Нещодавні коментарі додають найбільше токенів.

#### Include page title, subtitle, description, and meta tags

Додає блок **PAGE_CONTEXT** — заголовок, підзаголовок, опис та будь-які мета-теги, які FastComments зібрав для сторінки.

Корисно для: привітальних ботів і підсумувачів гілок, де знання теми сторінки значно покращує якість виходу.

Вартість: мала.

### Community guidelines

Четверте поле, **Community guidelines**, — це поле політики у вільному тексті, яке включається в контекстне повідомлення з роллю користувача при кожному запуску, огороджене як ненадійний текст так само, як тіла коментарів та інший контент, наданий користувачем. Агент читає його як текст політики, але платформа не розглядає його як системну інструкцію. Див. [Community Guidelines](#community-guidelines), щоб дізнатися, що туди помістити.

### Adding context selectively

Ці прапорці застосовуються для кожного агента окремо, а не глобально. Поширений підхід:

- Привітальний бот: page context **включено**, thread context **вимкнено**, user history **вимкнено**.
- Модератор: thread context **вимкнено**, user history **включено**, page context **вимкнено**.
- Підсумувач гілки: thread context **включено**, page context **включено**, user history **вимкнено**.

Надавайте мінімальний контекст, необхідний агенту для коректності викликів, які він реально робить — зайвий контекст коштує токенів при кожному запуску, навіть коли агент його не використовує.
