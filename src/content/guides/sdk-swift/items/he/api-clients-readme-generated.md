The FastComments SDK מספק שלושה לקוחות API:

### PublicAPI - שיטות בטוחות בצד הלקוח

The `PublicAPI` מכיל שיטות הבטוחות לקריאה מתוך קוד בצד הלקוח (iOS/macOS apps). שיטות אלה:
- אינם דורשים מפתח API
- יכולים להשתמש ב-SSO tokens לאימות
- מוגבלים בקצב לפי משתמש/מכשיר
- מתאימות ליישומים המופנים למשתמש הקצה

**דוגמה לשימוש**: שליפת ויצירת תגובות באפליקציית iOS שלך

### DefaultAPI - שיטות בצד השרת

The `DefaultAPI` מכיל שיטות מאומתות שהן דורשות מפתח API. שיטות אלה:
- דורשות את מפתח ה-FastComments API שלך
- יש לקרוא להן רק מתוך קוד בצד השרת
- מספקות גישה מלאה לנתוני FastComments שלך
- מוגבלות בקצב לפי tenant

**דוגמה לשימוש**: פעולות מנהל, ייצוא נתונים בכמות גדולה, ניהול משתמשים

### ModerationAPI - שיטות ללוח הבקרה של המודרטורים

The `ModerationAPI` מכיל שיטות שמפעילות את לוח הבקרה של המודרטורים. שיטות אלה כוללות:
- **בקרה על תגובות** - רישום, ספירה, חיפוש, שליפת לוגים, וייצוא תגובות
- **פעולות בקרה** - הסרה/שחזור תגובות, סימון (flag), קביעת סטטוס בדיקה/ספאם/אישור, ניהול הצבעות, ופתיחה/סגירת שרשורים
- **חסימות** - לחסום משתמש מתגובה, לבטל חסימות, לשלוף סיכומי טרם חסימה, לבדוק סטטוס והעדפות חסימה, ולקרוא את ספירות המשתמשים החסומים
- **תגים ואמון** - להעניק/להסיר תגיות (badges), לרשום תגיות ידניות, לקבל/לקבוע את גורם האמון של משתמש, ולקרוא את הפרופיל הפנימי של משתמש

Every `ModerationAPI` method accepts an `sso` parameter so moderators can be authenticated via SSO.

**דוגמה לשימוש**: בניית חוויית בקרה עבור המודרטורים של הקהילה שלך

**חשוב**: לעולם לא לחשוף את מפתח ה-API שלך בקוד בצד הלקוח. יש להשתמש במפתחות API רק בצד השרת.
