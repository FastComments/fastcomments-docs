FastComments SDK предоставляет три клиента API:

### PublicAPI - Client-Safe Methods

The `PublicAPI` contains methods that are safe to call from client-side code (iOS/macOS apps). These methods:
- Не требуют API key
- Могут использовать SSO-токены для аутентификации
- Ограничены по частоте запросов для каждого пользователя/устройства
- Подходят для приложений, ориентированных на конечного пользователя

**Пример использования**: Получение и создание комментариев в вашем iOS-приложении

### DefaultAPI - Server-Side Methods

The `DefaultAPI` contains authenticated methods that require an API key. These methods:
- Требуют ваш FastComments API key
- ДОЛЖНЫ ВЫЗЫВАТЬСЯ ТОЛЬКО из серверного кода
- Обеспечивают полный доступ к вашим данным FastComments
- Ограничены по частоте запросов для каждого tenant

**Пример использования**: Административные операции, массовый экспорт данных, управление пользователями

### ModerationAPI - Moderator Dashboard Methods

The `ModerationAPI` contains methods that power the moderator dashboard. These methods cover:
- **Модерация комментариев** - перечисление, подсчёт, поиск, получение логов и экспорт комментариев
- **Действия модерации** - удаление/восстановление комментариев, пометка, установка статусов review/spam/approval, управление голосами и повторное открытие/закрытие веток
- **Баны** - забанить пользователя в комментарии, отменить баны, получить предварительные сводки перед баном, проверить статус бана и предпочтения, и прочитать количество забаненных пользователей
- **Значки & доверие** - присваивать/удалять значки, перечислять вручную назначенные значки, получать/устанавливать фактор доверия пользователя, и читать внутренний профиль пользователя

Каждый метод `ModerationAPI` принимает параметр `sso`, чтобы модераторы могли аутентифицироваться через SSO.

**Пример использования**: Создание интерфейса модерации для модераторов вашего сообщества

**ВАЖНО**: Никогда не раскрывайте ваш API key в клиентском коде. API keys должны использоваться только на сервере.
