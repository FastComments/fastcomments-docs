Το FastComments SDK παρέχει τρεις πελάτες API:

### PublicAPI - Μέθοδοι ασφαλείς για client-side

Το `PublicAPI` περιέχει μεθόδους που είναι ασφαλές να καλούνται από κώδικα client-side (iOS/macOS apps). Αυτές οι μέθοδοι:
- Δεν απαιτούν API key
- Μπορούν να χρησιμοποιήσουν SSO tokens για αυθεντικοποίηση
- Έχουν περιορισμό ρυθμού ανά χρήστη/συσκευή
- Είναι κατάλληλες για εφαρμογές που απευθύνονται σε τελικούς χρήστες

**Παράδειγμα χρήσης**: Ανάκτηση και δημιουργία σχολίων στην εφαρμογή iOS σας

### DefaultAPI - Μέθοδοι server-side

Το `DefaultAPI` περιέχει αυθεντικοποιημένες μεθόδους που απαιτούν API key. Αυτές οι μέθοδοι:
- Απαιτούν το FastComments API key σας
- Πρέπει ΝΑ καλούνται ΜΟΝΟ από κώδικα server-side
- Παρέχουν πλήρη πρόσβαση στα δεδομένα FastComments σας
- Έχουν περιορισμό ρυθμού ανά tenant

**Παράδειγμα χρήσης**: Διοικητικές λειτουργίες, εξαγωγή μαζικών δεδομένων, διαχείριση χρηστών

### ModerationAPI - Μέθοδοι Πίνακα Ελέγχου Συντονιστή

Το `ModerationAPI` περιέχει μεθόδους που τροφοδοτούν τον πίνακα ελέγχου συντονιστή. Αυτές οι μέθοδοι καλύπτουν:
- **Διαχείριση σχολίων** - λίστα, καταμέτρηση, αναζήτηση, ανάκτηση αρχείων καταγραφής και εξαγωγή σχολίων
- **Ενέργειες διαχείρισης** - αφαίρεση/επαναφορά σχολίων, σήμανση, ρύθμιση κατάστασης επανεξέτασης/spam/έγκρισης, διαχείριση ψήφων, και επανάνοιγμα/κλείσιμο νημάτων
- **Αποκλεισμοί** - αποκλεισμός χρήστη από σχόλιο, άρση αποκλεισμών, ανάκτηση συνοπτικών στοιχείων πριν τον αποκλεισμό, έλεγχος κατάστασης αποκλεισμού και προτιμήσεων, και ανάγνωση αριθμών αποκλεισμένων χρηστών
- **Διακριτικά & εμπιστοσύνη** - απονομή/αφαίρεση διακριτικών, λίστα χειροκίνητων διακριτικών, λήψη/ρύθμιση του δείκτη εμπιστοσύνης ενός χρήστη, και ανάγνωση του εσωτερικού προφίλ ενός χρήστη

Κάθε μέθοδος του `ModerationAPI` δέχεται παράμετρο `sso` ώστε οι συντονιστές να μπορούν να αυθεντικοποιηθούν μέσω SSO.

**Παράδειγμα χρήσης**: Δημιουργία εμπειρίας moderation για τους συντονιστές της κοινότητάς σας

**ΣΗΜΑΝΤΙΚΟ**: Ποτέ μην αποκαλύπτετε το API key σας σε κώδικα client-side. Τα API keys θα πρέπει να χρησιμοποιούνται μόνο server-side.
