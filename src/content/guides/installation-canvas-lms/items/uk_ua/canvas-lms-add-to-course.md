#### Як коментарі відображаються у ваших курсах

Після того як інтеграція LTI увімкнена і External App встановлено, FastComments працює автоматично на основі розміщень, які ви налаштували:

#### Assignment View

Якщо увімкнено розміщення **Assignment View**, коментарі автоматично відображаються під кожним завданням у курсі. Студенти та викладачі бачать структурувану гілку коментарів під час перегляду завдання — додаткових налаштувань для окремого завдання не потрібно.

Кожне завдання має власну окрему гілку коментарів.

#### Rich Content Editor Button

Якщо увімкнено розміщення **Editor Button**, викладачі можуть вбудувати FastComments у будь-який вміст, який використовує Rich Content Editor:

1. Редагуйте **Page**, **Quiz** або **Announcement**.
2. У панелі інструментів Rich Content Editor натисніть кнопку **FastComments**.
3. FastComments автоматично вбудовується у вміст.
4. Збережіть сторінку.

Коли студенти переглядають сторінку, вбудований віджет FastComments завантажує гілку коментарів, унікальну для цієї сторінки.

#### Automatic SSO

У обох розміщеннях студенти автоматично входять через свій обліковий запис Canvas. Імена, електронні адреси та аватари синхронізуються під час запуску LTI, окремий вхід не потрібен.

#### Lock Down Public Access (Recommended)

За замовчуванням дані коментарів FastComments є загальнодоступними для читання. Будь-хто, хто зможе вгадати URL гілки або API endpoint, може переглянути її коментарі, навіть поза межами Canvas. Для дискусій у курсі майже завжди потрібно обмежити перегляд лише для зареєстрованих студентів.

Відкрийте свою <a href="https://fastcomments.com/auth/my-account/customize-widget" target="_blank">сторінку налаштувань віджета</a> та створіть правило з увімкненим **Require SSO To View Comments**, потім встановіть рівень безпеки на **Secure SSO**, щоб гілки могли завантажуватися лише через підписаний запуск LTI.

Див. [Protecting Comment Threads With Single-Sign-On](/guide-customizations-and-configuration.html#sso-require-to-view-comments) для повного покрокового керівництва, включно з тим, як обмежити правило для одного домену або сторінки.
