#### Как комментарии отображаются в ваших курсах

После включения интеграции LTI и установки External App, FastComments работает автоматически в соответствии с размещениями, которые вы настроили:

#### Assignment View

Если размещение **Assignment View** включено, комментарии автоматически появляются под каждым заданием в курсе. Студенты и преподаватели видят древовидную секцию комментариев при просмотре задания — дополнительная настройка для каждого задания не требуется.

Каждое задание получает свою отдельную ветку комментариев.

#### Rich Content Editor Button

Если размещение **Editor Button** включено, преподаватели могут встроить FastComments в любой контент, использующий Rich Content Editor:

1. Отредактируйте **Page**, **Quiz** или **Announcement**.
2. В панели инструментов Rich Content Editor нажмите кнопку **FastComments**.
3. FastComments автоматически встраивается в контент.
4. Сохраните страницу.

Когда студенты просматривают страницу, встроенный виджет FastComments загружается с веткой комментариев, уникальной для этой страницы.

#### Automatic SSO

В обоих размещениях студенты автоматически входят через свой аккаунт Canvas. Имена, адреса электронной почты и аватары синхронизируются через LTI-запуск; отдельный вход не требуется.

#### Lock Down Public Access (Recommended)

По умолчанию данные комментариев FastComments доступны для публичного чтения. Любой, кто сможет угадать URL темы или API-эндпоинт, сможет просмотреть её комментарии, даже вне Canvas. Для обсуждений в курсе вы почти наверняка захотите ограничить просмотр только зачисленными студентами.

Откройте вашу <a href="https://fastcomments.com/auth/my-account/customize-widget" target="_blank">страницу настройки виджета</a> и создайте правило с включенной опцией **Require SSO To View Comments**, затем установите уровень безопасности **Secure SSO**, чтобы темы могли загружаться только через подписанный LTI-запуск.

Смотрите [Защита тем комментариев с помощью Single-Sign-On](/guide-customizations-and-configuration.html#sso-require-to-view-comments) для полного пошагового руководства, включая то, как ограничить действие правила до одного домена или страницы.
