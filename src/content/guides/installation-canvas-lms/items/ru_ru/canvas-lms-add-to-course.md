#### Как комментарии отображаются в ваших курсах

После того как интеграция LTI включена и External App установлено, FastComments работает автоматически в соответствии с размещениями, которые вы настроили:

#### Assignment View

Если размещение **Assignment View** включено, комментарии автоматически отображаются под каждым заданием в курсе. Студенты и преподаватели видят ветвящийся раздел комментариев при просмотре задания — дополнительной настройки для каждого задания не требуется.

Каждое задание получает свою отдельную ветку комментариев.

#### Rich Content Editor Button

Если размещение **Editor Button** включено, преподаватели могут встроить FastComments в любой контент, который использует Rich Content Editor:

1. Отредактируйте **Page**, **Quiz**, или **Announcement**.
2. В панели инструментов Rich Content Editor нажмите кнопку **FastComments**.
3. FastComments автоматически встраивается в содержимое.
4. Сохраните страницу.

Когда студенты просматривают страницу, встроенный виджет FastComments загружается с веткой комментариев, уникальной для этой страницы.

#### Automatic SSO

В обоих размещениях студенты автоматически входят через свою учетную запись Canvas. Имена, адреса электронной почты и аватары синхронизируются через LTI launch, отдельный вход не требуется.

#### Ограничить публичный доступ (рекомендуется)

По умолчанию данные комментариев FastComments доступны для чтения публично. Любой, кто сможет угадать URL потока или API endpoint, сможет просматривать его комментарии, даже за пределами Canvas. Для дискуссий в курсе вам почти наверняка потребуется ограничить просмотр только для зачисленных студентов.

Откройте вашу <a href="https://fastcomments.com/auth/my-account/customize-widget" target="_blank">страницу настройки виджета</a> и создайте правило с включенной опцией **Require SSO To View Comments**, затем установите уровень безопасности на **Secure SSO**, чтобы потоки могли загружаться только через подписанный LTI launch.

См. [Защита потоков комментариев с помощью единого входа](/guide-customizations-and-configuration.html#sso-require-to-view-comments) для полного пошагового руководства, включая то, как ограничить действие правила одним доменом или страницей.
