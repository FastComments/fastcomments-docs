Webhooks 管理画面には各イベントタイプ (Create, Update, Delete) ごとに `Send Test Payload` ボタンがあります。

Create と Update のイベントはダミーの `WebhookComment` オブジェクトを送信し、Delete をテストする場合は ID のみを含むダミーのリクエストボディを送信します。

テストは 2 回の呼び出しを行い、"happy"（正しい API Key）と "sad"（無効な API key）のシナリオでレスポンスコードを検証します。

テストが無効な API key を送信した場合、テストを完全にパスさせるにはステータスコード 401 を返す必要があります。トークンの値を正しくチェックしないと、エラーになります。

これはリクエストを正しく認証していることを確認するためです。
