Στο διαχειριστικό Webhooks υπάρχουν `Send Test Payload` κουμπιά για κάθε τύπο γεγονότος (Create, Update, Delete). Τα γεγονότα Create και Update στέλνουν ένα δοκιμαστικό αντικείμενο WebhookComment, ενώ η δοκιμή του Delete θα στείλει ένα δοκιμαστικό σώμα αιτήματος που περιέχει μόνο ένα ID.

Η δοκιμή θα κάνει δύο κλήσεις για να επαληθεύσει τον κωδικό απόκρισης για τα σενάρια "happy" (correct API Key) και "sad" (invalid API key).

Όταν η δοκιμή στέλνει ένα μη έγκυρο API key θα πρέπει να επιστρέψετε έναν κωδικό κατάστασης 401 για να περάσει πλήρως η δοκιμή. Αν δεν ελέγξετε σωστά την τιμή του token, θα δείτε ένα σφάλμα.

Αυτό γίνεται για να διασφαλιστεί ότι αυθεντικοποιείτε σωστά το αίτημα.
