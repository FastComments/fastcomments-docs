В интерфейсе Webhooks admin есть кнопки `Send Test Payload` для каждого типа события (Create, Update, Delete). События Create и Update отправляют фиктивный объект WebhookComment, тогда как при тестировании Delete будет отправлено фиктивное тело запроса только с ID.

Тест выполнит два запроса, чтобы проверить код ответа для сценариев "happy" (correct API Key) и "sad" (invalid API key).

Когда тест отправляет неверный API key, вы должны возвращать код статуса 401, чтобы тест считался полностью пройденным. Если вы неверно проверяете значение token, вы увидите ошибку.

Это необходимо, чтобы убедиться, что вы правильно аутентифицируете запрос.
