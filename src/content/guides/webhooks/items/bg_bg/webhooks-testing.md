В Webhooks admin има бутони `Send Test Payload` за всеки тип събитие (Create, Update, Delete). Събитията Create и Update изпращат фиктивен обект `WebhookComment`, докато при тестване на Delete ще бъде изпратено фиктивно тяло на заявката само с ID.

Тестът ще направи две заявки, за да провери кода на отговора за "happy" (correct API Key) и "sad" (invalid API key) сценарии.

Когато тестът изпрати invalid API key, вие трябва да върнете статус код 401, за да премине тестът напълно. Ако не проверите правилно стойността на token-а, ще видите грешка.

Това е, за да се уверите, че правилно удостоверявате заявката.
