У Webhooks admin є кнопки `Send Test Payload` для кожного типу події (Create, Update, Delete). Події Create і Update надсилають тестовий об'єкт WebhookComment, тоді як при тестуванні Delete буде надіслано тестове тіло запиту, що містить лише ID.

Тест виконуватиме два виклики, щоб перевірити код відповіді для сценаріїв "happy" (correct API Key) і "sad" (invalid API key).

Коли тест надсилає недійсний API key, ви повинні повертати код стану 401, щоб тест пройшов повністю. Якщо ви некоректно перевірите значення token, ви побачите помилку.

Це зроблено для того, щоб переконатися, що ви правильно автентифікуєте запит.
