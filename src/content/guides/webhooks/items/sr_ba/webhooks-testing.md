У Webhooks администрацији постоје дугмад `Send Test Payload` за сваки тип догађаја (Create, Update, Delete).

Догађаји Create и Update шаљу пробни (dummy) објекат WebhookComment, док ће тестирање Delete послати пробно тијело захтјева које садржи само ID.

Тест ће направити два позива да провјери статусни код за „успјешни“ (ispravan API Key) и „неуспјешни“ (nevažeći API Key) сценарио.

Када тест пошаље неважећи API Key, требало би да вратите статусни код 401 да би тест у потпуности прошао. Ако правилно не провјерите вриједност token-а, видећете грешку.

Ово је да би се осигурало да правилно аутентификујете захтјев.
