בממשק ניהול Webhooks יש כפתורי `Send Test Payload` עבור כל סוג אירוע (Create, Update, Delete). אירועי Create ו-Update שולחים אובייקט דמה מסוג `WebhookComment`, בעוד שבדיקת Delete תשלח גוף בקשה דמה עם רק ID.

המבחן יבצע שתי קריאות כדי לאמת את קוד התגובה עבור התרחישים "happy" (correct API Key) ו-"sad" (invalid API key).

כאשר המבחן שולח API key לא תקף, עליכם להחזיר קוד סטטוס של 401 כדי שהמבחן יעבור במלואו. אם אינכם בודקים כראוי את ערך ה-token, תראו שגיאה.

זה נועד להבטיח שאתם מאמתים כהלכה את הבקשה.
