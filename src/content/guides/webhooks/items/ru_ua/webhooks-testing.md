В Webhooks admin есть `Send Test Payload` кнопки для каждого типа события (Create, Update, Delete). События Create и Update отправляют фиктивный объект WebhookComment, а при тестировании Delete будет отправлено фиктивное тело запроса только с ID.

Тест выполнит два вызова, чтобы проверить код ответа для «успешного» (корректный API Key) и «ошибочного» (недействительный API Key) сценариев.

Когда тест отправляет недействительный API Key, вы должны возвращать код состояния 401, чтобы тест прошёл полностью. Если вы неправильно проверите значение токена, вы увидите ошибку.

Это необходимо, чтобы убедиться, что вы правильно аутентифицируете запрос.
