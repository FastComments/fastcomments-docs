一旦管理员将 FastComments 注册为 LTI 1.3 Advantage 工具并批准机构策略，教师即可通过标准的 Blackboard 放置点将其添加到课程中。Ultra 课程视图和原始课程视图的具体步骤不同，因此下面对两者都进行了说明。

#### Ultra 课程视图

Ultra 课程视图自 2026 年起在 Blackboard Learn SaaS 中为默认设置。

1. 打开课程并转到 **课程内容** 页面。
2. 将鼠标悬停或点击大纲中希望放置评论线程的位置，然后点击紫色的 **+**（添加内容）按钮。
3. 选择 **内容市场**。内容市场面板会列出机构已批准的所有 LTI 工具和 Building Block 放置项。
4. 找到 **FastComments** 磁贴并点击它。Blackboard 会在打开 **+** 菜单的位置创建一个内容项。
5. 对于个人默认将 **对学生隐藏** 关闭的教师，项目默认在大纲中作为“对学生可见”条目落位。如果你的默认是 **隐藏**，则项目会以隐藏状态创建，准备就绪后可在项目行上切换可见性选择器。
6. 要重命名该项目，请在大纲中点击标题并输入新标签。学生在大纲中看到的标题独立于 FastComments 线程标识符，因此随时重命名是安全的。

如果你看不到 **内容市场** 选项，说明你所在机构已将该放置隐藏。你也可以通过同一 **+** 菜单下的 **更多工具** 中的 **LTI 工具** 组访问相同的选择器。

#### 原始课程视图

原始课程视图在 Learn SaaS 中仍受支持，并且在 Q4 2024 CU 发布线上的自托管 Learn 9.1 站点中仍然是主要体验。

1. 打开课程并进入一个 **内容区**（例如课程菜单中的默认 **信息** 或 **内容** 区域）。
2. 在页面右上角的切换按钮将 **编辑模式** 打开。
3. 在操作栏中点击 **创建内容**。
4. 在 **学习工具** 子菜单下，点击 **FastComments**。学习工具子菜单由管理员注册工具后填充 LTI 1.3 工具放置项。如果你看不到它，请参见下面的注意事项部分。
5. 在 **创建 FastComments** 表单中设置：
   - **名称**：学生在内容区看到的标签。
   - **描述**：可选文本，显示在嵌入线程上方。
   - **允许用户查看此内容**：是/否 可用性切换。
   - **跟踪查看次数**：如果你想要 Blackboard 的单项查看统计，请启用此项。FastComments 自身会独立运行分析。
   - **日期和时间限制**：可选的 **显示开始** / **显示截止** 窗口。
6. 提交。该工具作为可点击项目出现在内容区中。

#### 在项目或文档内嵌入

在两种课程视图中，教师都可以通过内容编辑器的 LTI Advantage 按钮将 FastComments 内联嵌入到项目、文档或任何富文本字段的主体中。

Ultra 课程视图：

1. 创建或编辑一个 **文档**。
2. 在文档正文中点击 **添加内容**，在希望线程出现的位置插入。
3. 在编辑器工具栏中，打开 **插入内容** 菜单并点击 **内容市场**（LTI Advantage / 深度链接的入口点）。
4. 选择 **FastComments**。FastComments 返回一个深度链接负载，Blackboard 会在光标位置的文档正文中插入一个嵌入块。
5. 保存文档。学生在滚动经过时会在内联处看到渲染的线程。

原始课程视图：

1. 编辑任何具有富文本正文的项目。
2. 在内容编辑器工具栏中，点击 **添加内容** 的加号图标并选择 **内容市场**（在较旧的 Q4 2024 CU 中标为 **从外部工具添加内容**）。
3. 选择 **FastComments**。编辑器会插入一个引用深度链接资源的占位块。
4. 提交该项目。

每个深度链接嵌入都会产生它自己的 FastComments 线程，因此在同一项目中嵌入两个 FastComments 块会有两个独立的评论流。

#### 可见性、发布条件和小组限制

FastComments 内容项在其访问控制规则上表现得像任何其他 Blackboard 内容项。

- Ultra：在行上点击可见性选择器（**对学生可见**、**对学生隐藏**、**条件可用性**）。条件可用性支持日期/时间窗口、基于成绩册项目的绩效规则以及基于课程小组的成员规则。
- 原始：打开项目的上下文菜单并选择 **自适应发布** 或 **自适应发布：高级** 以按日期、成员、成绩或审核状态对工具进行限制。使用项目上的 **设置小组可用性** 将其限制为特定课程小组。

FastComments 会遵从 Blackboard 的任何门控决策。如果 Blackboard 将项目对某位学生隐藏，则该学生不会触发 LTI 启动，也不会出现在版主视图中。

#### 成绩册行为

FastComments 不会通过 LTI Advantage 的作业与成绩服务报告成绩。不会为 FastComments 内容项自动创建成绩列。

如果你的 Blackboard 租户被配置为无论评分元数据如何都为每个新内容项自动创建成绩册列，则仍会出现一个空列。要隐藏它：

- Ultra：打开 **成绩册**，点击列标题，选择 **编辑**，并关闭 **对学生显示** 以及 **计入计算**。如果你们的机构允许对未评分项目删除列，也可以使用 **删除**。
- 原始：打开 **成绩中心**，点击该列的下拉箭头，选择 **对用户隐藏（开/关）**，并且在 **列组织** 下可选择 **在教师视图中隐藏**（可选）。

#### 学生看到的内容

当学生打开 FastComments 项目或滚动到嵌入块时：

1. Blackboard 向 FastComments 发起 LTI 1.3 消息。学生通过其 Blackboard 身份（姓名、电子邮件、头像、角色）使用 SSO 登录，无需看到登录表单。
2. 评论线程在 iframe 中渲染。根据在 FastComments 中配置的评论小部件设置，可用线程化、回复、提及和反应等功能。
3. 他们的评论会归属于其 Blackboard 帐户。如果学生随后在 Blackboard 中编辑了姓名或照片，下一次启动会更新 FastComments 配置文件。

从 Blackboard 到 FastComments 的角色映射：

- **系统管理员** 和 **课程构建者** 映射为 FastComments 的 **admin**。
- **教师** 和 **助教** 映射为 FastComments 的 **moderator**。
- **学生**、**访客** 和 **观察者** 映射为 FastComments 的 **commenter**。

版主在每条评论的内联处都会看到管理控制（置顶、隐藏、封禁、删除）。

#### 线程作用域

FastComments 按 **（Blackboard 主机、课程 ID、资源链接 ID）** 对每个线程进行作用域划分。同一课程中的两个 FastComments 项目会产生两个线程。相同项目在两个课程壳之间复制（例如通过课程复制）会产生两个线程，因为 Blackboard 在复制过程中会发放新的资源链接 ID。要在课程复制间保留共享线程，请在复制之前使用深度链接并在 FastComments 中配置显式线程 URN。

#### Blackboard 特有注意事项

**原始视图的“创建内容”菜单（或 Ultra 的内容市场）中缺少 FastComments 磁贴。** 管理员批准了该工具但留下了阻止相关放置的机构策略。转到 **管理员面板** > **集成** > **LTI 工具提供者**，编辑 FastComments 条目，并确认已启用 **课程内容工具**（原始）和 **课程内容工具 - 允许学生** / **深度链接内容工具**（Ultra）放置。保存并刷新课程页面。

**启动时出现“该工具未为此上下文配置”或“该工具未部署”错误。** 在动态注册期间注册的部署范围与课程所属的机构上下文不匹配。在 Blackboard 的工具提供者条目中，验证 **部署 ID** 是否与 FastComments 在此租户的 LTI 1.3 配置页面上显示的值相匹配。如果不一致，删除该放置并从新的注册 URL 重新运行动态注册。

**Iframe 高度看起来固定或内容被截断。** 一些 Blackboard 租户随附严格的内容安全策略（CSP），会阻止默认的 LTI iframe 大小调整 postMessage。FastComments 发出 Canvas 风格的 `lti.frameResize` 消息以及 IMS 规范形式的 `org.imsglobal.lti.frameResize` 消息以最大化兼容性，但租户层面的 CSP 覆盖会阻止父页面的监听器。请管理员确认 `*.fastcomments.com` 已列入 LTI 工具允许列表，并且没有自定义的 CSP 头在剥离 postMessage 事件。确认后，大小调整即可正常工作，无需进一步配置。

**课程复制会导致线程重复。** Blackboard 课程复制会为 LTI 放置发放新的资源链接 ID，因此复制的课程会以空线程开始。这是预期行为。如果你需要复制的课程继承原始线程，请在复制前使用具有显式线程 URN 的深度链接，或联系 FastComments 支持进行批量线程 ID 重映射。

**学生在启动时看到 Blackboard 的通用错误。** 原因可能是缺失或过期的 `email` 声明。确认机构为 FastComments 的机构策略在 **要发送的用户字段** 下启用了 **角色**、**姓名** 和 **电子邮件地址**。保存后在新的浏览器会话中重新启动。
