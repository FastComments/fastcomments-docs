Для розробників, яких ви можливо не хочете робити `Administrators`, розгляньте створення користувача `Administrator`
з наступними дозволами:

1. Адміністратор аналітики
2. Адміністратор кастомізацій
3. Адміністратор управління даними
4. Адміністратор модерації коментарів
5. Адміністратор API/SSO

Цей набір дозволів надасть розробнику все, що потрібно для налаштування FastComments, а також видимість у системі, необхідну для перевірки її роботи.

Обґрунтування цих дозволів таке:

1. **Адміністратор аналітики**: Це можна використовувати для налагодження використання FastComments.
2. **Адміністратор кастомізацій**: Це буде потрібно, щоб застосувати власні стилі до віджета коментарів.
3. **Адміністратор управління даними**: Це буде потрібно для керування імпортами та експортами, а також для налаштування вебхуків.
4. **Адміністратор модерації коментарів**: Це буде потрібно, щоб бачити дані коментарів, принаймні під час налаштування.
5. **Адміністратор API/SSO**: Це дозволить їм отримувати API keys безпосередньо з нашої платформи. Ми вважаємо це більш безпечним, ніж коли `Administrator` копіює їх для них та надсилає API Secret електронною поштою, що може бути не дуже безпечно.
