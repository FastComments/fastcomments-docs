Για προγραμματιστές που ίσως να μην θέλετε να είναι `Administrators`, σκεφτείτε να δημιουργήσετε έναν χρήστη `Administrator`
με τα ακόλουθα δικαιώματα:

1. Διαχειριστής Αναλύσεων
2. Διαχειριστής Προσαρμογών
3. Διαχειριστής Διαχείρισης Δεδομένων
4. Διαχειριστής Μετριασμού Σχολίων
5. Διαχειριστής API/SSO

Αυτό το σύνολο δικαιωμάτων θα δώσει σε έναν προγραμματιστή όλα όσα χρειάζεται για να ρυθμίσει το FastComments καθώς και
την ορατότητα στο σύστημα που απαιτείται για να διασφαλιστεί ότι λειτουργεί.

Η αιτιολόγηση για αυτά τα δικαιώματα είναι η εξής:

1. **Διαχειριστής Αναλύσεων**: Αυτό μπορεί να χρησιμοποιηθεί για τον εντοπισμό σφαλμάτων στη χρήση του FastComments.
2. **Διαχειριστής Προσαρμογών**: Αυτό θα χρειαστεί για να εφαρμοστεί προσαρμοσμένο στυλ στο widget σχολίων.
3. **Διαχειριστής Διαχείρισης Δεδομένων**: Αυτό θα χρειαστεί για τη διαχείριση εισαγωγών και εξαγωγών, καθώς και για τη ρύθμιση webhooks.
4. **Διαχειριστής Μετριασμού Σχολίων**: Αυτό θα χρειαστεί για να βλέπει δεδομένα σχολίων, τουλάχιστον κατά τη ρύθμιση.
5. **Διαχειριστής API/SSO**: Αυτό θα τους επιτρέψει να ανακτούν τα API keys απευθείας από την πλατφόρμα μας. Το θεωρούμε
πιο ασφαλές από το να το αντιγράφει ένας `Administrator` για αυτούς και να στέλνει το API Secret μέσω ηλεκτρονικού ταχυδρομείου, το οποίο
ενδέχεται να μην είναι πολύ ασφαλές.
