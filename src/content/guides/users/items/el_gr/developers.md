Για προγραμματιστές που ίσως να μην θέλετε να είναι `Administrators`, σκεφτείτε να δημιουργήσετε έναν χρήστη `Administrator`
με τα ακόλουθα δικαιώματα:

1. Analytics Admin
2. Customizations Admin
3. Data Management Admin
4. Comment Moderation Admin
5. API/SSO Admin

Το σύνολο αυτών των δικαιωμάτων θα δώσει σε έναν προγραμματιστή όλα όσα χρειάζεται για να ρυθμίσει το FastComments καθώς και
την ορατότητα μέσα στο σύστημα που απαιτείται για να διασφαλίσει ότι λειτουργεί.

Η λογική για αυτά τα δικαιώματα είναι η εξής:

1. **Analytics Admin**: Αυτό μπορεί να χρησιμοποιηθεί για την αντιμετώπιση προβλημάτων στη χρήση του FastComments.
2. **Customizations Admin**: Αυτό θα απαιτηθεί για την εφαρμογή προσαρμοσμένων στυλ στο widget σχολίων.
3. **Data Management Admin**: Αυτό θα απαιτηθεί για τη διαχείριση εισαγωγών και εξαγωγών, καθώς και για τη ρύθμιση webhooks.
4. **Comment Moderation Admin**: Αυτό θα απαιτηθεί για να δείτε τα δεδομένα των σχολίων, τουλάχιστον κατά τη ρύθμιση.
5. **API/SSO Admin**: Αυτό θα τους επιτρέψει να ανακτήσουν απευθείας τα κλειδιά API από την πλατφόρμα μας. Θεωρούμε
αυτό πιο ασφαλές από το να το αντιγράψει ένας `Administrator` για αυτούς και να στείλει το API Secret μέσω email το οποίο
   ενδέχεται να μην είναι ιδιαίτερα ασφαλές.
