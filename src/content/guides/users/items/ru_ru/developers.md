Для разработчиков, которых вы, возможно, не хотите назначать `Administrators`, рассмотрите создание пользователя `Administrator` с следующими правами:

1. Analytics Admin
2. Customizations Admin
3. Data Management Admin
4. Comment Moderation Admin
5. API/SSO Admin

Этот набор прав даст разработчику все необходимое для настройки FastComments, а также доступ к информации в системе, необходимой для проверки её работоспособности.

Обоснование выбора этих прав следующее:

1. **Analytics Admin**: Это можно использовать для отладки использования FastComments.
2. **Customizations Admin**: Это потребуется для применения пользовательского оформления виджета комментариев.
3. **Data Management Admin**: Это потребуется для управления импортами и экспортами и настройки webhooks.
4. **Comment Moderation Admin**: Это потребуется для просмотра данных комментариев, по крайней мере на этапе настройки.
5. **API/SSO Admin**: Это позволит им получить ключи API непосредственно с нашей платформы. Мы считаем, что это более безопасно, чем когда `Administrator` копирует их для них и отправляет API Secret по электронной почте, что может быть небезопасно.
