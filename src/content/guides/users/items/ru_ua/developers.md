Для разработчиков, которых вы, возможно, не хотите делать `Administrators`, рассмотрите создание пользователя `Administrator` со следующими разрешениями:

1. Analytics Admin
2. Customizations Admin
3. Data Management Admin
4. Comment Moderation Admin
5. API/SSO Admin

Этот набор разрешений даст разработчику все, что нужно для настройки FastComments, а также предоставит необходимую видимость системы, чтобы убедиться, что она работает.

Обоснование этих разрешений следующее:

1. **Analytics Admin**: Эту роль можно использовать для отладки использования FastComments.
2. **Customizations Admin**: Эта роль потребуется для применения пользовательского оформления виджета комментариев.
3. **Data Management Admin**: Эта роль потребуется для управления импортом и экспортом, а также для настройки webhooks.
4. **Comment Moderation Admin**: Эта роль потребуется, чтобы просматривать данные комментариев, по крайней мере во время настройки.
5. **API/SSO Admin**: Это позволит им напрямую получать API keys с нашей платформы. Мы считаем это более безопасным, чем если `Administrator` скопирует их для них и отправит API Secret по электронной почте, что может быть не очень безопасно.
