### 브로드캐스트 ID

일부 API 호출에서 `broadcastId`를 전달해야 한다는 것을 보게 될 것입니다. 이벤트를 수신하면 이 ID가 다시 반환되므로, 클라이언트에서 낙관적으로 변경을 적용하려는 경우 해당 이벤트를 무시해야 하는지 알 수 있습니다
(대부분 그렇게 하는 것이 가장 좋은 사용자 경험을 제공하므로 아마 그렇게 하게 될 것입니다). 여기에는 `UUID`를 전달하세요. 이 ID는 브라우저 세션에서 두 번 발생하지 않을 만큼 충분히 고유해야 합니다.
