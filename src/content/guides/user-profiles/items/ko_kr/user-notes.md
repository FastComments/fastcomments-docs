User Notes를 사용하면 다른 FastComments 사용자에 대해 개인적이고 비공개적인 메모를 보관할 수 있습니다. 이 노트는 완전히 비공개이며 오직 본인만 볼 수 있습니다.

### What Are User Notes?

User Notes는 다음과 같습니다:
- **다른 사용자에 대해 본인이 작성하는 개인적인 메모**
- **본인만 볼 수 있음** - 메모의 대상 사용자는 이를 볼 수 없습니다
- **지속적 저장** - 해당 사용자의 프로필을 볼 때마다 저장되어 접근 가능합니다
- **사용자에 대한 맥락, 상호작용 또는 중요한 세부사항을 기억하는 데 유용**

### Why Use User Notes?

User Notes는 다음과 같은 경우에 유용합니다:

**모더레이터:**
- 특정 사용자와의 상호작용 기록 추적
- 경고 패턴이나 행동 관찰 사항 기록
- 중재 결정의 맥락을 기억
- 특정 조치가 취해진 이유 문서화

**커뮤니티 매니저:**
- 가치 있는 기여자와 그들의 전문성을 기억
- 특별 역할에 적합한 후보 사용자 메모
- 커뮤니티 구성원과의 후속 조치 추적

**일반 사용자:**
- 여러 커뮤니티에서 누군지 기억
- 특정 사용자와 논의한 주제 기록
- 관심 분야의 유용한 기여자 추적
- 대화에서 공유된 개인적 세부사항(적절한 범위 내) 기억

### Creating a User Note

사용자에 대한 노트를 추가하려면:

1. 해당 사용자의 프로필 페이지로 이동
2. "User Note" 또는 "Add Note" 섹션을 찾음
3. 새 노트를 생성하려면 클릭
4. 노트를 입력(최대 1000자)
5. 노트 저장

노트는 저장되며 해당 사용자의 프로필을 볼 때마다 표시됩니다.

### Viewing User Notes

작성한 노트는 해당 사용자의 프로필을 볼 때 표시됩니다:

- **위치:** 일반적으로 프로필 페이지의 전용 섹션
- **가시성:** 로그인한 본인만 볼 수 있음
- **맥락:** 사용자의 공개 정보와 함께 표시

이로 인해 자주 상호작용하는 사용자에 대한 중요한 맥락을 쉽게 기억할 수 있습니다.

### Editing User Notes

기존 노트를 업데이트하려면:

1. 사용자의 프로필 방문
2. 기존 노트를 찾음
3. "Edit"를 클릭하거나 노트 필드를 클릭
4. 변경사항 입력
5. 업데이트된 노트 저장

### Deleting User Notes

노트를 삭제하려면:

1. 사용자의 프로필 방문
2. 본인의 노트 찾기
3. "Delete" 또는 제거 옵션 클릭
4. 삭제 확인

삭제되면 노트는 영구적으로 제거되며 복구할 수 없습니다.

### Character Limit

User Notes는 **1000자**로 제한됩니다. 이는 다음을 장려합니다:
- 간결하고 집중된 메모
- 긴 서술보다는 핵심 포인트
- 빠른 참고용 정보

더 많은 공간이 필요하면 FastComments 외부에 별도의 문서를 보관하는 것을 고려하세요.

### Privacy and Security

**완전한 비공개:**
- 노트는 작성자 사용자 ID와 함께 저장됩니다
- 본인이 작성한 노트는 본인만 볼 수 있습니다
- 메모의 대상 사용자는 본인의 노트를 볼 수 없습니다
- 다른 사용자도 본인의 노트를 볼 수 없습니다
- 사이트 관리자도 개인 노트를 볼 수 없습니다(중재 로그와는 별도로 저장됨)

**중재 관련 주의:**
User Notes는 중재용 메모나 플래그와 다릅니다:
- 개인 참조용으로 User Notes를 사용하세요
- 위반 사항이나 문제에 대해서는 중재 신고 도구를 사용하세요
- User Notes는 어떤 조치나 알림을 유발하지 않습니다

### Best Practices

**포함하면 좋은 내용:**
- 사용자를 어떻게 알게 되었는지 또는 아는 관계
- 그들이 잘 아는 주제
- 이전의 유용한 상호작용
- 과거 대화의 맥락
- 전문 분야나 관심 분야
- 후속 조치에 대한 리마인더

**피해야 할 내용:**
- 민감한 개인 정보를 노트에 보관하지 마세요
- 비공개 노트라도 선동적이거나 부적절한 내용은 피하세요
- 중요한 정보는 노트에만 의존하지 말고(다른 백업 유지)
- 노트는 개인 참조용일 뿐 공식 기록이 아님을 기억하세요

**전문적인 사용:**
모더레이터와 커뮤니티 매니저의 경우:
- 사실적이고 객관적으로 기록
- 시간 민감한 정보에는 날짜 포함
- 개별 사건보다는 패턴을 기록
- 공식 중재 로그를 대체하지 않고 보완하도록 사용

### Use Cases

**예시 1: 커뮤니티 매니저**
> "기술 토론에 활발히 참여함. 백엔드 개발자로 근무. 문서화에 도움을 제안함. 위키 기여에 대해 후속 조치."

**예시 2: 모더레이터**
> "도움이 되는 사용자지만 때때로 논쟁이 격해짐. 전반적으로 예의 바름. 이전의 예의 관련 알림에 잘 반응함."

**예시 3: 일반 사용자**
> "게임 쓰레드에서 만남. 같은 게임을 플레이함. 유용한 튜토리얼 링크를 공유함. 영국 시간대에 거주."

### Limitations

**User Notes가 하지 않는 것:**
- 기기 간 동기화되지 않음(동일 계정으로 로그인하지 않는 한)
- 댓글이나 다른 컨텍스트에는 나타나지 않음, 프로필에만 표시
- 사용자 프로필이나 가시성에 영향을 주지 않음
- 검색 가능하지 않음(현재)
- 알림이나 조치를 유발하지 않음

**저장:**
- 노트는 삭제하지 않는 한 무기한 저장됨
- 노트당 최대 1000자
- 사용자당 한 개의 노트(여러 개 생성 대신 기존 노트를 업데이트)

### Technical Details

User Notes는:
- 작성자 사용자 ID로 데이터베이스에 저장
- 대상 사용자 ID와 연관
- 전송 중 암호화(HTTPS)
- 표준 FastComments 데이터 보존 정책 적용
- 공개 데이터 내보내기나 API 응답에 포함되지 않음

### Troubleshooting

**노트가 저장되지 않음:**
- 로그인 상태인지 확인
- 노트가 1000자 이내인지 확인
- 페이지를 새로고침하고 다시 추가 시도
- 인터넷 연결 확인

**노트가 나타나지 않음:**
- 노트를 작성한 계정으로 로그인했는지 확인
- 사용자 프로필 페이지 새로고침
- 필요한 경우 브라우저 캐시 삭제

**실수로 노트를 삭제함:**
안타깝게도 삭제된 노트는 복구할 수 없습니다. 중요한 정보는 FastComments 외부에 백업해 두는 것을 고려하세요.
