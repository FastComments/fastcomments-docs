用户备注允许你对其他 FastComments 用户保留私人、个人的注释。这些备注完全是私密的，仅你可见。

### 什么是用户备注？

用户备注是：
- **私人注释** 你为其他用户创建的注释
- **仅你可见** - 你书写的用户无法看到它们
- **持久保存** - 已保存，且在你查看该用户资料时可访问
- **有助于记忆** 上下文、互动或关于用户的重要细节

### 为什么使用用户备注？

用户备注对于以下情况很有用：

**版主：**
- 跟踪与特定用户的互动历史
- 记录警告模式或行为观察
- 记住用于审查决策的上下文
- 记录为何采取某些操作

**社区管理员：**
- 记住有价值的贡献者及其专长
- 记录可能适合担任特殊角色的用户
- 跟踪需要与社区成员跟进的事项

**普通用户：**
- 记住你在不同社区如何认识某人
- 记录你与特定用户讨论过的话题
- 记录你感兴趣领域中有帮助的贡献者
- 记住对话中分享的个人细节（在适当范围内）

### 创建用户备注

要添加关于某个用户的备注：

1. 访问该用户的资料页面
2. 查找“用户备注”或“添加备注”部分
3. 点击以创建新备注
4. 输入你的备注（最多 1000 个字符）
5. 保存备注

该备注将被保存，并在你查看该用户资料时显示。

### 查看用户备注

当查看关联用户的资料时，你的备注会显示：

- **位置：** 通常在资料页的专用部分
- **可见性：** 仅在你登录时对你可见
- **上下文：** 与用户的公开信息一同显示

这使你能轻松记住与你经常互动的用户的重要上下文。

### 编辑用户备注

要更新现有备注：

1. 访问该用户的资料
2. 找到你现有的备注
3. 点击“编辑”或点击备注字段
4. 进行修改
5. 保存更新后的备注

### 删除用户备注

要删除备注：

1. 访问该用户的资料
2. 找到你的备注
3. 点击“删除”或移除选项
4. 确认删除

一旦删除，备注将被永久移除，无法恢复。

### 字数限制

用户备注限制为 **1000 个字符**。这有助于：
- 简洁、重点突出的备注
- 强调关键点而不是冗长叙述
- 快速参考信息

如果你需要更多空间，考虑在 FastComments 之外另行保存文档。

### 隐私与安全

**完全私密：**
- 备注与创建者的用户 ID 一起存储
- 只有你能看到你创建的备注
- 被你书写的用户无法看到你的备注
- 其他用户无法看到你的备注
- 站点管理员无法查看你的个人备注（它们与审查日志分开存储）

**关于审查的说明：**
用户备注不同于审查备注或举报：
- 将用户备注用于个人参考
- 使用审查举报工具来处理违规或问题
- 用户备注不会触发任何操作或提醒

### 最佳实践

**建议包含：**
- 你如何认识或与该用户相识
- 他们熟悉的话题
- 以往有帮助的互动
- 过去对话的上下文
- 专业或兴趣领域
- 需要跟进的提醒

**应避免：**
- 不要在备注中存储敏感个人信息
- 即使是私人备注，也应避免煽动性或不当内容
- 不要依赖备注作为关键信息的唯一来源（请另行备份）
- 记住备注仅供你参考，并非官方记录

**专业使用：**
针对版主和社区管理员：
- 保持备注事实性与客观性
- 对于有时间敏感性的事项标注日期
- 记录模式而非孤立事件
- 将备注用于补充（而非替代）官方审查日志

### 使用场景

**示例 1：社区管理员**
> “在技术讨论中活跃。担任后端开发。曾主动提出帮助文档编写。跟进贡献 wiki 的事宜。”

**示例 2：版主**
> “是个有帮助的用户，但有时会卷入激烈争论。总体上很尊重他人。对之前关于文明交流的提醒反应良好。”

**示例 3：普通用户**
> “在游戏讨论串里认识的。玩我同样的游戏。分享过有用的教程链接。位于英国时区。”

### 限制

**用户备注不能：**
- 在设备间同步（除非你使用相同账户登录）
- 出现在评论或其他上下文中，仅在资料页显示
- 影响该用户的资料或可见性
- 被搜索到（当前不支持）
- 触发通知或操作

**存储：**
- 备注会无限期存储，除非你删除它们
- 每条备注最多 1000 个字符
- 每个用户只能有一条备注（请更新现有备注，而不是创建多条）

### 技术细节

用户备注：
- 与你的用户 ID 一起存储在数据库中
- 与目标用户的 ID 关联
- 传输过程加密（HTTPS）
- 遵循 FastComments 的标准数据保留政策
- 不包含在公共数据导出或 API responses 中

### 故障排除

**备注未保存：**
- 检查你是否已登录
- 确认备注少于 1000 个字符
- 尝试刷新页面并重新添加
- 检查你的网络连接

**备注未显示：**
- 确保你已登录创建该备注的账户
- 刷新该用户的资料页面
- 如有需要，清除浏览器缓存

**误删备注：**
很遗憾，已删除的备注无法恢复。请考虑在 FastComments 之外备份重要信息。
