Le processus de configuration du White Label est le suivant :

1. Choisissez comment la facturation sera gérée.
   1. Avec FastComments Pro, vous payez un montant fixe pour un certain nombre maximal de tenants en marque blanche.
   2. Avec FastComments Flex, vous payez pour chaque tenant et l'utilisation de ce tenant.
   3. Vous définissez les limites de chaque tenant dans les deux cas.
      1. Les limites peuvent être personnalisées par tenant. De plus, si vous mettez à jour les forfaits que vous vendez, vous pouvez le faire sans perturber la tarification que vous avez déjà fournie aux clients existants.
2. Familiarisez-vous avec la terminologie FastComments.com :
   1. `Tenant` est un "client".
   2. `TenantUser` est un administrateur avec un ensemble de privilèges dans le `Tenant`.
   3. `TenantPackage` est un forfait avec un ensemble de limites et de tarification disponible pour un `Tenant`.
3. Intégrez-vous à notre [API](/guide-api.html) ou utilisez des [scripts](https://github.com/FastComments/fastcomments-code-examples/tree/master/white-labeling) pour intégrer des tenants.
