### ID-ови за емитовање

Видјећете да треба да пошаљете `broadcast_id` у неким API позивима. Када примите догађаје, добићете овај ID назад, па ћете знати да игноришете догађај ако планирате оптимистички да примените промјене на клијенту (што ћете вјероватно желјети урадити јер пружа најбоље корисничко искуство). Овдје прослиједите UUID. ID би требало да буде довољно јединствен да се не појави два пута у току сесије претраживача.
