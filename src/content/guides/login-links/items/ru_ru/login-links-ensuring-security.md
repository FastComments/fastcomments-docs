Поскольку ссылки для входа по сути являются паролями, мы относимся к безопасности очень серьёзно.

Все ссылки для входа в нашей системе имеют срок действия и истекают через определённый период времени, а также у нас есть механизмы для обнаружения
подбора ссылки для входа. Некоторые ссылки для входа разделены на несколько паролей, и если один из них будет угадан,
другой станет недействительным.

### Security Compared to Passwords

Во многих системах, требующих пароль, вы можете воспользоваться механизмом восстановления пароля
если у вас есть электронная почта пользователя. Это означает, что если у вас есть доступ к почтовому ящику пользователя,
то не имеет значения, использует ли атакуемая система пароли или магические ссылки.

### New IP Login Alerts

Когда вход происходит с IP-адреса, который ранее не был замечен для данной учётной записи, FastComments отправляет уведомление по электронной почте о безопасности
с примерным местоположением и IP-адресом. Это помогает пользователям обнаружить неавторизованный доступ. Обратите внимание, что FastComments не хранит
сырые IP-адреса — хранится только замаскированная форма для обеспечения безопасности.

### Security Compared to MFA

Ссылки для входа менее безопасны, чем MFA. FastComments теперь поддерживает двухфакторную аутентификацию (2FA)
для учётных записей администраторов, чтобы обеспечить повышенную безопасность. Когда 2FA включена, она обязательна даже при использовании ссылок для входа.
