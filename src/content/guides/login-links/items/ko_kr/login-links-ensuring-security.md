로그인 링크는 본질적으로 비밀번호와 같으므로, 보안을 매우 중요하게 다룹니다.

시스템의 모든 로그인 링크는 일정 기간 후 만료되도록 설정되어 있으며, 로그인 링크를 추측하는 시도를 감지하는 메커니즘도 마련되어 있습니다.
일부 로그인 링크는 여러 개의 비밀번호로 분할되어 있고, 하나가 추측될 경우,
다른 것은 무효화됩니다.

### 비밀번호와 비교한 보안

대부분의 비밀번호를 요구하는 시스템에서는, 사용자의 이메일이 있으면 비밀번호 재설정 메커니즘을 통해 진행할 수 있습니다
사용자의 이메일 계정에 접근할 수 있다면,
공격 대상 시스템이 비밀번호를 사용하든 로그인 링크를 사용하든 상관없습니다.

### 새로운 IP 로그인 알림

해당 계정에서 이전에 본 적이 없는 IP 주소로부터 로그인이 발생하면, FastComments는 대략적인 위치와 IP 주소를 포함한 보안 경고 이메일을 전송합니다
이 기능은 사용자가 무단 접근을 감지하는 데 도움이 됩니다. FastComments는 원시 IP 주소를 저장하지 않으며,
보안 목적으로 난독화된 형태만 저장됩니다.

### MFA와 비교한 보안

로그인 링크는 MFA보다 보안이 낮습니다. FastComments는 이제 강화된 보안을 위해 관리자 계정에 대해 2단계 인증 (2FA)를 지원합니다
2FA가 활성화되어 있으면 로그인 링크를 사용하더라도 2FA가 필수입니다.
