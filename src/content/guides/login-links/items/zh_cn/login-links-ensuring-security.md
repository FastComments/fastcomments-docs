由于登录链接本质上相当于密码，我们非常重视其安全性。

系统中的所有登录链接都会在一定时间后过期，我们也有机制来检测
对登录链接的猜测。某些登录链接被拆分为多个密码，如果其中一个被猜中，
另一个将被作废。

### 与密码相比的安全性

对于大多数需要密码的系统，如果你有用户的电子邮件，就可以通过“忘记密码”机制
重设密码。这意味着，如果你能访问用户的电子邮件账户，
无论该系统使用的是密码还是魔术链接都没有差别。

### 新 IP 登录提醒

当某个账户从此前未见过的 IP 地址登录时，FastComments 会发送一封安全提醒电子邮件
其中包含大致位置和 IP 地址。这有助于用户发现未授权访问。请注意，FastComments 不会存储
原始 IP 地址——出于安全目的仅存储经混淆的形式。

### 与 MFA（多因素认证）相比的安全性

登录链接的安全性低于 MFA（多因素认证）。FastComments 现在支持两步验证（2FA）
用于管理员账户以提供更高的安全性。当启用 2FA 时，即使用登录链接也必须进行 2FA 验证。
