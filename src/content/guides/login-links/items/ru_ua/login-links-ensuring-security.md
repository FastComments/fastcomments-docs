Поскольку ссылки для входа фактически являются паролями, мы относимся к безопасности очень серьёзно.

Все ссылки для входа в нашей системе настроены так, чтобы истекать через определённый период времени, а также у нас есть механизмы для обнаружения
угадывания ссылки для входа. Некоторые ссылки для входа разделены на несколько паролей, и если один из них будет угадан,
другой будет аннулирован.

### Безопасность по сравнению с паролями

В большинстве систем, требующих пароль, вы можете воспользоваться механизмом восстановления пароля,
если у вас есть электронная почта пользователя. Это означает, что если у вас есть доступ к почтовому ящику пользователя,
неважно, использует ли атакуемая система пароли или магические ссылки.

### Оповещения о входе с нового IP

Когда вход происходит с IP-адреса, который ранее не встречался для данной учётной записи, FastComments отправляет на почту предупреждение о безопасности
с примерным местоположением и IP-адресом. Это помогает пользователям обнаруживать неавторизованный доступ. Обратите внимание, что FastComments не хранит
исходные IP-адреса — для безопасности хранится только обфусцированная форма.

### Безопасность по сравнению с MFA

Ссылки для входа менее безопасны, чем MFA. FastComments теперь поддерживает двухфакторную аутентификацию (2FA)
для учётных записей администраторов для повышения уровня безопасности. Когда 2FA включена, она требуется даже при использовании ссылок для входа.
