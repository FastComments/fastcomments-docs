Оскільки посилання для входу по суті є паролями, ми ставимося до безпеки дуже серйозно.

Усі посилання для входу в нашій системі налаштовані на те, щоб закінчувати термін дії після певного періоду часу, а також у нас є механізми для виявлення
вгадування посилання для входу. Деякі посилання для входу розбиті на кілька паролів, і якщо один вгаданий,
інший буде скасований.

### Безпека порівняно з паролями

У більшості систем, що вимагають пароль, ви можете скористатися механізмом «Забули пароль»,
якщо у вас є електронна адреса користувача. Це означає, що якщо у вас є доступ до поштової скриньки користувача,
то не має значення, чи система під атакою використовує паролі або магічні посилання.

### Сповіщення про вхід з нового IP

Коли вхід відбувається з IP-адреси, яка раніше не фіксувалася для даного облікового запису, FastComments надсилає електронне сповіщення про безпеку
з приблизним місцезнаходженням та IP-адресою. Це допомагає користувачам виявляти несанкціонований доступ. Зверніть увагу, що FastComments не зберігає
необроблені IP-адреси — з міркувань безпеки зберігається лише обфускована форма.

### Резервна електронна адреса для відновлення облікового запису

Якщо ви втратите доступ до основної електронної адреси, ви можете скористатися перевіреною резервною електронною адресою, щоб відновити свій обліковий запис. Ваша резервна електронна адреса працює
з усіма механізмами входу. Ви можете ввести її на сторінці «Забули ім'я користувача», використовувати її для входу через магічне посилання, або ввести її в
поле ім'я користувача/електронна адреса для входу за допомогою пароля.

Щоб налаштувати резервну електронну адресу, перейдіть до [Деталі облікового запису](https://fastcomments.com/auth/my-account/edit-details) і натисніть
**Вказати резервну електронну адресу**. Ваша резервна електронна адреса використовується лише для відновлення облікового запису і не отримуватиме сповіщень.

### Безпека порівняно з MFA

Посилання для входу менш безпечні, ніж MFA. FastComments тепер підтримує двофакторну автентифікацію (2FA)
для адміністративних облікових записів з метою підвищення безпеки. Коли 2FA ввімкнено, вона обов’язкова навіть при використанні посилань для входу.
