Оскільки посилання для входу фактично є паролями, ми ставимося до безпеки дуже серйозно.

Усі посилання для входу в нашій системі налаштовані так, щоб термін їх дії закінчувався через певний період часу, і ми також маємо механізми для виявлення
вгадування посилання для входу. Деякі посилання для входу розділені на кілька паролів, і якщо один з них буде вгадано,
інший буде анульований.

### Безпека порівняно з паролями

У більшості систем, що вимагають пароль, можна скористатися механізмом «Забули пароль», якщо у вас є електронна адреса користувача. Це означає, що якщо ви маєте доступ до поштової скриньки користувача,
не має значення, чи система під атакою використовує паролі або магічні посилання.

### Сповіщення про вхід з нового IP

Якщо вхід відбувається з IP-адреси, яка раніше не використовувалася для даного облікового запису, FastComments надсилає електронного листа з попередженням про безпеку
з приблизним місцеположенням та IP-адресою. Це допомагає користувачам виявляти несанкціонований доступ. Зауважте, що FastComments не зберігає необроблені IP-адреси — з міркувань безпеки зберігається лише замаскована форма.

### Безпека порівняно з MFA

Посилання для входу менш безпечні, ніж MFA. FastComments тепер підтримує двофакторну автентифікацію (2FA)
для облікових записів адміністраторів для підвищення рівня безпеки. Коли 2FA увімкнена, вона є обов'язковою навіть при використанні посилань для входу.
