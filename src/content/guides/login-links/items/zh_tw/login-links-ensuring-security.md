由於登入連結本質上等同密碼，我們非常重視其安全性。

我們系統中的所有登入連結都設定為在一段時間後過期，我們也有機制來偵測
對登入連結的猜測。有些登入連結會被拆分成多個密碼，若其中一個被猜中，
另一個將會被作廢。

### 與密碼相比的安全性

在大多數需要密碼的系統中，如果你有使用者的電子郵件，就可以透過忘記密碼的機制
重設。這表示，如果你能存取使用者的電子郵件帳戶，不論被攻擊的系統使用的是密碼還是魔法連結，
都沒有差別。

### 新 IP 登入警示

當某個帳戶出現從該帳戶先前未見過的 IP 位址登入時，FastComments 會寄送一封包含大致位置與 IP 位址的安全警示電子郵件，
協助使用者偵測未經授權的存取。請注意，FastComments 不會儲存
原始 IP 位址 — 為了安全，只會儲存經過模糊化的形式。

### 與 MFA 相比的安全性

登入連結不如 MFA 安全。FastComments 現在支援管理員帳戶的兩因素驗證（2FA）
以提供更強的安全性。啟用 2FA 時，即使使用登入連結也必須通過 2FA。
