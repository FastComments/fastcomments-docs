Объект `Comment` представляет собой комментарий, оставленный пользователем.

Связь между родительскими и дочерними комментариями определяется с помощью `parentId`.

Структура объекта Comment выглядит следующим образом:

[inline-code-attrs-start title = 'Структура объекта Comment'; type = 'typescript'; isFunctional = false; inline-code-attrs-end]
[inline-code-start]
interface Comment {
    /** READONLY: Установлено в true, если движок спама определил комментарий как спам. **/
    aiDeterminedSpam?: boolean
    /** Является ли комментарий одобренным для показа. При сохранении комментария устанавливается в true, иначе он будет скрыт. **/
    approved?: boolean
    /** Аватар пользователя. **/
    avatarSrc?: string
    /** Дочерние комментарии. Заполняется не во всех сценариях. Используется, когда asTree установлен в true через API. **/
    children: Comment[]
    /** Исходный текст комментария. **/
    comment: string
    /** READONLY: Текст комментария, распарсенный в HTML. **/
    commentHTML?: string
    /** Электронная почта комментатора. Требуется, если анонимные комментарии отключены. **/
    commenterEmail?: string
    /** Ссылка комментатора (например, их блог). **/
    commenterLink?: string
    /** Имя комментатора. Всегда обязательно. Если недоступно, установите что-то вроде "Anonymous". **/
    commenterName: string
    /** Дата оставления комментария, в формате UTC epoch. **/
    date: number
    /** "Отображаемая метка" для комментария — например "Admin", "Moderator" или что-то вроде "VIP User". **/
    displayLabel?: string
    /** Домен, на котором был размещён комментарий. **/
    domain?: string
    /** READONLY: Количество раз, когда комментарий был помечен. **/
    flagCount?: number
    /** Хэштеги, написанные в комментарии, которые были успешно распознаны. Также можно вручную добавить хэштеги для запросов, но они не будут автоматически отображаться в тексте комментария. **/
    hashTags?: CommentHashTag[]
    /** READONLY: Содержит ли комментарий изображения? **/
    hasImages?: boolean
    /** READONLY: Содержит ли комментарий ссылки? **/
    hasLinks?: boolean
    /** READONLY: Уникальный идентификатор комментария. **/
    id: string
    /** Только при создании! Для хранения значение хешируется. **/
    ip?: string
    /** READONLY: Заблокировал ли текущий пользователь автора этого комментария? **/
    isBlocked?: boolean
    /** READONLY: Комментарий написан администратором? Устанавливается автоматически на основе userId. **/
    isByAdmin?: boolean
    /** READONLY: Комментарий написан модератором? Устанавливается автоматически на основе userId. **/
    isByModerator?: boolean
    /** Установлено в true, если комментарий был мягко удалён (плейсхолдер оставлен из-за другой конфигурации). **/
    isDeleted?: boolean
    /** Установлено в true, если аккаунт пользователя был удалён и комментарий пришлось сохранить. **/
    isDeletedUser?: boolean
    /** READONLY: Был ли комментарий помечен текущим вошедшим пользователем (contextUserId)? **/
    isFlagged?: boolean
    /** Закреплён ли комментарий? **/
    isPinned?: boolean
    /** Заблокирован ли комментарий? Когда true, никто (включая модераторов) не может отвечать, редактировать или удалять его, пока он не будет разблокирован. **/
    isLocked?: boolean
    /** Является ли комментарий спамом? **/
    isSpam?: boolean
    /** READONLY: Проголосовал ли текущий пользователь (contextUserId) против этого комментария? **/
    isVotedDown?: boolean
    /** READONLY: Проголосовал ли текущий пользователь (contextUserId) за этот комментарий? **/
    isVotedUp?: boolean
    /** Локаль комментария. Если не указана, будет определена по HTTP-заголовку Accept-Language. **/
    locale?: 'de_de' | 'en_us' | 'es_es' | 'fr_fr' | 'it_it' | 'ja_jp' | 'ko_kr' | 'pl_pl' | 'pt_br' | 'ru_ru' | 'tr_tr' | 'zh_cn' | 'zh_tw'
    /** READONLY: @mentions, которые были успешно распознаны в комментарии. **/
    mentions?: CommentUserMention[]
    /** Необязательные метаданные, связанные с комментарием. **/
    meta?: Record<string, string | number | boolean>
    /** Необязательный список идентификаторов групп модерации, связанных с этим комментарием. **/
    moderationGroupIds?: string[]|null
    /** READONLY: Идентификатор объекта голосования, соответствующий голосу текущего пользователя (contextUserId) за этот комментарий. **/
    myVoteId?: string
    /** Были ли отправлены уведомления об этом комментарии комментаторам. Чтобы предотвратить отправку уведомлений при импорте, установите это в true. **/
    notificationSentForParent?: boolean
    /** Были ли отправлены уведомления об этом комментарии пользователям tenant. Чтобы предотвратить отправку уведомлений при импорте, установите это в true. **/
    notificationSentForParentTenant?: boolean
    /** Заголовок страницы, на которой был этот комментарий. **/
    pageTitle?: string
    /** Если мы отвечаем на комментарий, это ID, на который мы отвечаем. **/
    parentId?: string|null
    /** Помечен ли комментарий как просмотренный. **/
    reviewed: boolean
    /** Идентификатор tenant, к которому принадлежит комментарий. **/
    tenantId: string
    /** Пользователь, который написал комментарий. Создаётся автоматически при сохранении комментария с именем/электронной почтой. **/
    userId?: string|null
    /** URL страницы, где виден этот комментарий, например запись в блоге. **/
    url: string
    /** "Очищенная" версия urlId, которую вы передали. При сохранении вы указываете это поле, но при получении комментария оно будет "очищено", а ваше оригинальное значение перемещено в "urlIdRaw". **/
    urlId: string
    /** READONLY: Оригинальный urlId, который вы передали. **/
    urlIdRaw?: string
    /** Подтверждены ли пользователь и этот комментарий? **/
    verified: boolean
    /** Количество голосов "за". **/
    votesUp?: number
    /** Количество голосов "против". **/
    votesDown?: number
    /** "Карма" комментария (= голоса за - голоса против). **/
    votes?: number
}
[inline-code-end]

Некоторые из этих полей помечены `READONLY` — они возвращаются API, но не могут быть установлены.

### Структура текста комментария

Комментарии пишутся в варианте markdown от FastComments, который представляет собой обычный markdown плюс традиционные `bbcode`-теги для изображений, такие как `[img]path[/img]`.

Текст хранится в двух полях. Текст, введённый пользователем, хранится без изменений в поле `comment`. Он рендерится и сохраняется в поле `commentHTML`.

Разрешённые HTML-теги: `b, u, i, strike, pre, span, code, img, a, strong, ul, ol, li, and br`.

Рекомендуется рендерить HTML, так как это очень небольшой поднабор HTML, и создание рендерера достаточно простое. Существуют разные библиотеки, например для React Native и Flutter, которые помогут в этом.

Вы можете выбрать рендеринг ненормализованного значения поля `comment`. [Пример парсера здесь.](https://github.com/FastComments/fastcomments-code-examples/blob/master/custom-client/client/parse-comment.js).

Примерный парсер также можно настроить для работы с HTML и преобразовать HTML-теги в ожидаемые элементы для рендеринга на вашей платформе. 

### Тегирование

Когда пользователи отмечаются в комментарии, информация сохраняется в списке с названием `mentions`. Каждый объект в этом списке имеет следующую структуру.

[inline-code-attrs-start title = 'Объект упоминания комментария'; type = 'typescript'; inline-code-attrs-end]
[inline-code-start]
interface CommentUserMention {
    /** Идентификатор пользователя. Для SSO-пользователей сюда будет добавлен префикс tenant id. **/
    id: string
    /** Финальный текст @упоминания, включая символ @. **/
    tag: string
    /** Оригинальный текст @упоминания, включая символ @. **/
    rawTag: string
    /** Тип отмеченного пользователя. user = аккаунт FastComments.com. sso = SSOUser. **/
    type: 'user'|'sso'
    /** Если пользователь отказался от уведомлений, это всё равно будет установлено в true. **/
    sent: boolean
}
[inline-code-end]

### Хештеги

Когда используются хештеги и успешно распознаются, информация сохраняется в списке с названием `hashTags`. Каждый объект в этом списке имеет следующую структуру. Хештеги также можно вручную добавить в массив `hashTags` комментария для запросов, если установлен `retain`.

[inline-code-attrs-start title = 'Объект хештега комментария'; type = 'typescript'; inline-code-attrs-end]
[inline-code-start]
interface CommentHashTag {
    /** Идентификатор хэштега. **/
    id: string
    /** Финальный текст #хэштега, включая символ #. **/
    tag: string
    /** Если хэштег связан с пользовательским URL, он будет здесь задан. **/
    url?: string
    /** Если хэштег следует сохранять, даже если он отсутствует в тексте комментария при обновлении. Полезно для пометки комментариев без изменения текста. **/
    retain?: boolean
}
[inline-code-end]
