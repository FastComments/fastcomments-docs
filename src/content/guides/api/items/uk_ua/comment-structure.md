Об'єкт `Comment` представляє собою коментар, залишений користувачем.

Відношення між батьківськими та дочірніми коментарями визначається через `parentId`.

Структура об'єкта Comment виглядає так:

[inline-code-attrs-start title = 'Структура об’єкта Comment'; type = 'typescript'; isFunctional = false; inline-code-attrs-end]
[inline-code-start]
interface Comment {
    /** READONLY: Встановлено true, якщо антиспам-движок визначив коментар як спам. **/
    aiDeterminedSpam?: boolean
    /** Чи схвалено показ коментаря. Встановлюється в true при збереженні коментаря, інакше він буде прихований. **/
    approved?: boolean
    /** Аватар користувача. **/
    avatarSrc?: string
    /** Дочірні коментарі. Не заповнюється в усіх сценаріях. Використовується, коли через API встановлено asTree = true. **/
    children: Comment[]
    /** Сирий (необроблений) текст коментаря. **/
    comment: string
    /** READONLY: Коментар розпарсовано в HTML. **/
    commentHTML?: string
    /** Електронна адреса коментатора. Обов'язкова, якщо анонімні коментарі вимкнені. **/
    commenterEmail?: string
    /** Посилання коментатора (наприклад, їхній блог). **/
    commenterLink?: string
    /** Ім'я коментатора. Завжди обов'язкове. Якщо недоступне, вкажіть щось на кшталт "Anonymous". **/
    commenterName: string
    /** Дата створення коментаря, у форматі UTC epoch. **/
    date: number
    /** Мітка для відображення коментаря - наприклад "Admin", "Moderator", або щось на кшталт "VIP User". **/
    displayLabel?: string
    /** Домен, на якому опубліковано коментар. **/
    domain?: string
    /** READONLY: Кількість разів, коли коментар було позначено. **/
    flagCount?: number
    /** Хештеги (#...), написані в коментарі та успішно розпарсені. Ви також можете вручну додавати хештеги для запитів, але вони не відобразяться в тексті коментаря автоматично. **/
    hashTags?: CommentHashTag[]
    /** READONLY: Чи містить коментар зображення? **/
    hasImages?: boolean
    /** READONLY: Чи містить коментар посилання? **/
    hasLinks?: boolean
    /** READONLY: Унікальний id коментаря. **/
    id: string
    /** Лише при створенні! Це хешується для збереження. **/
    ip?: string
    /** READONLY: Чи заблокував поточний користувач автора цього коментаря? **/
    isBlocked?: boolean
    /** READONLY: Чи автор коментаря є адміном? Встановлюється автоматично на основі userId. **/
    isByAdmin?: boolean
    /** READONLY: Чи автор коментаря є модератором? Встановлюється автоматично на основі userId. **/
    isByModerator?: boolean
    /** Встановити true, якщо коментар було м'яко видалено (була залишена тимчасова заміна через інші налаштування). **/
    isDeleted?: boolean
    /** Встановити true, якщо акаунт користувача було видалено і коментар потрібно було зберегти. **/
    isDeletedUser?: boolean
    /** READONLY: Чи було позначено (flagged) поточним увійденим користувачем (contextUserId)? **/
    isFlagged?: boolean
    /** Чи закріплено коментар? **/
    isPinned?: boolean
    /** Чи заблоковано коментар? Якщо true, ніхто (включно з модераторами) не може відповідати, редагувати або видаляти його, поки його не розблокують. **/
    isLocked?: boolean
    /** Чи є коментар спамом? **/
    isSpam?: boolean
    /** READONLY: Чи проголосовано проти цього коментаря поточним користувачем (contextUserId)? **/
    isVotedDown?: boolean
    /** READONLY: Чи проголосовано за цей коментар поточним користувачем (contextUserId)? **/
    isVotedUp?: boolean
    /** Локаль коментаря. Якщо не вказано, буде визначено із HTTP-заголовка Accept-Language. **/
    locale?: 'de_de' | 'en_us' | 'es_es' | 'fr_fr' | 'it_it' | 'ja_jp' | 'ko_kr' | 'pl_pl' | 'pt_br' | 'ru_ru' | 'tr_tr' | 'zh_cn' | 'zh_tw'
    /** READONLY: @згадки, написані в коментарі та успішно розпарсені. **/
    mentions?: CommentUserMention[]
    /** Опціональні метадані, пов'язані з коментарем. **/
    meta?: Record<string, string | number | boolean>
    /** Необов'язковий список id груп модерації, пов'язаних з цим коментарем. **/
    moderationGroupIds?: string[]|null
    /** READONLY: id об'єкта голосу, що відповідає голосу поточного користувача (contextUserId) за цей коментар. **/
    myVoteId?: string
    /** Чи було надіслано повідомлення про цей коментар коментаторам. Щоб запобігти надсиланню сповіщень під час імпорту, встановіть це в true. **/
    notificationSentForParent?: boolean
    /** Чи було надіслано повідомлення про цей коментар користувачам tenant. Щоб запобігти надсиланню сповіщень під час імпорту, встановіть це в true. **/
    notificationSentForParentTenant?: boolean
    /** Заголовок сторінки, на якій був цей коментар. **/
    pageTitle?: string
    /** Якщо ми відповідаємо на коментар, це ID коментаря, на який відповідаємо. **/
    parentId?: string|null
    /** Чи позначено коментар як переглянутий. **/
    reviewed: boolean
    /** id орендатора (tenant), до якого належить коментар. **/
    tenantId: string
    /** Користувач, який написав коментар. Створюється автоматично при збереженні коментаря з ім'ям/емейлом. **/
    userId?: string|null
    /** URL місця, де видно цей коментар, наприклад запис блогу. **/
    url: string
    /** "Очищена" версія urlId, який ви передали. Під час збереження ви вказуєте це поле, але при отриманні коментаря воно буде "очищене" і ваше оригінальне значення переміщено до "urlIdRaw". **/
    urlId: string
    /** READONLY: Початковий urlId, який ви передавали. **/
    urlIdRaw?: string
    /** Чи верифіковано користувача та цей коментар? **/
    verified: boolean
    /** Кількість голосів за. **/
    votesUp?: number
    /** Кількість голосів проти. **/
    votesDown?: number
    /** "Карма" коментаря (= votes up - votes down). **/
    votes?: number
}
[inline-code-end]

Деякі з цих полів позначено як `READONLY` — вони повертаються API, але не можуть бути встановлені.

### Структура тексту коментаря

Коментарі пишуться у фірмовому діалекті markdown від FastComments — це просто markdown плюс традиційні `bbcode`-стиль теги для зображень, як-от `[img]path[/img]`.

Текст зберігається у двох полях. Текст, введений користувачем, зберігається без змін у полі `comment`. Він рендериться та зберігається в полі `commentHTML`.

Дозволені HTML-теги: `b, u, i, strike, pre, span, code, img, a, strong, ul, ol, li, and br`.

Рекомендується рендерити HTML, оскільки це дуже невелика підмножина HTML, тож створити рендерер досить просто. Існує кілька бібліотек для React Native і Flutter, наприклад, які можуть у цьому допомогти.

Ви можете обрати рендеринг ненормалізованого значення поля `comment`. [Приклад парсера тут.](https://github.com/FastComments/fastcomments-code-examples/blob/master/custom-client/client/parse-comment.js).

Приклад парсера також можна налаштувати для роботи з HTML і перетворення HTML-тегів у очікувані елементи для відображення на вашій платформі. 

### Тегування

Коли користувачів тегують у коментарі, інформація зберігається в списку `mentions`. Кожен об'єкт у цьому списку має таку структуру.

[inline-code-attrs-start title = 'Об’єкт згадок коментаря'; type = 'typescript'; inline-code-attrs-end]
[inline-code-start]
interface CommentUserMention {
    /** id користувача. Для SSO-користувачів тут буде префікс вашого tenant id. **/
    id: string
    /** Фінальний текст @згадки, включно з символом @. **/
    tag: string
    /** Оригінальний текст @згадки, включно з символом @. **/
    rawTag: string
    /** Тип користувача, який було згадано. user = обліковий запис FastComments.com. sso = SSOUser. **/
    type: 'user'|'sso'
    /** Якщо користувач відмовився від сповіщень, це все одно буде встановлено в true. **/
    sent: boolean
}
[inline-code-end]

### Хештеги

Коли хештеги використовуються та успішно розпарсуються, інформація зберігається в списку `hashTags`. Кожен об'єкт у цьому списку має таку структуру. Хештеги також можна додавати вручну до масиву `hashTags` коментаря для запитів, якщо встановлено `retain`.

[inline-code-attrs-start title = 'Об’єкт хештегу коментаря'; type = 'typescript'; inline-code-attrs-end]
[inline-code-start]
interface CommentHashTag {
    /** id хештегу. **/
    id: string
    /** Фінальний текст #хештегу, включно зі символом #. **/
    tag: string
    /** Якщо хештег пов’язано з кастомним URL, він буде вказаний. **/
    url?: string
    /** Чи потрібно зберігати хештег, навіть якщо він не присутній у тексті коментаря при оновленні. Корисно для тегування коментарів без зміни тексту коментаря. **/
    retain?: boolean
}
[inline-code-end]
