Объект `Comment` представляет комментарий, оставленный пользователем.

Взаимоотношение между родительскими и дочерними комментариями определяется через `parentId`.

Структура объекта Comment выглядит следующим образом:

[inline-code-attrs-start title = 'Структура Comment'; type = 'typescript'; isFunctional = false; inline-code-attrs-end]
[inline-code-start]
interface Comment {
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: Установлено в true, если спам-движок определил, что комментарий — спам. **/
    aiDeterminedSpam?: boolean
    /** Утверждён ли комментарий для показа. Устанавливается в true при сохранении комментария, иначе он будет скрыт. **/
    approved?: boolean
    /** Аватар пользователя. **/
    avatarSrc?: string
    /** Дочерние комментарии. Не заполняется во всех сценариях. Используется, когда asTree установлено в true через API. **/
    children: Comment[]
    /** Исходный текст комментария. **/
    comment: string
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: Текст комментария, преобразованный в HTML. **/
    commentHTML?: string
    /** Email автора комментария. Обязателен, если анонимные комментарии отключены. **/
    commenterEmail?: string
    /** Ссылка автора комментария (например, его блог). **/
    commenterLink?: string
    /** Имя автора комментария. Всегда обязательно. Если недоступно, укажите что-то вроде "Anonymous". **/
    commenterName: string
    /** Дата оставления комментария, в виде UTC epoch. **/
    date: number
    /** «Отображаемая метка» для комментария — например "Admin", "Moderator" или что-то вроде "VIP User". **/
    displayLabel?: string
    /** Домен, на котором опубликован комментарий. **/
    domain?: string
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: Количество раз, когда комментарий был помечен. **/
    flagCount?: number
    /** #хэштеги, написанные в комментарии и успешно распарсенные. Также вы можете вручную добавить хэштеги для запроса, но они не будут автоматически отображаться в тексте комментария. **/
    hashTags?: CommentHashTag[]
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: Содержит ли комментарий изображения? **/
    hasImages?: boolean
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: Содержит ли комментарий ссылки? **/
    hasLinks?: boolean
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: Уникальный идентификатор комментария. **/
    id: string
    /** Только при создании! Для хранения значение хешируется. **/
    ip?: string
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: Заблокировал ли текущий пользователь автора этого комментария? **/
    isBlocked?: boolean
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: Комментарий написан администратором? Автоматически задаётся на основе userId. **/
    isByAdmin?: boolean
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: Комментарий написан модератором? Автоматически задаётся на основе userId. **/
    isByModerator?: boolean
    /** Установлено в true, если комментарий был мягко удалён (оставлен заполнитель из-за какой-либо другой конфигурации). **/
    isDeleted?: boolean
    /** Установлено в true, если аккаунт пользователя был удалён, а комментарий нужно было сохранить. **/
    isDeletedUser?: boolean
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: Помечен ли комментарий текущим пользователем (contextUserId)? **/
    isFlagged?: boolean
    /** Закреплён ли комментарий? **/
    isPinned?: boolean
    /** Заблокирован ли комментарий? Если true, никто (включая модераторов) не может отвечать, редактировать или удалять его, пока он не будет разблокирован. **/
    isLocked?: boolean
    /** Является ли комментарий спамом? **/
    isSpam?: boolean
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: Проголосовал ли текущий пользователь (contextUserId) против этого комментария? **/
    isVotedDown?: boolean
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: Проголосовал ли текущий пользователь (contextUserId) за этот комментарий? **/
    isVotedUp?: boolean
    /** Локаль комментария. Если не указана, будет определена по HTTP-заголовку Accept-Language. **/
    locale?: 'de_de' | 'en_us' | 'es_es' | 'fr_fr' | 'it_it' | 'ja_jp' | 'ko_kr' | 'pl_pl' | 'pt_br' | 'ru_ru' | 'tr_tr' | 'zh_cn' | 'zh_tw'
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: @упоминания, написанные в комментарии и успешно распарсенные. **/
    mentions?: CommentUserMention[]
    /** Необязательные метаданные, связанные с комментарием. **/
    meta?: Record<string, string | number | boolean>
    /** Необязательный список идентификаторов групп модерации, связанных с этим комментарием. **/
    moderationGroupIds?: string[]|null
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: Идентификатор объекта голоса, соответствующего голосу текущего пользователя (contextUserId) за этот комментарий. **/
    myVoteId?: string
    /** Были ли отправлены уведомления об этом комментарии комментаторам. Чтобы предотвратить отправку уведомлений при импорте, установите это в true. **/
    notificationSentForParent?: boolean
    /** Были ли отправлены уведомления об этом комментарии пользователям арендатора. Чтобы предотвратить отправку уведомлений при импорте, установите это в true. **/
    notificationSentForParentTenant?: boolean
    /** Заголовок страницы, на которой был оставлен комментарий. **/
    pageTitle?: string
    /** Если мы отвечаем на комментарий, это идентификатор, на который мы отвечаем. **/
    parentId?: string|null
    /** Помечен ли комментарий как просмотренный. **/
    reviewed: boolean
    /** Идентификатор арендатора, которому принадлежит комментарий. **/
    tenantId: string
    /** Пользователь, который написал комментарий. Создаётся автоматически при сохранении комментария с именем/email. **/
    userId?: string|null
    /** URL места, где виден этот комментарий, например, запись в блоге. **/
    url: string
    /** «Очищенная» версия urlId, которую вы передали. При сохранении вы указываете это поле, но при получении комментария оно будет «очищено», а ваше оригинальное значение перемещено в "urlIdRaw". **/
    urlId: string
    /** ТОЛЬКО ДЛЯ ЧТЕНИЯ: Оригинальный urlId, который вы передали. **/
    urlIdRaw?: string
    /** Подтверждены ли пользователь и этот комментарий? **/
    verified: boolean
    /** Количество положительных голосов. **/
    votesUp?: number
    /** Количество отрицательных голосов. **/
    votesDown?: number
    /** «Карма» комментария (= votes up - votes down). **/
    votes?: number
}
[inline-code-end]

Некоторые из этих полей помечены `READONLY` — они возвращаются API, но не могут быть установлены.

### Структура текста комментария

Комментарии пишутся в варианте markdown от FastComments, который представляет собой просто markdown плюс традиционные bbcode-подобные теги для изображений, например `[img]path[/img]`.

Текст хранится в двух полях. Текст, введённый пользователем, хранится без изменений в поле `comment`. Он рендерится и сохраняется в поле `commentHTML`.

Разрешённые HTML-теги: `b, u, i, strike, pre, span, code, img, a, strong, ul, ol, li, and br`.

Рекомендуется рендерить HTML, так как это очень небольшое подмножество HTML, создание рендера довольно простое. Существуют несколько библиотек для React Native и Flutter, например, которые помогут с этим.

Вы можете выбрать рендеринг нормализованного значения поля `comment`. [Пример парсера доступен здесь.](https://github.com/FastComments/fastcomments-code-examples/blob/master/custom-client/client/parse-comment.js).

Примерный парсер также можно настроить для работы с HTML и преобразовать HTML-теги в ожидаемые элементы для рендера на вашей платформе.

### Тегирование

Когда пользователи отмечаются в комментарии, информация сохраняется в списке, называемом `mentions`. Каждый объект в этом списке имеет следующую структуру.

[inline-code-attrs-start title = 'Объект CommentUserMention'; type = 'typescript'; inline-code-attrs-end]
[inline-code-start]
interface CommentUserMention {
    /** Идентификатор пользователя. Для SSO-пользователей к нему будет добавлен префикс tenant id. **/
    id: string
    /** Финальный текст @упоминания, включая символ @. **/
    tag: string
    /** Оригинальный текст @упоминания, включая символ @. **/
    rawTag: string
    /** Тип отмеченного пользователя. user = аккаунт FastComments.com. sso = SSOUser. **/
    type: 'user'|'sso'
    /** Если пользователь отключил уведомления, это всё равно будет установлено в true. **/
    sent: boolean
}
[inline-code-end]

### Хэштеги

Когда используются хэштеги и успешно распарсены, информация сохраняется в списке, называемом `hashTags`. Каждый объект в этом списке имеет следующую структуру. Хэштеги также можно вручную добавить в массив `hashTags` комментария для запросов, если установлено `retain`.

[inline-code-attrs-start title = 'Объект CommentHashTag'; type = 'typescript'; inline-code-attrs-end]
[inline-code-start]
interface CommentHashTag {
    /** Идентификатор хэштега. **/
    id: string
    /** Финальный текст #хэштега, включая символ #. **/
    tag: string
    /** Если хэштег связан с пользовательским URL, он будет задан. **/
    url?: string
    /** Если следует сохранять хэштег, даже если он отсутствует в тексте комментария при обновлении. Полезно для пометки комментариев без изменения их текста. **/
    retain?: boolean
}
[inline-code-end]
