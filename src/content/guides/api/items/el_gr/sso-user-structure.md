FastComments παρέχει μια εύχρηστη λύση SSO. Η ενημέρωση των στοιχείων ενός χρήστη με την ενσωμάτωση βασισμένη σε HMAC είναι
τόσο απλή όσο το να φορτώσει ο χρήστης τη σελίδα με ένα ενημερωμένο payload.

Ωστόσο, ίσως είναι επιθυμητό να διαχειρίζεστε έναν χρήστη εκτός αυτής της ροής, για να βελτιώσετε τη συνοχή της εφαρμογής σας.

Το SSO User API παρέχει έναν τρόπο για CRUD αντικειμένων που ονομάζουμε SSOUsers. Αυτά τα αντικείμενα διαφέρουν από τους κανονικούς Users και
διατηρούνται ξεχωριστά για ασφάλεια τύπων.

Η δομή του αντικειμένου SSOUser είναι η εξής:

[inline-code-attrs-start title = 'Δομή SSOUser'; type = 'typescript'; isFunctional = false; inline-code-attrs-end]
[inline-code-start]
interface SSOUser {
    id: string
    username: string
    email?: string
    websiteUrl?: string
    signUpDate: number
    createdFromUrlId?: string
    loginCount?: number
    avatarSrc?: string
    optedInNotifications?: boolean
    optedInSubscriptionNotifications?: boolean
    displayLabel?: string
    displayName?: string
    isAccountOwner?: boolean // Δικαίωμα διαχειριστή - Οι SSO χρήστες με αυτήν τη σημαία χρεώνονται ως SSO Admins (ξεχωριστά από τους κανονικούς SSO χρήστες)
    isAdminAdmin?: boolean // Δικαίωμα διαχειριστή - Οι SSO χρήστες με αυτήν τη σημαία χρεώνονται ως SSO Admins (ξεχωριστά από τους κανονικούς SSO χρήστες)
    isCommentModeratorAdmin?: boolean // Δικαίωμα moderator - Οι SSO χρήστες με αυτήν τη σημαία χρεώνονται ως SSO Moderators (ξεχωριστά από τους κανονικούς SSO χρήστες)
    /** Εάν null, ο Έλεγχος Πρόσβασης δεν θα εφαρμοστεί στον χρήστη. Εάν είναι κενή λίστα, αυτός ο χρήστης δεν θα μπορεί να δει καμία σελίδα ή να κάνει @mention άλλους χρήστες. **/
    groupIds?: string[] | null
    createdFromSimpleSSO?: boolean
    /** Μην επιτρέπετε σε άλλους χρήστες να βλέπουν τη δραστηριότητα αυτού του χρήστη, συμπεριλαμβανομένων των σχολίων, στο προφίλ του. Η προεπιλογή είναι true για να παρέχονται ασφαλή προφίλ από προεπιλογή. **/
    isProfileActivityPrivate?: boolean
    /** Μην επιτρέπετε σε άλλους χρήστες να αφήνουν σχόλια στο προφίλ του χρήστη ή να βλέπουν υπάρχοντα σχόλια προφίλ. Η προεπιλογή είναι false. **/
    isProfileCommentsPrivate?: boolean
    /** Μην επιτρέπετε σε άλλους χρήστες να στέλνουν άμεσα μηνύματα σε αυτόν τον χρήστη. Η προεπιλογή είναι false. **/
    isProfileDMDisabled?: boolean
    karma?: number
    /** Προαιρετική ρύθμιση για τα badges του χρήστη. **/
    badgeConfig?: {
        /** Πίνακας με IDs badges που θα εκχωρηθούν στον χρήστη. Περιορίζεται σε 30 badges. Η σειρά τηρείται. Αυτά είναι παγκόσμια badges ορατά σε όλες τις σελίδες. **/
        badgeIds: string[]
        /** Πίνακας με IDs badges που είναι περιορισμένα στην τρέχουσα σελίδα (urlId). Αυτά τα badges εμφανίζονται μόνο στη σελίδα όπου ανατέθηκαν. **/
        pageBadgeIds?: string[]
        /** Εάν true, αντικαθιστά όλα τα υπάρχοντα εμφανιζόμενα badges με τα παρεχόμενα. Τα παγκόσμια και σελίδας badges υπερκαλύπτονται ανεξάρτητα. Εάν false, προσθέτει στα υπάρχοντα badges. **/
        override?: boolean
        /** Εάν true, ενημερώνει τις ιδιότητες εμφάνισης των badges από τη διαμόρφωση του tenant. **/
        update?: boolean
    }
}

### Χρέωση για χρήστες SSO

Οι χρήστες SSO χρεώνονται διαφορετικά βάσει των δικαιωμάτων τους:

- **Κανονικοί χρήστες SSO**: Χρήστες χωρίς δικαιώματα admin ή moderator χρεώνονται ως κανονικοί χρήστες SSO
- **SSO Admins**: Χρήστες με τις σημαίες `isAccountOwner` ή `isAdminAdmin` χρεώνονται ξεχωριστά ως SSO Admins (ίδιο ποσοστό με τους κανονικούς tenant admins)
- **SSO Moderators**: Χρήστες με τη σημαία `isCommentModeratorAdmin` χρεώνονται ξεχωριστά ως SSO Moderators (ίδιο ποσοστό με τους κανονικούς moderators)

**Σημαντικό**: Για να αποφευχθεί η διπλή χρέωση, το σύστημα αυτόματα αφαιρεί διπλότυπα SSO χρηστών σε σχέση με τους κανονικούς tenant users και moderators βάσει της διεύθυνσης email. Εάν ένας SSO χρήστης έχει την ίδια email με έναν κανονικό tenant user ή moderator, δεν θα χρεωθεί δύο φορές.

### Έλεγχος Πρόσβασης

Οι χρήστες μπορούν να χωριστούν σε ομάδες. Αυτός είναι ο λόγος ύπαρξης του πεδίου `groupIds`, και είναι προαιρετικό.

### @Mentions

Κατά προεπιλογή τα `@mentions` θα χρησιμοποιούν το `username` για να αναζητήσουν άλλους SSO χρήστες όταν πληκτρολογηθεί ο χαρακτήρας `@`. Εάν χρησιμοποιείται το `displayName`, τότε τα αποτελέσματα που ταιριάζουν με το `username` θα αγνοηθούν όταν υπάρχει αντιστοιχία για το `displayName`, και τα αποτελέσματα αναζήτησης του `@mention` θα χρησιμοποιούν το `displayName`.

### Συνδρομές

Με το FastComments, οι χρήστες μπορούν να εγγραφούν σε μια σελίδα κάνοντας κλικ στο εικονίδιο του κουδουνιού στο widget σχολίων και επιλέγοντας Εγγραφή.

Με έναν κανονικό χρήστη, τους στέλνουμε ειδοποιητικά email βάσει των ρυθμίσεων ειδοποιήσεων τους.

Για τους χρήστες SSO, το διαιρούμε αυτό για συμβατότητα προς τα πίσω. Οι χρήστες θα λαμβάνουν αυτά τα επιπλέον emails ειδοποιήσεων συνδρομής μόνο εάν ορίσετε `optedInSubscriptionNotifications` σε `true`.

### Badges

Μπορείτε να αναθέσετε badges σε SSO χρήστες χρησιμοποιώντας την ιδιότητα `badgeConfig`. Τα badges είναι οπτικοί δείκτες που εμφανίζονται δίπλα στο όνομα του χρήστη στα σχόλια.

- `badgeIds` - Πίνακας με IDs badge που θα εκχωρηθούν στον χρήστη. Αυτά είναι παγκόσμια badges ορατά σε όλες τις σελίδες. Πρέπει να είναι έγκυρα IDs badge που έχουν δημιουργηθεί στον λογαριασμό σας στο FastComments. Περιορίζονται σε 30 badges.
- `pageBadgeIds` - Προαιρετικός πίνακας με IDs badge περιορισμένων στην τρέχουσα σελίδα (`urlId`). Αυτά τα badges εμφανίζονται μόνο στη σελίδα όπου ανατέθηκαν. Διάφορες σελίδες μπορούν να έχουν διαφορετικά badges περιορισμένα στη σελίδα για τον ίδιο χρήστη.
- `override` - Εάν true, όλα τα υπάρχοντα εμφανιζόμενα badges θα αντικατασταθούν με τα παρεχόμενα. Τα παγκόσμια και τα περιορισμένα σε σελίδα badges υπερκαλύπτονται ανεξάρτητα — η υπερκάλυψη των παγκόσμιων badges δεν επηρεάζει τα σελίδας badges, και αντίστροφα. Εάν false ή παραλειφθεί, τα παρεχόμενα badges θα προστεθούν σε οποιαδήποτε υπάρχοντα badges.
- `update` - Εάν true, οι ιδιότητες εμφάνισης των badges θα ενημερώνονται από τη διαμόρφωση του tenant κάθε φορά που ο χρήστης συνδέεται.
