Блок **FastComments** — це основний віджет коментарів. Додайте його до шаблонів публікацій блогу, шаблонів товарів або будь-якої іншої сторінки, де ви хочете мати дискусію або живий чат.

### Додати блок

1. Відкрийте редактор тем Shopify (**Online Store > Themes > Customize**).
2. Оберіть шаблон, на якому ви хочете мати коментарі: **Blog post**, **Product**, або будь-який інший шаблон сторінки чи секції.
3. У секції, де ви хочете, щоб з\'являлися коментарі, натисніть **Add block**.
4. У розділі **Apps** оберіть **FastComments**.
5. Натисніть **Save**.

Блок з\'являється негайно. Вводити Tenant ID не потрібно; tenant вашого магазину підключається автоматично під час встановлення додатку.

### Налаштування

| Налаштування | Що робить | За замовчуванням |
|---|---|---|
| Tenant ID (optional) | Перевизначає, до якого tenant FastComments рендериться блок. Залиште порожнім, щоб використовувати tenant, налаштований автоматично для магазину. Знайдіть ручний Tenant ID на fastcomments.com/auth/my-account/api-secret. | (порожньо) |
| SSO | Автоматично входить відвідувача як його обліковий запис клієнта Shopify перед коментуванням. Див. [Автоматичний вхід клієнтів Shopify](/guide-installation-shopify.html#shopify-sso). | Увімкнено |
| Commenting Style | **Вкладена** для вкладених відповідей і голосувань, або **Потокова** для реального часу у вигляді чату. | Вкладена |
| Custom URL ID | Перевизначає автоматично виявлений ідентифікатор сторінки. Використовуйте це, коли ви хочете, щоб дві URL поділяли одну нитку коментарів. | (визначається автоматично) |

### Як обирається ідентифікатор сторінки

Кожна нитка коментарів ідентифікується за URL ID. Блок обирає його автоматично:

- **Blog post template:** `shopify-article-{article.id}`, який залишається сталим при зміні slug або заголовка.
- **Product template:** `shopify-product-{product.id}`, який залишається сталим при зміні slug або заголовка.
- **Інші шаблони:** шлях запиту.

Якщо ви встановите **Custom URL ID**, буде використано це значення. Використовуйте однаковий Custom URL ID у кількох блоках (наприклад, на локалізованій версії сторінки товару), щоб спільно використовувати одну нитку коментарів.

### Вкладена (Threaded) проти Потокової (Streaming)

**Вкладена** — це поведінка за замовчуванням. Відвідувачі відповідають один одному, голосують, а інструменти модерації працюють як очікується. Найкраще підходить для публікацій у блогах і оглядів товарів.

**Потокова** прибирає вкладеність і показує нові коментарі в реальному часі по мірі їх публікації, як у стрічці чату. Найкраще підходить для запусків продуктів, живих подій і спільнот.

### Кілька блоків на одній сторінці

Блок можна додати більше одного разу до того самого шаблону. Наприклад, підсумок відгуків у верхній частині сторінки товару й блок FastComments унизу. Блоки ділять URL ID, тому підсумок відображає коментарі знизу.

### Поради

- Блок ховається у попередньому перегляді редактора тем із жовтим повідомленням, якщо не може знайти tenant. Якщо це з\'являється у вашому живому магазині, перевстановіть додаток FastComments.
- Для сторінки товару блок FastComments одночасно виконує роль віджета відгуків про товар. Поєднайте його з **FastComments - Reviews Summary** для підсумку рейтингу зірочками у верхній частині сторінки.
