FastComments позволяет требовать от пользователей, комментирующих впервые, принятия Условий обслуживания перед отправкой комментария.

Когда включено:
- **Анонимные пользователи** будут видеть флажок принятия Условий обслуживания при каждом добавлении комментария
- **Аутентифицированные пользователи** увидят флажок только при первом комментарии или когда вы обновите Условия обслуживания

### Конфигурация

Перейдите на страницу настройки виджета и включите флажок "Требовать принятия Условий обслуживания". После включения вы увидите следующие опции:

[app-screenshot-start url='/auth/my-account/customize-widget/new'; clickSelector = '.tos-enabled'; selector = '#tos-options'; title='Terms of Service Options' app-screenshot-end]

- **Режим текста Условий обслуживания**: По умолчанию флажок отображает "Я согласен с Условиями обслуживания и Политикой конфиденциальности" со ссылками на оба документа. Выберите "Customize text per locale", чтобы предоставить собственный текст для каждого языка.
- **Дата последнего обновления Условий обслуживания**: Когда вы обновляете Условия обслуживания, установите эту дату. Пользователи, которые приняли до этой даты, будут обязаны принять повторно.

### Как это работает

- Отметка времени принятия Условий хранится для каждого пользователя и для каждого комментария
- Когда пользователь принимает Условия обслуживания, дата записывается в его профиль пользователя (на уровне тенанта)
- Если вы установите дату «Последнего обновления», которая позже даты принятия пользователем, ему потребуется принять снова
- Для анонимных пользователей, которых нельзя отследить, флажок появляется при каждой отправке комментария
