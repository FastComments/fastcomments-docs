Esses componentes são encapsuladores em torno dos nossos componentes VanillaJS principais. Podemos atualizar automaticamente esses componentes (corrigir bugs, adicionar recursos) sem publicar esta biblioteca, então, embora ela possa não ser publicada por um tempo, isso não significa que o FastComments não esteja em desenvolvimento ativo! Fique à vontade para conferir [nosso blog](https://blog.fastcomments.com/) para atualizações. Mudanças incompatíveis na API ou novos recursos nunca serão enviados para a biblioteca principal subjacente sem um aumento de versão nesta biblioteca.
