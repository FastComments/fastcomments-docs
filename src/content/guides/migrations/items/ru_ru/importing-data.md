Пока служба поддержки FastComments может помочь с миграциями, большинство из них можно выполнить и отслеживать легко без вмешательства
сотрудников поддержки.

Мы нативно поддерживаем импорт экспортов из следующих провайдеров:

- Commento
- Disqus
- Hyvor Talk
- Muut Comments
- IntenseDebate
- Just-Comments
- WordPress (via the plugin)

Перейдя по [ссылке](https://fastcomments.com/auth/my-account/manage-data/import) мы можем загрузить файл, содержащий данные для миграции.

[app-screenshot-start url='/auth/my-account/manage-data/import'; selector = '.account-block'; title='The Import Page Form' app-screenshot-end]

### Отслеживание импортов

FastComments использует систему обработки задач для обработки импортов и экспортов. Как только система возьмёт вашу задачу, она будет
периодически сообщать о статусе задачи в интерфейсе импорта или экспорта.

[app-screenshot-start url='/auth/my-account/manage-data/import?demo=true'; selector = '.content'; title='Import Job Status' app-screenshot-end]

Обратите внимание, что статусы импортов и экспортов доступны для просмотра всем администраторам в аккаунте.

Если ваша задача завершится неудачно, она не будет автоматически перезапущена. Импорт придётся попытаться выполнить снова. Если какой-либо импорт или экспорт завершится неудачно,
наши системные администраторы будут автоматически уведомлены. Если мы обнаружим проблему, мы свяжемся с вами, чтобы выяснить, можем ли мы помочь.

### Повторный запуск импорта

Во время некоторых миграций необходимо запускать импорт несколько раз. Например, часто выполняют первоначальную миграцию для тестирования, а затем запускают импорт снова с актуальными данными перед переключением.

Повторный импорт того же содержимого **не создаст дубликатов**.

### Безопасность данных и срок хранения

Файлы импорта недоступны извне никаким образом, и файлы импорта удаляются из нашей системы, как только
импорт завершится.
