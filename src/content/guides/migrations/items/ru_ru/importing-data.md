Хотя служба поддержки FastComments может помочь с миграциями, большинство из них можно выполнить и отслеживать легко без какого-либо вмешательства
сотрудников поддержки.

Мы нативно поддерживаем импорт экспортов от следующих провайдеров:

- Commento
- Disqus
- Hyvor Talk
- Muut Comments
- IntenseDebate
- Just-Comments
- WordPress (через плагин)
- AnyComment (Через WordPress Import/Export)

Перейдя по [этой ссылке](https://fastcomments.com/auth/my-account/manage-data/import) мы можем загрузить файл, содержащий данные для миграции.

[app-screenshot-start url='/auth/my-account/manage-data/import'; selector = '.account-block'; title='The Import Page Form' app-screenshot-end]

### Мониторинг импортов

FastComments использует систему обработки задач для обработки импортов и экспортов. Как только система возьмёт вашу задачу в обработку, она будет
периодически сообщать статус задачи в интерфейсе импорта или экспорта.

[app-screenshot-start url='/auth/my-account/manage-data/import?demo=true'; selector = '.content'; title='Import Job Status' app-screenshot-end]

Обратите внимание, что статусы импортов и экспортов доступны для просмотра всем администраторам в аккаунте.

Если ваша задача завершится с ошибкой, она не будет автоматически перезапущена. Импорт придётся попытаться выполнить снова. Если какой-либо импорт или экспорт завершится с ошибкой,
наши системные администраторы будут автоматически уведомлены. Если мы обнаружим проблему, мы свяжемся с вами, чтобы узнать, можем ли мы помочь.

### Повторный запуск импорта

Во время некоторых миграций необходимо запускать импорт несколько раз. Например, часто сначала выполняют первичный проход
миграции для тестирования, а затем запускают импорт снова с последними данными перед переключением.

Повторный импорт того же содержимого **не создаст дубликатов**.

### Безопасность данных и срок хранения

Файлы импорта никоим образом не доступны извне, и файлы импорта удаляются из нашей системы, как только
импорт завершается.
