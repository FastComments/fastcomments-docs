Ces composants servent de wrappers pour nos composants VanillaJS core. Nous pouvons mettre automatiquement à jour ces composants (corriger des bugs, ajouter des fonctionnalités) sans publier cette bibliothèque, donc même s'il se peut qu'elle ne soit pas publiée pendant un certain temps, cela ne signifie pas que FastComments n'est pas en cours de développement actif ! N'hésitez pas à consulter [notre blog](https://blog.fastcomments.com/) pour des mises à jour. Les modifications incompatibles de l'API ou les nouvelles fonctionnalités ne seront jamais intégrées à la bibliothèque core sous-jacente sans une augmentation de version dans cette bibliothèque.
