Для SSO нужно учитывать следующую конфигурацию для уведомлений:

- Подписан ли пользователь на уведомления.
  - Для этого в объекте `SSOUser` устанавливается флаг `optedInNotifications` в `true` или `false`.
  - Это можно задать через API.
  - Кроме того, если вы передаёте значение этого флага в полезной нагрузке, оно автоматически обновится, когда пользователь загрузит поток комментариев.
- Подписан ли пользователь на уведомления о **подписке**.
  - Для этого в объекте `SSOUser` устанавливается флаг `optedInSubscriptionNotifications` в `true` или `false`.
  - Это можно задать через API.
  - Кроме того, если вы передаёте значение этого флага в полезной нагрузке, оно автоматически обновится, когда пользователь загрузит поток комментариев.
- Указание их электронной почты.
  - Если она отсутствует, мы не сможем отправлять уведомления по электронной почте.
- Отключать ли ссылки для отписки в письмах.
  - Это задаётся через флаг `disableUnsubscribeLinks` в объекте `Tenant`.
  - Это можно задать через API.
- Использовать ли пользовательскую ссылку для отписки.
  - Это задаётся через свойство `footerUnsubscribeURL` в объекте `DomainConfig`.
  - Это можно задать через API.
  - Также вы можете рассмотреть возможность установки соответствующих заголовков отписки через `emailHeaders` в том же объекте.
