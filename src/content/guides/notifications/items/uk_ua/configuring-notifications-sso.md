Для SSO слід врахувати таку конфігурацію для сповіщень:

- Чи підписався користувач на отримання сповіщень.
  - Це робиться шляхом встановлення прапорця `optedInNotifications` в `SSOUser` в значення `true` або `false`.
  - Це можна встановити через API.
  - Також, якщо ви передасте значення для цього прапорця в payload, воно автоматично оновиться, коли користувач завантажить потік коментарів.
- Чи підписався користувач на **сповіщення про підписку**.
  - Це робиться шляхом встановлення прапорця `optedInSubscriptionNotifications` в `SSOUser` в значення `true` або `false`.
  - Це можна встановити через API.
  - Також, якщо ви передасте значення для цього прапорця в payload, воно автоматично оновиться, коли користувач завантажить потік коментарів.
- Вказання їхньої електронної пошти.
  - Якщо вона відсутня, ми не можемо надсилати сповіщення електронною поштою.
- Чи вимкнути посилання для відписки у листах.
  - Це робиться через прапорець `disableUnsubscribeLinks` в об'єкті `Tenant`.
  - Це можна встановити через API.
- Чи використовувати власне посилання для відписки.
  - Це робиться через властивість `footerUnsubscribeURL` в об'єкті `DomainConfig`.
  - Це можна встановити через API.
  - Ви також можете розглянути налаштування відповідних заголовків для відписки через `emailHeaders` у тому ж об'єкті.
