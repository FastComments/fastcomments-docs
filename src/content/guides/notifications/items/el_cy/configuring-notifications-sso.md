Για το SSO υπάρχει η εξής διαμόρφωση που πρέπει να ληφθεί υπόψη για τις ειδοποιήσεις:

- Εάν ο χρήστης έχει επιλέξει να λαμβάνει ειδοποιήσεις.
  - Αυτό γίνεται ορίζοντας τη σημαία `optedInNotifications` σε `true` ή `false` στο αντικείμενο `SSOUser`.
  - Αυτό μπορεί να οριστεί μέσω του API.
  - Επίσης, αν περάσετε μια τιμή για αυτή τη σημαία στο payload, θα ενημερώνεται αυτόματα όταν ο χρήστης φορτώσει ένα νήμα σχολίων.
- Εάν ο χρήστης έχει επιλέξει να λαμβάνει ειδοποιήσεις **εγγραφής**.
  - Αυτό γίνεται ορίζοντας τη σημαία `optedInSubscriptionNotifications` σε `true` ή `false` στο αντικείμενο `SSOUser`.
  - Αυτό μπορεί να οριστεί μέσω του API.
  - Επίσης, αν περάσετε μια τιμή για αυτή τη σημαία στο payload, θα ενημερώνεται αυτόματα όταν ο χρήστης φορτώσει ένα νήμα σχολίων.
- Ορισμός του email τους.
  - Αν δεν υπάρχει, δεν μπορούμε να στείλουμε ειδοποιήσεις μέσω email.
- Εάν θα απενεργοποιηθούν οι σύνδεσμοι απεγγραφής (unsubscribe) στα email.
  - Αυτό γίνεται μέσω της σημαίας `disableUnsubscribeLinks` στο αντικείμενο `Tenant`.
  - Αυτό μπορεί να οριστεί μέσω του API.
- Εάν θα χρησιμοποιηθεί προσαρμοσμένος σύνδεσμος απεγγραφής.
  - Αυτό γίνεται μέσω της ιδιότητας `footerUnsubscribeURL` στο αντικείμενο `DomainConfig`.
  - Αυτό μπορεί να οριστεί μέσω του API.
  - Ενδέχεται επίσης να θελήσετε να εξετάσετε τον ορισμό των σχετικών κεφαλίδων απεγγραφής μέσω του `emailHeaders` στο ίδιο αντικείμενο.
