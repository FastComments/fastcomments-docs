Deze componenten zijn wrappers rond onze kern-VanillaJS-componenten. We kunnen deze componenten automatisch bijwerken (bugs oplossen, functies toevoegen) zonder deze bibliotheek te publiceren, dus hoewel het misschien een tijdje niet gepubliceerd wordt, betekent dat niet dat FastComments niet actief in ontwikkeling is! Voel je vrij om [onze blog](https://blog.fastcomments.com/) te bekijken voor updates. Wijzigingen of functies die de API breken zullen nooit naar de onderliggende kernbibliotheek worden uitgebracht zonder een versienummerverhoging in deze bibliotheek.
