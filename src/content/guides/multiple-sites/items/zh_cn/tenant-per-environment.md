在 FastComments 中，通常会为每个测试或开发环境创建一个子租户。每个租户将拥有自己的配置、数据和 API 密钥。配置、数据和用户不能在租户之间共享。
一切都是相互独立的。不过，父租户的超级管理员可以模拟子租户中的用户。

有两种方法：

- 主租户用于生产环境，而子租户用于测试环境。
- 主租户仅用于计费，每个子租户用于 prod、test 等等。

第一种通常更容易让用户理解，但这可能取决于贵组织的情况。

如果您拥有相应的套餐，可以在 [here](https://eu.fastcomments.com/auth/my-account/tenants) 创建租户。这也是超级管理员模拟用户的地方。租户也可以通过 API 创建，以实现更自定义/自动化的设置。

无论采取哪种方法，您都必须将希望查看生产数据的版主和用户添加到 "prod" 租户中。因此，例如如果您想采用方案 B，将父租户用于计费，并为 "prod" 配置一个子租户，则需要添加租户、切换到新租户，然后为该子租户添加您的管理员和版主用户。

最后，为了说明清楚，在方案 B 下父租户的审核评论页面将是空的。
