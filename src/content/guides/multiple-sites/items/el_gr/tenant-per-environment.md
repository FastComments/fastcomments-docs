Είναι συνηθισμένο να υπάρχει ένας sub-tenant για κάθε test ή dev περιβάλλον με το FastComments. Κάθε tenant θα έχει τη δική του διαμόρφωση, δεδομένα και API keys. Οι ρυθμίσεις, τα δεδομένα και οι χρήστες δεν μπορούν να μοιράζονται μεταξύ tenants.
Όλα είναι απομονωμένα. Ωστόσο, οι super admins του γονικού tenant μπορούν να υποδυθούν (impersonate) χρήστες σε child tenants.

Υπάρχουν δύο προσεγγίσεις:

- Ο κύριος tenant προορίζεται για production, και οι sub-tenants για περιβάλλοντα δοκιμών.
- Ο κύριος tenant είναι απλώς για billing, και κάθε sub-tenant είναι για prod, test κ.ο.κ.

Η πρώτη προσέγγιση είναι γενικά πιο εύκολη για τους χρήστες να την κατανοήσουν, αλλά αυτό μπορεί να εξαρτάται από την οργάνωσή σας.

Οι tenants μπορούν να δημιουργηθούν [εδώ](https://eu.fastcomments.com/auth/my-account/tenants) εάν έχετε το πακέτο. Αυτό είναι επίσης το σημείο όπου οι super admins θα υποδύονται χρήστες. Οι tenants μπορούν επίσης να δημιουργηθούν μέσω του API για πιο προσαρμοσμένες/αυτοματοποιημένες ρυθμίσεις.

Ανεξάρτητα από την προσέγγιση που θα ακολουθήσετε, θα πρέπει να προσθέσετε τους moderators και τους χρήστες που θέλουν να βλέπουν δεδομένα παραγωγής στον tenant "prod". Έτσι, για παράδειγμα, αν θέλετε να επιλέξετε την επιλογή B και να έχετε τον parent tenant για billing, και έναν sub-tenant για το "prod", θα πρέπει να προσθέσετε τον tenant, να μεταβείτε στον νέο tenant και να προσθέσετε τους admin και moderator χρήστες για το sub-tenant.

Τέλος, για διευκρίνιση, η σελίδα Moderate Comments θα είναι κενή με την επιλογή B για τον parent tenant.
