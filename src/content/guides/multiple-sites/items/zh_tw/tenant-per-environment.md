在 FastComments 中，為每個測試或開發環境建立子租戶是很常見的做法。每個租戶都有自己的設定、資料和 API 金鑰。設定、資料和使用者無法在租戶之間共用。
一切都是隔離的。然而，父租戶的超級管理員可以冒充子租戶中的使用者。

有兩種做法：

- 主要租戶用於生產環境，子租戶用於測試環境。
- 主要租戶僅用於計費，每個子租戶則用於生產、測試等。

第一種通常較容易讓使用者理解，但這可能取決於你的組織。

如果你有該套件，可以在 [這裡](https://eu.fastcomments.com/auth/my-account/tenants) 建立租戶。這裡也是超級管理員會
冒充使用者的地方。租戶也可以透過 API 建立，以便進行更客製或自動化的設置。

無論採用哪種做法，你都必須在「prod」租戶中新增想要查看生產資料的管理員與使用者。所以例如，如果你想採用選項 B，並把父租戶用於計費，然後有一個名為「prod」的子租戶，你會需要新增該子租戶、切換到新租戶，並為該子租戶新增你的管理員和版主使用者。

最後說明，對於選項 B，父租戶的「審核留言」頁面將會是空的。
