FastComments에서는 테스트 또는 개발 환경마다 하위 테넌트를 두는 것이 일반적입니다. 각 테넌트는 자체 구성, 데이터 및 API 키를 가집니다. 구성, 데이터 및 사용자는 테넌트 간에 공유될 수 없습니다.
모든 것이 격리됩니다. 다만, 상위 테넌트의 슈퍼 관리자는 하위 테넌트의 사용자를 가장할 수 있습니다.

두 가지 접근 방식이 있습니다:

- 메인 테넌트는 프로덕션용이고, 하위 테넌트는 테스트 환경용입니다.
- 메인 테넌트는 단순히 청구용이며, 각 하위 테넌트는 프로덕션, 테스트 등으로 사용됩니다.

첫 번째 방식이 일반적으로 사용자가 이해하기 더 쉽지만, 조직에 따라 다를 수 있습니다.

테넌트는 패키지를 보유하고 있다면 [here](https://eu.fastcomments.com/auth/my-account/tenants)에서 생성할 수 있습니다. 이곳이 또한 슈퍼 관리자가
사용자를 가장하는 곳입니다. 테넌트는 더 맞춤형/자동화된 설정을 위해 API를 통해서도 생성할 수 있습니다.

어떤 접근 방식을 택하든, 프로덕션 데이터를 보려는 관리자와 사용자들을 "prod" 테넌트에 추가해야 합니다. 예를 들어 옵션 B를 선택하여 상위 테넌트를 청구용으로 두고 "prod"용 하위 테넌트를 만들려면, 테넌트를 추가하고 새 테넌트로 전환한 다음 하위 테넌트의 관리자 및 중재자 사용자를 추가해야 합니다. 

마지막으로 명확히 하자면, 옵션 B를 사용할 경우 상위 테넌트의 Moderate Comments 페이지는 비어 있게 됩니다.
